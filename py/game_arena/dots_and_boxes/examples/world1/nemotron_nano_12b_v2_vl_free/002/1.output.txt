To solve this problem, we need to create an AI that can effectively play the game of Dots and Boxes on a 4x4 grid. The goal is to select the next move in a way that maximizes the number of boxes captured while minimizing the opportunities for the opponent to capture their own boxes. The solution involves evaluating each potential move based on its immediate impact and potential to create advantageous positions.

### Approach
1. **Generate Legal Moves**: Identify all possible moves by checking which horizontal and vertical edges are still available (i.e., not already drawn).
2. **Evaluate Moves**: For each potential move, determine which boxes it could complete and how many three-edge boxes it would create. This helps in assessing the move's immediate impact and potential future consequences.
3. **Prioritize Capturing Moves**: If a move completes one or more boxes, prioritize such moves, especially those that complete the maximum number of boxes.
4. **Avoid Three-Edge Boxes**: If no capturing moves are available, choose moves that create the least number of three-edge boxes, which the opponent could easily capture next.
5. **Tie-Breaking**: If multiple moves have the same priority, they are sorted by their position (row, column) and edge direction (preferring horizontal edges).

    for r in range(5):
        for c in range(5):
            if horizontal[r, c] == 0:
                possible_moves.append((r, c, 'H'))
    for r in range(5):
        for c in range(5):
            if vertical[r, c] == 0:
                possible_moves.append((r, c, 'V'))

    captures = {}
    three_edge = {}
    for move in possible_moves:
        r, c, d = move
        boxes = get_adjacent_boxes(r, c, d)
        captured = 0
        three_edges_count = 0
        for (i, j) in boxes:
            sum_before = 0
            # Check top edge (box's top edge is horizontal[i][j])
            if not (d == 'H' and r == i and c == j):
                sum_before += horizontal[i, j] != 0
            # Check right edge (box's right edge is vertical[i][j+1])
            if not (d == 'V' and r == i and c == j + 1):
                sum_before += vertical[i, j + 1] != 0
            # Check bottom edge (box's bottom edge is horizontal[i+1][j])
            if not (d == 'H' and r == i + 1 and c == j):
                sum_before += horizontal[i + 1, j] != 0
            # Check left edge (box's left edge is vertical[i][j])
            if not (d == 'V' and r == i and c == j):
                sum_before += vertical[i, j] != 0
            # Determine if this move completes the box or creates a three-edge box
            if sum_before == 3:
                captured += 1
            if sum_before == 2:
                three_edges_count += 1
        # Update dictionaries
        captures[move] = captured
        three_edge[move] = three_edges_count

    # Determine capturing moves and select best move
    capturing_moves = [m for m in possible_moves if captures[m] > 0]
    if capturing_moves:
        max_caps = max(captures.values())
        candidates = [m for m in possible_moves if captures[m] == max_caps]
        candidates.sort(key=lambda m: (m[0], m[1], m[2] == 'V'))
        return f"{candidates[0][0]},{candidates[0][1]},{candidates[0][2]}"
    else:
        # Find move with minimum three-edge boxes created
        min_three = min(three_edge.values())
        candidates = [m for m in possible_moves if three_edge[m] == min_three]
        candidates.sort(key=lambda m: (m[0], m[1], m[2] == 'V'))
        return f"{candidates[0][0]},{candidates[0][1]},{candidates[0][2]}"

### Explanation
- **Legal Moves Generation**: The code iterates through all possible horizontal and vertical edges to collect those that are still available (value 0 in their respective arrays).
- **Box Evaluation**: For each potential move, the function checks which adjacent boxes it affects. It calculates how many boxes would be completed by the move and how many three-edge boxes would be created.
- **Move Selection**: The policy prioritizes moves that complete the most boxes. If no capturing moves are available, it selects moves that create the fewest three-edge boxes, minimizing opportunities for the opponent. The selection is further refined by sorting candidates by row and column, preferring horizontal edges when there's a tie.

This approach ensures the AI makes strategic moves that either capture boxes immediately or minimize the opponent's opportunities to capture boxes, thus increasing its chances of winning.
