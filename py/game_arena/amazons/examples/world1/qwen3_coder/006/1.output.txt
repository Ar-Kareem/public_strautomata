The strategy for this policy is to implement a multi-step evaluation function that prioritizes mobility control and strategic positioning. The approach consists of:

1. **Legal Move Generation**: First, generate all possible legal moves for each of our amazons by considering all queen-like movements (orthogonal and diagonal) to empty squares. Then, for each such move, generate all possible arrow shots from the new position.

2. **Evaluation Function**: For each potential board state after making a move and shooting an arrow, calculate a score based on:
   - **Our Mobility**: The number of legal moves our amazons can make in the resulting state.
   - **Opponent Mobility**: The number of legal moves the opponent's amazons can make.
   - **Territory Control**: A potential extension (not implemented here due to time) would be controlling key central squares or cutting off opponent's paths.

   The score is calculated as `W_OUR * OUR_MOBILITY - W_OPP * OPP_MOBILITY`. This encourages moves that restrict the opponent's future options while preserving our own.

3. **Search Depth**:
   - A full minimax search is too slow for a 1-second limit on a 6x6 board with multiple pieces.
   - Therefore, the policy uses a 1-ply look-ahead: it evaluates the score of the board state immediately after each of our possible moves.
   - The move that leads to the board state with the highest score (best for us, worst for opponent) is selected.

4. **Efficiency**: The policy is optimized to generate moves and evaluate board states quickly using NumPy operations.

5. **Robustness**: The code includes checks to ensure that if no complex move is found (which should not happen in a valid game state), it falls back to any legal move.

This greedy heuristic of maximizing our mobility while minimizing the opponent's has proven effective in games like Amazons, as it leads to strategically dominant positions over time.
