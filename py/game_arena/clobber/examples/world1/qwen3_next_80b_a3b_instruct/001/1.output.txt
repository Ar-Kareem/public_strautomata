The strategy is to implement a simple but effective policy that prioritizes moves that maximize immediate capture opportunities while avoiding unnecessary risks. Since the game is Clobber (a two-player impartial game), a greedy approach that captures an opponent piece whenever possible is sound, especially given the constraint that we must always return a legal move.

The algorithm:
1. Scan the entire board to find all your pieces (where `you[row, col] == 1`).
2. For each of your pieces, check all four orthogonal directions (up, right, down, left).
3. For each direction, compute the destination coordinates and check if it's within bounds and contains an opponent piece.
4. Among all legal moves, choose one. For simplicity and efficiency, we'll pick the first legal move found when scanning row by row, left to right, and in the order U, R, D, L.

This greedy "first found" policy is fast, guarantees a legal move (as per problem assumption), and avoids complex search (which is unnecessary under time constraints and because the board is small). The order of checking directions (U, R, D, L) provides a deterministic and consistent behavior.

We'll use simple nested loops over rows (0-4) and columns (0-5), and check directions in fixed order.

import numpy as np

def policy(you: list[int], opponent: list[int]) -> str:
    # Convert lists to 2D numpy arrays for easier indexing
    you = np.array(you).reshape(5, 6)
    opponent = np.array(opponent).reshape(5, 6)
    
    # Define direction offsets: U, R, D, L
    directions = [(-1, 0), (0, 1), (1, 0), (0, -1)]
    dir_names = ['U', 'R', 'D', 'L']
    
    # Scan board row by row, column by column
    for row in range(5):
        for col in range(6):
            # Check if this cell has your piece
            if you[row, col] == 1:
                # Check each direction
                for idx, (dr, dc) in enumerate(directions):
                    new_row, new_col = row + dr, col + dc
                    # Check bounds
                    if 0 <= new_row < 5 and 0 <= new_col < 6:
                        # Check if opponent piece is there
                        if opponent[new_row, new_col] == 1:
                            return f"{row},{col},{dir_names[idx]}"
    
    # According to the problem, there is always at least one legal move.
    # So we should never reach here.
    # But for completeness and safety, return a dummy move (should not happen).
    return "0,0,U"
