### Thinking Process

**Understanding Clobber:**
Clobber is a two-player game where players take turns moving their pieces to capture adjacent opponent pieces. The board is 5x6, and each cell can contain either your piece, the opponent's piece, or be empty. A legal move requires:
1. Moving one of your pieces orthogonally (U, R, D, L) onto an opponent's piece.
2. The destination must be within the board bounds.

**Key Observations:**
1. **Legal Moves:** The function `policy` is only called when there are legal moves available. Thus, we don't need to handle cases where no moves exist.
2. **Move Selection:** The goal is to define a move-selection policy that always returns a legal move. Since the problem doesn't specify a winning strategy or optimization criteria (e.g., maximizing captures, minimizing opponent's moves, etc.), we can focus on a simple but effective approach:
   - **Greedy Capture:** Prioritize moves that capture opponent pieces in a way that maximizes the number of future moves for yourself while minimizing the opponent's options. For example, capturing pieces that are surrounded by more of your pieces could be beneficial.
   - **Randomness:** To avoid predictable behavior (which might be exploited by an opponent), we can introduce randomness in selecting among legal moves. This is common in simple AI policies to prevent being easily countered.
   - **Fallback:** If no greedy or strategic move is found, default to a random legal move.

**Approach:**
1. **Find All Legal Moves:** Iterate over all your pieces (`you[row][col] == 1`) and check all four possible directions (U, R, D, L) to see if the destination is within bounds and contains an opponent piece.
2. **Score Moves:** Assign a score to each legal move based on some heuristic. For simplicity, we can score moves based on:
   - How many of your pieces are adjacent to the destination cell (higher score if more of your pieces are nearby, as this could lead to more captures in the future).
   - How many opponent pieces are adjacent to the destination cell (lower score if more opponent pieces are nearby, as this could threaten your piece).
3. **Select Best Move:** Choose the move with the highest score. If multiple moves have the same score, break ties randomly.
4. **Fallback to Random:** If no move is found (unlikely per problem statement), default to a random legal move.

**Edge Cases:**
- If all your pieces are isolated (no adjacent opponent pieces), any legal move is valid.
- If multiple moves have the same score, randomness ensures fairness.

**Optimization:**
- The scoring heuristic is simple but can be expanded (e.g., prioritize capturing pieces that are part of larger opponent clusters).
- The code must be efficient to run within 1 second on a 5x6 board.

Here's the code for the `policy` function. It finds all legal moves, scores them based on the heuristic above, and selects the best move. If no move is found (which shouldn't happen per problem statement), it defaults to a random legal move.

import numpy as np
from typing import List
import random

def policy(you: List[List[int]], opponent: List[List[int]]) -> str:
    # Convert to numpy arrays for easier manipulation
    you_board = np.array(you, dtype=int)
    opponent_board = np.array(opponent, dtype=int)

    # Directions: U, R, D, L (up, right, down, left)
    directions = [(-1, 0), (0, 1), (1, 0), (0, -1)]
    dir_chars = ['U', 'R', 'D', 'L']

    legal_moves = []

    # Iterate over all your pieces
    for row in range(5):
        for col in range(6):
            if you_board[row, col] == 1:
                # Check all four directions
                for i, (dr, dc) in enumerate(directions):
                    new_row, new_col = row + dr, col + dc
                    # Check if destination is within bounds and contains opponent piece
                    if 0 <= new_row < 5 and 0 <= new_col < 6 and opponent_board[new_row, new_col] == 1:
                        legal_moves.append((row, col, dir_chars[i]))

    # If no legal moves (shouldn't happen per problem statement)
    if not legal_moves:
        return "0,0,U"  # fallback, though problem says this won't occur

    # Score each legal move: higher if more of your pieces are adjacent to the destination
    scored_moves = []
    for move in legal_moves:
        row, col, dir_char = move
        dr, dc = next((d for d, c in zip(directions, dir_chars) if c == dir_char))
        new_row, new_col = row + dr, col + dc

        # Count adjacent your pieces to the destination (new_row, new_col)
        adjacent_you = 0
        for r in [new_row - 1, new_row, new_row + 1]:
            for c in [new_col - 1, new_col, new_col + 1]:
                if 0 <= r < 5 and 0 <= c < 6 and you_board[r, c] == 1:
                    adjacent_you += 1
        # Subtract the count of adjacent opponent pieces to the destination
        adjacent_opponent = 0
        for r in [new_row - 1, new_row, new_row + 1]:
            for c in [new_col - 1, new_col, new_col + 1]:
                if 0 <= r < 5 and 0 <= c < 6 and opponent_board[r, c] == 1:
                    adjacent_opponent += 1
        # Score is adjacent_you - adjacent_opponent
        score = adjacent_you - adjacent_opponent
        scored_moves.append((score, move))

    # Sort moves by score descending, then by randomness to break ties
    scored_moves.sort(key=lambda x: (-x[0], random.random()))
    best_move = scored_moves[0][1]

    return f"{best_move[0]},{best_move[1]},{best_move[2]}"

### Explanation of the Code:
1. **Board Representation:** The `you` and `opponent` lists are converted to numpy arrays for easier manipulation.
2. **Legal Moves:** For each of your pieces, the code checks all four directions (U, R, D, L) to see if the destination cell is within bounds and contains an opponent piece. Valid moves are stored as `(row, col, dir_char)` tuples.
3. **Scoring:** Each legal move is scored based on how many of your pieces are adjacent to the destination cell (after the move) minus how many opponent pieces are adjacent to the same destination. This heuristic favors moves that leave your pieces in more advantageous positions.
4. **Move Selection:** The moves are sorted by their scores in descending order. If two moves have the same score, the tie is broken randomly to avoid predictable behavior.
5. **Return Move:** The best move is returned in the required string format `'row,col,dir'`.

This approach ensures that the function always returns a legal move while attempting to make strategic decisions based on the current board state.
