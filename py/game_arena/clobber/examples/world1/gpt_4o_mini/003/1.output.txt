The game of Clobber involves strategically capturing the opponent's pieces by moving one's own pieces into positions adjacent to them. Given the board representation and the constraints outlined, the goal of the move-selection policy is to generate a valid move from the current game state.

Here's a structured approach to building the policy function:

1. **Input Format**: We receive two 5x6 boards, `you` and `opponent`, where:
   - `you` indicates your pieces (1 represents a piece in that cell).
   - `opponent` indicates opponent pieces (1 represents an opponent's piece in that cell).

2. **Finding Moves**: We need to check each piece of yours to see if it has any adjacent opponent pieces. We will consider the four possible directions for movement:
   - Up (U): (row-1, col)
   - Down (D): (row+1, col)
   - Left (L): (row, col-1)
   - Right (R): (row, col+1)

3. **Valid Moves**:
   - Ensure the starting cell contains your piece.
   - The destination cell should be within bounds and occupied by an opponent piece (1).

4. **Return Format**: The function should return a string in the format "row,col,dir".

5. **Random Selection**: In case there are multiple valid moves, we can either select one randomly or in a fixed order. For simplicity, I’ll just return the first available move that meets the conditions.

Here’s the implementation of the `policy` function based on this strategy:

```python
def policy(you: list[int], opponent: list[int]) -> int:
    for row in range(5):
        for col in range(6):
            if you[row][col] == 1:  # Check if it's your piece
                # Check Up
                if row > 0 and opponent[row - 1][col] == 1:
                    return f"{row},{col},U"
                # Check Down
                if row < 4 and opponent[row + 1][col] == 1:
                    return f"{row},{col},D"
                # Check Left
                if col > 0 and opponent[row][col - 1] == 1:
                    return f"{row},{col},L"
                # Check Right
                if col < 5 and opponent[row][col + 1] == 1:
                    return f"{row},{col},R"
    
    # The function should never reach this point due
    # to the constraint that always allows for at least one legal move.
    raise RuntimeError("Should have found a move, but did not.")

```

### Explanation of the Code:
- We loop through each cell on the board and check if it contains our piece (i.e., you[row][col] == 1).
- For each piece, we check the adjacent cells in the four directions to see if any contain an opponent piece.
- Upon finding an opponent piece, we generate the corresponding move string and return it immediately.
- The check conditions ensure that we do not go out of bounds.

This implementation adheres to all the rules and conditions set forth for climbing in this competitive gaming environment.
