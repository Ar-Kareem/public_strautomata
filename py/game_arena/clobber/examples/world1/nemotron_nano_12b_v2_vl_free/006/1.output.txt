To solve this problem, we need to create a policy for a Clobber game that plays on a 5x6 grid. The goal is to select the optimal move that maximizes our chances of winning while minimizing the opponent's future options. The policy should return a legal move string in the format 'row,col,dir'.

### Approach
The policy function will generate all possible legal moves for the current player and then evaluate each move based on its impact on the opponent's mobility. The key steps are:

1. **Generate Possible Moves**: For each piece owned by the current player, check all four possible directions (up, right, down, left) to see if there is an adjacent piece of the opponent. If so, this move is added to the list of possible moves.

2. **Evaluate Moves**: For each generated move, simulate the resulting board state and compute how many moves the opponent would have. The move that results in the opponent having the fewest possible moves is selected. If any move results in the opponent having no moves left, this move is chosen immediately as it guarantees a win.

3. **Return the Best Move**: After evaluating all possible moves, the best move is selected based on the minimum opponent's mobility. If multiple moves have the same minimal count, the first one encountered is chosen.

### Solution Code
```python
def policy(you, opponent):
    direction_map = {'U': (-1, 0), 'R': (0, 1), 'D': (1, 0), 'L': (0, -1)}
    
    # Generate all possible moves
    possible_moves = []
    for row in range(5):
        for col in range(6):
            pos = row * 6 + col
            if you[pos]:
                for dir_name in direction_map:
                    dr, dc = direction_map[dir_name]
                    new_row = row + dr
                    new_col = col + dc
                    if 0 <= new_row < 5 and 0 <= new_col < 6:
                        if opponent[new_row * 6 + new_col]:
                            possible_moves.append((row, col, dir_name))
    
    if not possible_moves:
        # According to problem constraints, this should not occur
        return '0,0,U'  # Fallback move (should never be used)
    
    best_move = possible_moves[0]
    min_opponent_moves = float('inf')
    
    direction_values = [(-1, 0), (0, 1), (1, 0), (0, -1)]
    
    for move in possible_moves:
        r, c, d = move
        dr, dc = direction_values[{'U':0, 'R':1, 'D':2, 'L':3}[d]]
        new_row = r + dr
        new_col = c + dc
        
        # Apply move to new_you and new_opponent
        new_you = list(you)
        new_opponent = list(opponent)
        current_pos = r * 6 + c
        new_you[current_pos] = 0
        new_opponent[new_row * 6 + new_col] = 0
        new_you[new_row * 6 + new_col] = 1
        
        # Count opponent's possible moves after this move
        count = 0
        for opp_row in range(5):
            opp_row_pos = opp_row * 6
            for opp_col in range(6):
                if new_opponent[opp_row_pos + opp_col]:
                    for odr, odc in direction_values:
                        o_new_r = opp_row + odr
                        o_new_c = opp_col + odc
                        if 0 <= o_new_r < 5 and 0 <= o_new_c < 6:
                            target_pos = o_new_r * 6 + o_new_c
                            if new_you[target_pos]:
                                count += 1
                                break  # No need to check other directions for this cell
        
        # Check if this move results in opponent having no moves
        if count == 0:
            return f"{r},{c},{d}"
        
        if count < min_opponent_moves:
            min_opponent_moves = count
            best_move = move
    
    r, c, d = best_move
    return f"{r},{c},{d}"
```

### Explanation
The code generates all possible moves for the current player by iterating through each of their pieces and checking all four potential directions. For each valid move, the code simulates the resulting board state and calculates how many moves the opponent would have. The move that results in the fewest opponent moves is selected. If a move leads to the opponent having no moves, it is chosen immediately to secure a win. This greedy approach ensures that the chosen move optimizes the current player's position by limiting the opponent's options as much as possible.
