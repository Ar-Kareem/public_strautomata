

The strategy is to prioritize capturingopponent pieces in a straightforward manner by iterating through all possible moves in a fixed order (row-major order) and returning the first valid move encountered. This ensures a legal move is always returned efficiently without complex computations, adhering to the 1-second execution constraint. The approach checks each of your pieces in sequence, then checks adjacent cells in the order up, right, down, left for opponent pieces to capture.

def policy(you, opponent):
    for row in range(5):
        for col in range(6):
            if you[row][col] == 1:
                # Check up
                if row - 1 >= 0 and opponent[row-1][col] == 1:
                    return f"{row},{col},U"
                # Check right
                if col + 1 < 6 and opponent[row][col+1] == 1:
                    return f"{row},{col},R"
                # Check down
                if row + 1 < 5 and opponent[row+1][col] == 1:
                    return f"{row},{col},D"
                # Check left
                if col - 1 >= 0 and opponent[row][col-1] == 1:
                    return f"{row},{col},L"
    # The problem guarantees at least one move exists, so this line is theoretically unreachable
    return "0,0,U"  # Fallback (though not needed per problem constraints)
