The strategy for this Clobber implementation is to use a Greedy Minimax approach with basic move ordering. Since Clobber is a game of attrition where the last player to move wins, the primary goal is to maximize the number of moves available to us while minimizing those available to the opponent.

1. **Move Generation**: We scan the 5x6 board for all valid moves where one of our pieces can land on an adjacent opponent piece.
2. **Evaluation**: We use a simple heuristic: `(Number of My Available Moves) - (Number of Opponent's Available Moves)`.
3. **Search**: We perform a shallow depth-first search (minimax) to look one or two steps ahead, ensuring we don't accidentally move into a state where the opponent has significantly more options than we do. 
4. **Fallback**: If the search time or complexity is a concern, we default to the move that immediately leaves the opponent with the fewest response options.

import numpy as np

def policy(you: list[list[int]], opponent: list[list[int]]) -> str:
    """
    Implements a move-selection policy for the game Clobber on a 5x6 board.
    Returns move in format 'row,col,dir'.
    """
    rows = 5
    cols = 6
    
    # Convert inputs to numpy for easier indexing if they aren't already
    you_arr = np.array(you)
    opp_arr = np.array(opponent)
    
    def get_legal_moves(player_pieces, enemy_pieces):
        moves = []
        directions = [(-1, 0, 'U'), (1, 0, 'D'), (0, -1, 'L'), (0, 1, 'R')]
        for r in range(rows):
            for c in range(cols):
                if player_pieces[r, c] == 1:
                    for dr, dc, dchar in directions:
                        nr, nc = r + dr, c + dc
                        if 0 <= nr < rows and 0 <= nc < cols:
                            if enemy_pieces[nr, nc] == 1:
                                moves.append((r, c, dchar, nr, nc))
        return moves

    legal_moves = get_legal_moves(you_arr, opp_arr)
    
    # Heuristic: Find move that minimizes the number of response moves the opponent has
    best_move = None
    min_opp_responses = float('inf')
    
    for r, c, dchar, nr, nc in legal_moves:
        # Simulate move
        # We move from (r, c) to (nr, nc)
        new_you = np.copy(you_arr)
        new_opp = np.copy(opp_arr)
        
        new_you[r, c] = 0
        new_you[nr, nc] = 1
        new_opp[nr, nc] = 0
        
        # Calculate opponent responses (opponent is now 'you' in this sub-calc)
        opp_responses = get_legal_moves(new_opp, new_you)
        num_responses = len(opp_responses)
        
        # We want to minimize the opponent's options
        if num_responses < min_opp_responses:
            min_opp_responses = num_responses
            best_move = f"{r},{c},{dchar}"
            
    # Default to first legal move if something goes wrong, though logic above should cover it
    if best_move is None and legal_moves:
        r, c, dchar, nr, nc = legal_moves[0]
        return f"{r},{c},{dchar}"
        
    return best_move
