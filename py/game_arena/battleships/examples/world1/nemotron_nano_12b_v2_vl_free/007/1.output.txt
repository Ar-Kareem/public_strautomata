To solve this problem, we need to develop a smart policy for a Battleship-like game on a 10x10 grid. The objective is to sink all the opponent's ships before they do the same by making strategic moves based on the current state of the board. The policy function must return a legal move (a cell that hasn't been previously shot) and must be efficient enough to run within the given constraints.

### Approach
The strategy for the policy function is divided into two main parts:

1. **Expanding from Hits**: If there are any hits on the board, the function should try to expand in the direction of the hit to find adjacent ship cells. This helps in continuing to sink the ship by checking neighboring cells.
2. **Systematic Search**: If there are no hits, the function uses a row-wise systematic search pattern to cover the grid efficiently. This ensures that the entire grid is covered in a structured manner when no information about ship positions is available.

### Solution Code
```python
def policy(board):
    def generate_row_wise_order():
        order = []
        for r in range(10):
            if r % 2 == 0:
                for c in range(10):
                    order.append((r, c))
            else:
                for c in range(9, -1, -1):
                    order.append((r, c))
        return order

    # Check for hits and expand
    hits = [(r, c) for r in range(10) for c in range(10) if board[r][c] == 1]
    if hits:
        candidates = []
        for (r, c) in hits:
            for dr, dc in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
                nr, nc = r + dr, c + dc
                if 0 <= nr < 10 and 0 <= nc < 10 and board[nr][nc] == 0:
                    candidates.append((nr, nc, r, c))
        
        sorted_candidates = []
        for (nr, nc, orig_r, orig_c) in candidates:
            dx = nc - orig_c
            dy = nr - orig_r
            if dx == 1 and dy == 0:
                priority = 0  # right
            elif dx == -1 and dy == 0:
                priority = 1  # left
            elif dy == 1 and dx == 0:
                priority = 2  # down
            elif dy == -1 and dx == 0:
                priority = 3  # up
            else:
                priority = 4  # not adjacent
            sorted_candidates.append((priority, (nr, nc)))
        
        if sorted_candidates:
            sorted_candidates.sort(key=lambda x: x[0])
            for candidate in sorted_candidates:
                if board[candidate[0]][candidate[1]] == 0:
                    return candidate[0], candidate[1]
    
    # If no hits, use row-wise search
    order = generate_row_wise_order()
    for (r, c) in order:
        if board[r][c] == 0:
            return r, c
    
    return (0, 0)
```

### Explanation
1. **Expanding from Hits**: When there are hits on the board, the code identifies all adjacent cells to these hits. Each candidate cell is prioritized based on its direction from the original hit (right, left, down, up). This prioritization helps in continuing the search in a logical direction to find the rest of the ship.
2. **Row-wise Systematic Search**: If there are no hits, the function generates a row-wise order starting from the top-left corner, alternating between left-right and right-left directions for each row. This ensures that the entire grid is covered systematically, increasing the chances of finding a ship segment.
3. **Fallback**: If all cells have been checked, the function returns a default move (0,0), which is guaranteed to be legal as per the problem constraints.

This approach efficiently balances between expanding from known hits to sink ships quickly and systematically covering the grid to find new ships when no information is available.
