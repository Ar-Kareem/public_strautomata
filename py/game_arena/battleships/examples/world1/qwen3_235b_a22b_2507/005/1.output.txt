To design an effective targeting policy for Battleship, we need to balance two main strategies: 

1. **Exploration (Hunt Mode):** Systematically search the board to find ships. A common and efficient method is to use a "parity" or "checkerboard" pattern to cover the board quickly. Since all ships are at least length 2, we can skip every other cell in a checkerboard pattern to reduce the total number of required search shots.

2. **Exploitation (Target Mode):** Once a ship is hit, we should switch to targeting adjacent cells (up/down/left/right) to try to sink it. We should prioritize cells that extend the hit in consistent directions and avoid dead ends.

We can enhance this with:
- Avoiding duplicate shots — only pick cells with value 0.
- Tracking sequences of hits to infer ship orientation (horizontal/vertical).
- Using probability-based targeting: prefer cells that are more likely to be occupied by a ship based on remaining ship lengths and board layout.

**Algorithm:**
1. Maintain a list of "hit" cells that haven't been fully explored (i.e., adjacent cells not yet shot).
2. If there are such hit cells with unshot neighbors, prioritize shooting one of those neighbors.
3. If no such hit exists, revert to a sparse search grid — use a 2x2 stride pattern (every other row and column) with parity to reduce redundancy.

We'll use a greedy "most likely" neighbor approach, checking only up/down/left/right of any hit.

This hybrid hunt-and-target strategy is simple, fast, and effective.
