The strategy for this policy is as follows: When the board shows active hits (cells with value 1), the policy enters "hunting mode" to target likely ship extensions. For each hit cell, it identifies unexplored adjacent cells in the four cardinal directions (up, down, left, right), prioritizing straight lines to potentially sink ships (e.g., extending vertically or horizontally until hitting a miss or board edge). Among these potential targets, it selects the first valid one (row 0-9, col 0-9, not previously shot). This approach aims to maximize consecutive hits and sink ships quickly. If no hunting targets exist (all hits isolated or sunk), it switches to "search mode" using a checkerboard pattern for efficient board coverage—targeting the first unexplored cell where (row + col) % 2 == 0, then falling back to any unexplored cell if needed. This balances exploration and exploitation, increasing hit rates over random shooting while staying stateless and avoiding invalid moves.
