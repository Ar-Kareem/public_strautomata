To solve this problem, we need to implement a targeting policy for a Battleship-like game on a 10x10 grid. The goal is to sink all opponent ships before they sink ours. The opponent places ships of lengths 5, 4, 3, 3, and 2 on the grid without overlaps. The player sees the results of their own shots: 0 (unknown), -1 (miss), or 1 (hit). The policy must return the coordinates of the next cell to fire at, which should not have been fired at before.

### Approach
1. **Hunt Phase**: When there are no known hits, we use a predefined pattern to efficiently search for ships. The pattern ensures that we cover the grid in a way that maximizes the chances of hitting ships while minimizing wasted shots.
2. **Target Phase**: When there are known hits, we identify contiguous segments of hits (ships) and attempt to extend them in both possible directions (left/right for horizontal ships, up/down for vertical ships). The candidate targets are prioritized based on the number of hits in the ship (more hits mean higher priority).
3. **BFS for Ship Grouping**: We use a breadth-first search (BFS) to group contiguous hits into ships. The BFS ensures that only aligned hits (same row or column) are grouped together, which is essential because ships are placed straight.
4. **Candidate Generation**: For each identified ship (with fewer than 5 hits), we generate potential target cells at both ends of the ship's alignment. If the ship has only one hit, we consider all four adjacent cells.
5. **Hunt Mode Fallback**: If there are no candidate targets (either no hits or no viable extensions), we fall back to the predefined hunt pattern to find new hits.

### Solution Code
```python
# Precomputed hunt pattern and total order for firing
HUNT_PATTERN = [
    (0,0), (0,3), (0,6), (0,9),
    (1,1), (1,4), (1,7),
    (2,2), (2,5), (2,8),
    (3,0), (3,3), (3,6), (3,9),
    (4,1), (4,4), (4,7),
    (5,2), (5,5), (5,8),
    (6,0), (6,3), (6,6), (6,9),
    (7,1), (7,4), (7,7),
    (8,2), (8,5), (8,8),
    (9,0), (9,3), (9,6), (9,9)
]

ALL_CELLS = [(r, c) for r in range(10) for c in range(10)]
HUNT_PATTERN_SET = set(HUNT_PATTERN)
TOTAL_ORDER = HUNT_PATTERN + [cell for cell in ALL_CELLS if cell not in HUNT_PATTERN_SET]

def policy(board: list[list[int]]) -> tuple[int, int]:
    visited = [[False]*10 for _ in range(10)]
    ships = []
    
    for r in range(10):
        for c in range(10):
            if board[r][c] == 1 and not visited[r][c]:
                queue = [(r, c)]
                visited[r][c] = True
                ship = [(r, c)]
                orientation = None
                
                while queue:
                    x, y = queue.pop()
                    for dx, dy in [(1,0), (-1,0), (0,1), (0,-1)]:
                        nx, ny = x+dx, y+dy
                        if 0 <= nx < 10 and 0 <= ny < 10 and not visited[nx][ny] and board[nx][ny] == 1:
                            if orientation is None:
                                if nx == x or ny == y:
                                    orientation = 'horizontal' if nx == x else 'vertical'
                                    visited[nx][ny] = True
                                    ship.append((nx, ny))
                                    queue.append((nx, ny))
                            else:
                                if orientation == 'horizontal' and nx == x:
                                    visited[nx][ny] = True
                                    ship.append((nx, ny))
                                    queue.append((nx, ny))
                                elif orientation == 'vertical' and ny == y:
                                    visited[nx][ny] = True
                                    ship.append((nx, ny))
                                    queue.append((nx, ny))
                
                ships.append(ship)
    
    candidate_list = []
    for ship in ships:
        if len(ship) == 5:
            continue
        if len(ship) == 1:
            r0, c0 = ship[0]
            for dr, dc in [(-1,0), (1,0), (0,-1), (0,1)]:
                nr, nc = r0+dr, c0+dc
                if 0 <= nr < 10 and 0 <= nc < 10 and board[nr][nc] == 0:
                    candidate_list.append((len(ship), (nr, nc)))
        else:
            rows = [cell[0] for cell in ship]
            cols = [cell[1] for cell in ship]
            if all(x == rows[0] for x in rows):
                r = rows[0]
                min_c = min(cols)
                max_c = max(cols)
                if min_c - 1 >= 0 and board[r][min_c-1] == 0:
                    candidate_list.append((len(ship), (r, min_c-1)))
                if max_c + 1 < 10 and board[r][max_c+1] == 0:
                    candidate_list.append((len(ship), (r, max_c+1)))
            elif all(x == cols[0] for x in cols):
                c = cols[0]
                min_r = min(rows)
                max_r = max(rows)
                if min_r - 1 >= 0 and board[min_r-1][c] == 0:
                    candidate_list.append((len(ship), (min_r-1, c)))
                if max_r + 1 < 10 and board[max_r+1][c] == 0:
                    candidate_list.append((len(ship), (max_r+1, c)))
    
    if candidate_list:
        candidate_list.sort(key=lambda x: x[0], reverse=True)
        return candidate_list[0][1]
    
    fired_cells = set()
    for r in range(10):
        for c in range(10):
            if board[r][c] != 0:
                fired_cells.add((r, c))
    
    for cell in TOTAL_ORDER:
        if cell not in fired_cells:
            return cell
    
    return (0, 0)
```

### Explanation
1. **Hunt Pattern**: The `HUNT_PATTERN` is a predefined list of coordinates optimized to cover the grid efficiently, ensuring that we minimize the number of shots needed to find ships.
2. **BFS for Ship Grouping**: The BFS algorithm groups contiguous hits into ships. It ensures that only aligned hits (same row or column) are grouped, which is crucial since ships are placed straight.
3. **Candidate Target Generation**: For each identified ship (with fewer than 5 hits), the policy generates candidate targets at both ends of the ship's alignment. For a single hit, all four adjacent cells are considered.
4. **Priority Handling**: Candidate targets are sorted by the number of hits in their ship, giving priority to extending ships with more hits, which are closer to being sunk.
5. **Hunt Mode Fallback**: If no candidate targets are available, the policy falls back to the predefined hunt pattern to find new hits, ensuring efficient coverage of the remaining grid.
6. **Efficiency**: The approach efficiently balances between exploring the grid in the hunt phase and exploiting known hits in the target phase, minimizing wasted shots and accelerating the sinking of ships.
