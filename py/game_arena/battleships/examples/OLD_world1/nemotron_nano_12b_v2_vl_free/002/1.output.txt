To solve this problem, we need to develop an efficient strategy to target cells in a Battleship-like game on a 10x10 grid. The goal is to sink all the opponent's ships before they sink ours. The function `policy` will determine the next cell to shoot based on the current state of the board, which includes hits (1), misses (-1), and unknown cells (0).

### Approach
1. **Exploration Phase**: If there are no hits (i.e., no ships have been hit yet), we explore the grid in a row-major order. This systematic approach ensures that we cover all cells methodically, increasing the chances of hitting a ship.
2. **Targeting Phase**: Once a hit is detected, we focus on the cells adjacent to the hit to determine the direction of the ship. The ship can be either horizontal or vertical, so we check the left, right, up, and down cells of each hit. These cells are considered as potential next targets.
3. **Duplicate Handling**: To avoid redundant targeting, we track the candidates using a set and convert it to a list while maintaining order. The candidates are then sorted in row-major order to ensure a deterministic and efficient targeting strategy.

### Solution Code
```python
def policy(board: list[list[int]]) -> tuple[int, int]:
    ROWS, COLS = 10, 10

    # Collect all current hits
    hits = []
    for x in range(ROWS):
        for y in range(COLS):
            if board[x][y] == 1:
                hits.append((x, y))

    # Exploration phase: if no hits, proceed with row-major order
    if not hits:
        for row in range(ROWS):
            for col in range(COLS):
                if board[row][col] == 0:
                    return (row, col)
        return (0, 0)  # Should never reach here as there must be unshot cells

    # Collect adjacent unshot cells to current hits
    candidates = []
    for x, y in hits:
        # Check all four possible neighboring cells
        for dx, dy in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
            nx, ny = x + dx, y + dy
            if 0 <= nx < ROWS and 0 <= ny < COLS and board[nx][ny] == 0:
                candidates.append((nx, ny))

    # Remove duplicates and maintain order
    seen = set()
    unique_candidates = []
    for cell in candidates:
        if cell not in seen:
            seen[cell] = True
            unique_candidates.append(cell)

    # Return the first candidate in row-major order
    if unique_candidates:
        unique_candidates.sort()
        return unique_candidates[0]

    # Fallback exploration if no candidates found (unlikely)
    for x in range(ROWS):
        for y in range(COLS):
            if board[x][y] == 0:
                return (x, y)
    return (0, 0)
```

### Explanation
1. **Exploration Phase**: When there are no hits, the code systematically explores the grid in row-major order (left to right, top to bottom). This ensures that each cell is checked in a predictable and ordered manner, maximizing the likelihood of hitting a ship early on.
2. **Targeting Phase**: Once a hit is detected, the code checks all four adjacent cells (left, right, up, down) for potential ship segments. These cells are collected and processed without duplicates. The candidates are then sorted in row-major order to ensure a deterministic and efficient selection of the next target.
3. **Efficiency**: The code efficiently handles both exploration and targeting phases. By focusing on cells adjacent to known hits, it maximizes the probability of hitting the remaining parts of a ship, which is a common strategy in Battleship to sink ships quickly.

This approach balances exploration and targeted shooting around known hits, ensuring that the AI efficiently narrows down the locations of the opponent's ships.
