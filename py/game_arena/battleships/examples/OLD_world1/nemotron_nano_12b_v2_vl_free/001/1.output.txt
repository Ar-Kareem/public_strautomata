To solve this problem, we need to develop a scoring algorithm for a Battleship-like game where the goal is to efficiently sink all ships placed by the opponent on a 10x10 grid. The algorithm should dynamically decide the next best cell to target based on previous hits and misses.

### Approach
The approach to solving this problem involves two main strategies:
1. **Initial Random Exploration**: When no ships have been hit yet, the algorithm starts by targeting the center of the grid (5,5). This is a strategic starting point as it maximizes the chance of hitting a ship.
2. **Adaptive Targeting**: Once a ship is hit, the algorithm prioritizes targeting adjacent cells to the existing hits. This is based on the logic that ships are placed in straight lines, so adjacent cells to a hit are more likely to be part of the same ship.

### Solution Code
```python
import random
from typing import List

def policy(board: List[List[int]]) -> tuple[int, int]:
    # Generate all unknown cells (cells not yet fired at)
    candidates = []
    for i in range(10):
        for j in range(10):
            if board[i][j] == 0:
                candidates.append((i, j))
    
    # Check if there are any hits on the board to adjust strategy
    has_hits = any(1 in row for row in board)
    
    if not has_hits:
        # If no hits, start from the center of the grid (5,5) if available
        return (5, 5) if (5, 5) in candidates else candidates[0]
    
    # Sort candidates in row-major order to ensure deterministic behavior
    candidates_sorted = sorted(candidates)
    best_score = -1
    best_cell = None
    
    # Evaluate each candidate based on adjacent hits
    for cell in candidates_sorted:
        i, j = cell
        score = 0
        # Check all four neighboring cells
        for dx, dy in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
            ni, nj = i + dx, j + dy
            if 0 <= ni < 10 and 0 <= nj < 10 and board[ni][nj] == 1:
                score += 1
        # Update the best cell if this cell has a higher score
        if score > best_score:
            best_score = score
            best_cell = cell
    
    return best_cell
```

### Explanation
1. **Candidate Collection**: The algorithm first collects all cells that have not been targeted yet (denoted by `0` in the input board).
2. **Initial Exploration**: If there are no hits yet (`has_hits` is `False`), the algorithm starts by targeting the center cell (5,5). If the center cell has already been fired, it defaults to the first available cell in row-major order.
3. **Adaptive Targeting**: When hits are present, the algorithm evaluates each unknown cell by scoring it based on the number of adjacent hits. Cells adjacent to more hits are considered more likely to be part of a ship.
4. **Deterministic Selection**: Candidates are sorted in row-major order to ensure consistent selection when multiple cells have the same score. The cell with the highest score (adjacent hits) is chosen to maximize the chance of hitting a ship.

This approach efficiently narrows down potential ship locations by leveraging previous hits, ensuring optimal targeting while adhering to the constraints of the game.
