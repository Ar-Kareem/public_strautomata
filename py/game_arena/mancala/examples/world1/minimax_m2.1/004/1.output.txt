To solve this problem, we need to develop a smart policy for the Kalah game that can make optimal moves to maximize the player's store seeds while minimizing the opponent's chances. The policy must be efficient enough to run within the constraints and always return a legal move.

### Approach
1. **Problem Analysis**: The Kalah game involves two players, each with 6 houses and a store. Players take turns sowing seeds from their houses into subsequent houses, skipping the opponent's store. The game ends when one player has no seeds left in their houses, and the opponent collects all remaining seeds in their houses. The player with the most seeds in their store wins.

2. **Key Insights**:
   - **Sowing Mechanics**: Seeds are sown one by one into subsequent houses, skipping the opponent's store. If the last seed lands in the player's store, they get an extra turn.
   - **Capture Rule**: If the last seed lands in an empty house of the player and the opposite house of the opponent has seeds, all seeds from both houses are captured into the player's store.
   - **Terminal State**: The game ends when one player has no seeds in their houses. The opponent collects all remaining seeds from the other player's houses into their store.

3. **Algorithm Selection**: 
   - **Minimax with Alpha-Beta Pruning**: This algorithm efficiently explores possible moves by simulating the game tree up to a fixed depth (3 moves in this case). It uses alpha-beta pruning to cut off suboptimal branches, optimizing performance.
   - **Heuristic Evaluation**: For non-terminal states, a heuristic function evaluates the game state based on the difference in store seeds and the number of seeds in houses, weighted to prioritize store seeds.

4. **Complexity Analysis**: 
   - **Time Complexity**: The algorithm explores up to \(6^3 = 216\) nodes per move, which is feasible within the given constraints. The alpha-beta pruning further reduces the number of explored nodes.
   - **Space Complexity**: The space usage is linear with respect to the depth of the recursion, which is manageable.

### Solution Code
```python
import math

INF = 10**9

def simulate_move(you, opponent, move):
    board = you[:6] + [you[6]] + opponent[:6]
    seeds = you[move]
    board[move] = 0
    i = move + 1
    last_index = None
    while seeds > 0:
        last_index = i
        if i in [0, 1, 2, 3, 4, 5] and seeds == 1 and board[i] == 0 and board[12 - i] > 0:
            captured = 1 + board[12 - i]
            board[12 - i] = 0
            board[6] += captured
            seeds = 0
        else:
            board[i] += 1
            seeds -= 1
        i = (i + 1) % 13
    new_you = board[:7]
    new_opponent = board[7:13] + [opponent[6]]
    extra_move = (last_index == 6)
    return new_you, new_opponent, extra_move

def terminal_score(you, opponent, is_maximizing):
    if is_maximizing:
        if all(x == 0 for x in you[:6]):
            return you[6]
        else:
            return you[6] + sum(opponent[:6])
    else:
        if all(x == 0 for x in you[:6]):
            return opponent[6] + sum(you[:6])
        else:
            return opponent[6]

def heuristic_score(you, opponent, is_maximizing):
    if is_maximizing:
        our_store = you[6]
        opponent_store = opponent[6]
        our_houses = sum(you[:6])
        opponent_houses = sum(opponent[:6])
    else:
        our_store = opponent[6]
        opponent_store = you[6]
        our_houses = sum(opponent[:6])
        opponent_houses = sum(you[:6])
    return (our_store - opponent_store) * 2 + (our_houses - opponent_houses)

def minimax(you, opponent, depth, alpha, beta, is_maximizing):
    if all(x == 0 for x in you[:6]) or all(x == 0 for x in opponent[:6]):
        return terminal_score(you, opponent, is_maximizing)
    if depth == 0:
        return heuristic_score(you, opponent, is_maximizing)
    
    moves = [i for i in range(6) if you[i] > 0]
    if not moves:
        return -INF if is_maximizing else INF
    
    if is_maximizing:
        best_value = -INF
        for move in moves:
            new_you, new_opponent, extra_move = simulate_move(you, opponent, move)
            next_flag = True if extra_move else False
            value = minimax(new_you, new_opponent, depth-1, alpha, beta, next_flag)
            best_value = max(best_value, value)
            alpha = max(alpha, best_value)
            if beta <= alpha:
                break
        return best_value
    else:
        best_value = INF
        for move in moves:
            new_you, new_opponent, extra_move = simulate_move(you, opponent, move)
            next_flag = True if extra_move else False
            value = minimax(new_you, new_opponent, depth-1, alpha, beta, next_flag)
            best_value = min(best_value, value)
            beta = min(beta, best_value)
            if beta <= alpha:
                break
        return best_value

def policy(you, opponent):
    depth = 3
    moves = [i for i in range(6) if you[i] > 0]
    if not moves:
        return 0
    if len(moves) == 1:
        return moves[0]
    best_move = moves[0]
    best_value = -INF
    for move in moves:
        new_you, new_opponent, extra_move = simulate_move(you, opponent, move)
        next_flag = True if extra_move else False
        value = minimax(new_you, new_opponent, depth-1, -INF, INF, next_flag)
        if value > best_value:
            best_value = value
            best_move = move
    return best_move
```

### Explanation
1. **simulate_move**: This function simulates the sowing of seeds for a given move, handling captures and extra turns. It returns the new state of the game and whether an extra turn was granted.
2. **terminal_score**: Computes the score for terminal game states (when one player has no seeds left in their houses) based on the store seeds and captured house seeds.
3. **heuristic_score**: Evaluates non-terminal states by considering the difference in store seeds and house seeds, prioritizing store seeds.
4. **minimax**: Uses the minimax algorithm with alpha-beta pruning to explore possible moves up to a fixed depth. It alternates between maximizing (current player) and minimizing (opponent) roles, efficiently pruning suboptimal branches.
5. **policy**: The main policy function that selects the best move by evaluating all possible moves using the minimax algorithm. It ensures the move is legal and optimally balances immediate gains with future prospects.
