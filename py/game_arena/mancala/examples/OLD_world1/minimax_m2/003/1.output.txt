To solve this problem, we need to implement a move-selection policy for the game Kalah. The policy should choose a legal move (a non-empty house) that optimizes the player's position based on the current game state. The solution involves simulating each possible move, evaluating the resulting state, and selecting the move that leads to the best outcome.

### Approach
1. **Problem Analysis**: The game Kalah involves two players, each with 6 houses and a store. The player to move selects a non-empty house, distributes the seeds clockwise, skipping the opponent's store. Special rules include gaining an extra turn if the last seed lands in the player's store and capturing seeds if the last seed lands in an empty house with non-empty opposite house in the opponent's side.
2. **Intuition**: For each possible move, simulate the game state after the move. Evaluate the state by considering the store difference, seeds on the board, and any extra moves gained. The move leading to the highest score is selected.
3. **Algorithm Selection**: The algorithm involves:
   - **Simulation**: For each non-empty house, simulate the distribution of seeds and apply capture rules if applicable.
   - **Evaluation**: Calculate a score based on the store difference, seeds on the board, and bonuses for extra moves.
   - **Selection**: Choose the move with the highest score.
4. **Complexity Analysis**: Each move simulation involves distributing up to the total seeds (typically 72 for a standard game). Evaluating each move involves O(1) operations per seed distribution, leading to an overall complexity of O(n) per move, where n is the total seeds. With 6 possible moves, the total complexity is O(6n), which is efficient for the given constraints.

```python
def simulate_move(you, opponent, i):
    new_you = you[:]
    new_opponent = opponent[:]
    seeds = new_you[i]
    if seeds == 0:
        return None, None, False
    new_you[i] = 0
    current_player = 'you'
    current_index = i
    last_house = None
    
    while seeds > 0:
        current_index += 1
        if current_player == 'you':
            if current_index == 7:
                current_player = 'opponent'
                current_index = 0
            else:
                new_you[current_index] += 1
                seeds -= 1
                last_house = ('you', current_index)
        else:
            if current_index == 6:
                current_player = 'you'
                current_index = 0
            else:
                new_opponent[current_index] += 1
                seeds -= 1
                last_house = ('opponent', current_index)
                
    if last_house[0] == 'you' and 0 <= last_house[1] <= 5:
        if new_you[last_house[1]] == 1:
            opposite_index = 5 - last_house[1]
            if new_opponent[opposite_index] > 0:
                captured_seeds = new_you[last_house[1]] + new_opponent[opposite_index]
                new_you[6] += captured_seeds
                new_you[last_house[1]] = 0
                new_opponent[opposite_index] = 0
                
    extra_move = (last_house[0] == 'you' and last_house[1] == 6)
    return new_you, new_opponent, extra_move

def policy(you, opponent):
    best_move = -1
    best_score = -10**9
    
    for i in range(6):
        if you[i] == 0:
            continue
            
        new_you, new_opponent, extra_move = simulate_move(you, opponent, i)
        
        if sum(new_opponent[0:6]) == 0:
            our_remaining = sum(new_you[0:6])
            new_you[6] += our_remaining
            new_you[0:6] = [0] * 6
            score = (new_you[6] - new_opponent[6]) * 100
        elif sum(new_you[0:6]) == 0:
            opp_remaining = sum(new_opponent[0:6])
            new_opponent[6] += opp_remaining
            new_opponent[0:6] = [0] * 6
            score = (new_you[6] - new_opponent[6]) * 100
        else:
            score = (new_you[6] - new_opponent[6]) * 100 + (sum(new_you[0:6]) - sum(new_opponent[0:6]))
            
        if extra_move:
            score += 50
            
        if score > best_score:
            best_score = score
            best_move = i
            
    return best_move
```

### Explanation
1. **Simulation (simulate_move)**:
   - **Initialization**: Copies the current state to avoid modifying the original lists.
   - **Seed Distribution**: Simulates distributing seeds clockwise, skipping the opponent's store. Tracks the last house where a seed was dropped.
   - **Capture Check**: If the last seed lands in an empty house with a non-empty opposite house in the opponent's side, captures the seeds and moves them to the player's store.
   - **Extra Move Detection**: Checks if the last seed lands in the player's store, granting an extra move.

2. **Policy Evaluation (policy)**:
   - **Move Selection**: Iterates over each non-empty house, simulating the move and evaluating the resulting state.
   - **Endgame Handling**: If a move leaves the opponent with no seeds, all remaining seeds are moved to the player's store. Similarly, if the player has no seeds, the opponent collects their seeds.
   - **Score Calculation**: Computes a score based on the store difference, seeds on the board, and bonuses for extra moves. The move with the highest score is selected.

This approach efficiently evaluates each possible move, considering game-ending conditions and strategic advantages, to select the optimal move within the given constraints.
