To solve this problem, we need to implement a move-selection policy for the game Kalah. The policy must choose a legal move (i.e., selecting a non-empty house from the player's side) that maximizes the immediate gain, considering factors like captures and extra moves. The solution involves simulating each possible move, evaluating the resulting state, and selecting the move that provides the highest gain.

### Approach
1. **Problem Analysis**: The game state is represented by two lists, each containing 7 elements: 6 houses and 1 store. The player can only choose from non-empty houses. The distribution of seeds follows a specific cyclic order, skipping the opponent's store. Special effects include extra moves if the last seed lands in the player's store and captures if the last seed lands in an empty house with seeds in the opposite house.
2. **Key Insight**: The policy must evaluate each possible move by simulating its outcome. The gain from a move is calculated as the change in the player's store, plus a bonus for an extra move. Captures are naturally included in the store change since seeds are moved to the store during simulation.
3. **Algorithm Selection**: For each possible move (non-empty house), simulate the move to determine the new game state. Calculate the gain (store change plus extra move bonus) and select the move with the highest gain.
4. **Optimization**: The simulation of each move involves distributing seeds in the cyclic order (houses, store, opponent's houses, etc.), handling captures and extra moves. The simulation is efficient as it processes each seed once per move, making it feasible within the given time constraints.

### Solution Code
```python
def simulate_move(you, opponent, i):
    if you[i] == 0:
        return you, opponent, False
        
    seeds = you[i]
    new_you = you[:]
    new_opp = opponent[:]
    new_you[i] = 0
    
    pos = (i + 1) % 13
    for _ in range(seeds - 1):
        if pos == 0:
            new_you[0] += 1
        elif 1 <= pos <= 5:
            new_you[pos] += 1
        elif pos == 6:
            new_you[6] += 1
        else:
            new_opp[pos - 7] += 1
        pos = (pos + 1) % 13
        
    last_pos = pos
    if last_pos in [0, 1, 2, 3, 4, 5]:
        new_you[last_pos] += 1
        if new_you[last_pos] == 1:
            if new_opp[5 - last_pos] > 0:
                capture_seeds = new_you[last_pos] + new_opp[5 - last_pos]
                new_you[6] += capture_seeds
                new_you[last_pos] = 0
                new_opp[5 - last_pos] = 0
    elif last_pos == 6:
        new_you[6] += 1
    else:
        new_opp[last_pos - 7] += 1
        
    extra_move = (last_pos == 6)
    return new_you, new_opp, extra_move

def policy(you, opponent):
    best_move = None
    best_gain = -10**9
    
    for i in range(6):
        if you[i] == 0:
            continue
        new_you, new_opp, extra = simulate_move(you, opponent, i)
        gain = new_you[6] - you[6]
        if extra:
            gain += 10
        if gain > best_gain:
            best_gain = gain
            best_move = i
            
    return best_move
```

### Explanation
1. **simulate_move Function**: This function simulates a move by distributing seeds from a chosen house (index `i`) in the cyclic order (houses, store, opponent's houses, etc.). It handles captures by checking if the last seed lands in an empty house with seeds in the opposite house, moving these seeds to the player's store. It also checks if the last seed lands in the store to grant an extra move.
2. **policy Function**: This function evaluates all possible moves (non-empty houses) by simulating each move using `simulate_move`. It calculates the gain (store change plus bonus for extra moves) for each simulated move and selects the move with the highest gain. The selected move is returned as the policy decision.
3. **Efficiency**: The simulation processes each seed once per move, making the overall complexity manageable. The policy ensures optimal decision-making within the given constraints by leveraging efficient simulation and gain calculation.
