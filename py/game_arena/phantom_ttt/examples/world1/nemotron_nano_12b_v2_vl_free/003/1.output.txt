### Strategy Explanation

This policy for Phantom Tic Tac Toe prioritizes moves based on their potential to create immediate wins, forks, or strategic positions that maximize future opportunities. The strategy follows a structured approach:

1. **Immediate Win Check**: First, the policy checks if any legal move would complete a winning line (row, column, or diagonal) with its current marks. This is the highest priority because winning immediately ends the game.

2. **Fork Detection**: If no immediate win is found, the policy looks for moves that would create a fork—placing a mark that would potentially complete two different winning lines. This forces the opponent to block only one line, leaving the other open for a future win.

3. **Strategic Prioritization**: If neither an immediate win nor a fork is possible, the policy selects the most strategically valuable cell based on the following order:
   - **Center Cell (4)**: Offers the highest potential to influence multiple lines (row, column, and two diagonals).
   - **Corners (0, 2, 6, 8)**: These are advantageous as they are part of multiple lines and are less likely to be blocked by the opponent.
   - **Edges (1, 3, 5, 7)**: These are the least impactful in Tic Tac Toe strategy due to fewer overlapping lines.

The code ensures that the move returned is always from the `legal_moves` list and avoids already occupied cells by leveraging the provided `legal_moves` list.

### Python Code

```python
def priority(move):
    """Assigns priority to a cell based on its position in the Tic Tac Toe grid."""
    priority_map = [3, 2, 3, 2, 4, 2, 3, 2, 3]  # Priorities for cells 0-8
    return priority_map[move]

def policy(board: list[list[int]], legal_moves: list[int]) -> int:
    """
    Returns the next move for Phantom Tic Tac Toe as a tuple (row, col).
    
    - First checks for moves that immediately win the game.
    - Then checks for moves that create a fork (two possible winning lines).
    - Finally selects the highest priority move using the Tic Tac Toe strategy.
    """
    # Step 1: Check for immediate win
    for move in legal_moves:
        row, col = divmod(move, 3)
        # Check row
        row_sum = sum(board[row][c] for c in range(3))
        if row_sum + 1 == 3:
            return (row, col)
        # Check column
        col_sum = sum(board[r][col] for r in range(3))
        if col_sum + 1 == 3:
            return (row, col)
        # Check left diagonal
        if row == col:
            diag_sum = sum(board[i][i] for i in range(3))
            if diag_sum + 1 == 3:
                return (row, col)
        # Check right diagonal
        if row + col == 2:
            diag_sum = board[0][2] + board[1][1] + board[2][0]
            if diag_sum + 1 == 3:
                return (row, col)
    
    # Step 2: Check for fork (two lines that would be completed by this move)
    for move in legal_moves:
        row, col = divmod(move, 3)
        possible_lines = [
            [(row, c) for c in range(3)],  # row
            [(r, col) for r in range(3)],  # column
        ]
        # Add diagonals if applicable
        if row == col:
            possible_lines.append([(0,0), (1,1), (2,2)])
        if row + col == 2:
            possible_lines.append([(0,2), (1,1), (2,0)])
        
        fork_count = 0
        for line in possible_lines:
            line_sum = sum(board[r][c] for r, c in line)
            if line_sum == 2:
                fork_count += 1
        if fork_count >= 2:
            return (row, col)
    
    # Step 3: Prioritize moves based on strategic importance
    # Center first, then corners, then edges
    sorted_moves = sorted(legal_moves, key=lambda x: (-priority(x), x))
    chosen_move = sorted_moves[0]
    row, col = divmod(chosen_move, 3)
    return (row, col)
```

### Explanation of Key Components

- **Priority Mapping**:  
  The `priority` function assigns strategic values to each cell:
  - Center (4): Priority 4 (highest)
  - Corners (0, 2, 6, 8): Priority 3
  - Edges (1, 3, 5, 7): Priority 2
  - Other cells (faces same as above): Priority 2

- **Immediate Win Check**:  
  Iterates through all legal moves and checks if placing a mark in that cell would complete a winning line (row, column, or diagonal).

- **Fork Detection**:  
  For each legal move, the code checks how many lines (row, column, diagonals) would be completed by placing a mark in that cell. If two or more such lines exist, it's considered a fork move.

- **Prioritized Move Selection**:  
  If no immediate win or fork is found, legal moves are sorted by their strategic value and the highest-priority move is chosen.

This strategy ensures that the move is always legal and maximizes the chances of winning while minimizing the risk of being blocked by the opponent.
