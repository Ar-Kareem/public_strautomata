The strategy for this policy is to leverage probabilistic reasoning and pattern detection to maximize win probability in Phantom Tic Tac Toe. Since we only see our own confirmed marks (1) and cannot observe opponent marks (0 means unknown), we must assume that any cell marked 0 could be either empty or occupied by the opponent.

Our approach:
1. First, check if we can win in the next move by completing a row, column, or diagonal with two of our marks and one unknown (0) — this is likely a winning opportunity.
2. If no immediate win is possible, check for blocking opportunities: if an opponent is likely to win (two 0s in a line with one of our marks), we prioritize those 0 cells to block — because if they are empty, we claim them; if they are occupied by the opponent, we still make a legal move (and the game state updates accordingly).
3. If no immediate threat or win exists, use a preference order: prefer center (1,1), then corners (0,0), (0,2), (2,0), (2,2), then edges (0,1), (1,0), (1,2), (2,1). This maximizes future win potential.
4. Avoid any cell that is already confirmed as ours (1) — guaranteed illegal.
5. We simulate every possible configuration of unknowns (0s) as potential opponent marks, and compute expected utility, but due to time constraints, we approximate by checking lines: if a line has two of our marks and one 0, we play there. If a line has one of our marks and two 0s, we treat it as a potential threat and block one of the 0s.
6. We use a heuristic: if we have the only mark on the board, we play center. If we have two marks, we check for near-wins.

This policy is robust: it plays aggressively to win, defensively to block, and defaults to optimal opening moves for 3x3 tic-tac-toe under uncertainty.

import numpy as np

def policy(board: list[list[int]]) -> tuple[int, int]:
    # Convert to numpy array for easier manipulation
    board = np.array(board)
    
    # We can only place on 0s (unknown), not on 1s (already ours)
    possible_moves = [(i, j) for i in range(3) for j in range(3) if board[i][j] == 0]
    
    # If no possible moves (shouldn't happen in a valid game), return any legal move — but per spec, we must return legal, so fallback
    if not possible_moves:
        # This should never occur, but safety fallback
        return (0, 0)
    
    # Check for immediate wins: two 1s and one 0 in a line → play the 0
    for i in range(3):
        # Check rows
        if np.sum(board[i, :] == 1) == 2 and np.sum(board[i, :] == 0) == 1:
            j = np.where(board[i, :] == 0)[0][0]
            return (i, j)
        # Check columns
        if np.sum(board[:, i] == 1) == 2 and np.sum(board[:, i] == 0) == 1:
            j = np.where(board[:, i] == 0)[0][0]
            return (j, i)
    
    # Check diagonals
    diag1 = [board[0,0], board[1,1], board[2,2]]
    diag2 = [board[0,2], board[1,1], board[2,0]]
    if np.sum(diag1) == 2 and diag1.count(0) == 1:
        idx = diag1.index(0)
        return (idx, idx)
    if np.sum(diag2) == 2 and diag2.count(0) == 1:
        idx = diag2.index(0)
        if idx == 0:
            return (0, 2)
        elif idx == 1:
            return (1, 1)
        else:
            return (2, 0)
    
    # Check for blocking: opponent might have two in a row (two 0s) and we have one → block
    # We assume that if a line has two 0s and one 1, then the opponent is likely to win, so we block one of the 0s
    for i in range(3):
        if np.sum(board[i, :] == 0) == 2 and np.sum(board[i, :] == 1) == 1:
            j = np.where(board[i, :] == 0)[0][0]
            return (i, j)
        if np.sum(board[:, i] == 0) == 2 and np.sum(board[:, i] == 1) == 1:
            j = np.where(board[:, i] == 0)[0][0]
            return (j, i)
    
    if np.sum(diag1) == 1 and diag1.count(0) == 2:
        # Block diagonal 1
        idx = diag1.index(0)
        return (idx, idx)
    if np.sum(diag2) == 1 and diag2.count(0) == 2:
        # Block diagonal 2
        idx = diag2.index(0)
        if idx == 0:
            return (0, 2)
        elif idx == 1:
            return (1, 1)
        else:
            return (2, 0)
    
    # If no immediate win or block, use optimal opening strategy
    # Prioritize center, then corners, then edges
    optimal_order = [(1, 1), (0, 0), (0, 2), (2, 0), (2, 2), (0, 1), (1, 0), (1, 2), (2, 1)]
    
    for move in optimal_order:
        if move in possible_moves:
            return move
    
    # Fallback: return any possible move (should not reach here due to optimal_order covering all)
    return possible_moves[0]
