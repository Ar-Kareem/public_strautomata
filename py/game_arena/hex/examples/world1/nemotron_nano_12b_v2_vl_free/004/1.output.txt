To solve this problem, we need to create an AI policy for playing the game of Hex on an 11x11 grid. The objective for 'b' (black) is to connect the top and bottom edges of the board, while for 'w' (white), it is to connect the left and right edges. The AI must always return a legal move that is an empty cell on the board.

### Approach
1. **Adjacency Check**: Determine if a cell is adjacent to any of the player's stones. This is crucial as placing a stone adjacent to existing stones helps in expanding the territory.
2. **Empty Cells Identification**: Generate a list of all empty cells on the board.
3. **Adjacent Cells Collection**: Identify cells that are adjacent to the player's current stones to prioritize expanding territory.
4. **Heuristic Scoring**: Evaluate each candidate cell based on various factors such as:
   - The number of adjacent stones of the same color.
   - The proximity to the target edges (top/bottom for 'b' and left/right for 'w').
   - The distance from the center of the board to reduce redundancy and focus on central positions.
5. **Best Move Selection**: Select the cell with the highest score based on the heuristic. If no adjacent cells are available, default to a central position for the first move.

### Solution Code
```python
def policy(me, opp, color):
    me_set = set(me)
    opp_set = set(opp)
    empty = []
    for i in range(11):
        for j in range(11):
            if (i, j) not in me_set and (i, j) not in opp_set:
                empty.append((i, j))
    
    if not empty:
        return (0, 0)  # Handle unexpected case, though it should not occur
    
    def is_adjacent(a, b):
        dx = a[0] - b[0]
        dy = a[1] - b[1]
        return (dx == 0 and abs(dy) == 1) or (dy == 0 and abs(dx) == 1) or (abs(dx) == 1 and abs(dy) == 1)
    
    adjacent_cells = []
    for cell in empty:
        for m_cell in me:
            if is_adjacent(cell, m_cell):
                adjacent_cells.append(cell)
                break
    
    candidates = adjacent_cells if adjacent_cells else empty
    
    best_move = None
    best_score = -float('inf')
    
    for cell in candidates:
        cell_x, cell_y = cell
        adj_own = 0
        adj_opp = 0
        
        # Check all 6 neighbors
        neighbors = []
        for dx in [-1, 0, 1]:
            for dy in [-1, 0, 1]:
                if abs(dx) + abs(dy) != 1 or (dx in (0, 1) and dy in (0, 1)):
                    continue
                neighbor = (cell_x + dx, cell_y + dy)
                if 0 <= neighbor[0] < 11 and 0 <= neighbor[1] < 11:
                    if neighbor in me_set:
                        adj_own += 1
                    elif neighbor in opp_set:
                        adj_opp += 1
        
        # Heuristic score calculation
        score = adj_own * 10
        if color == 'b':
            row_priority = cell_x * 2 + (10 - abs(cell_x - 5)) * 0.5
            score += row_priority
        else:
            col_priority = cell_y * 2 + (10 - abs(cell_y - 5)) * 0.5
            score += col_priority
        
        # Penalize distance from center
        dist_penalty = (cell_x - 5)**2 + (cell_y - 5)**2
        score -= dist_penalty * 0.1
        
        # Penalize cells adjacent to opponent's stones
        if color == 'b':
            score -= adj_opp * 2
        else:
            score -= adj_opp * 2
        
        if score > best_score:
            best_move = cell
            best_score = score
    
    return best_move
```

### Explanation
1. **Adjacency Check**: The `is_adjacent` function checks if two cells are adjacent in the hexagonal grid, considering both horizontal/vertical and diagonal neighbors.
2. **Empty Cells Collection**: Iterate over the board to find all empty cells where a move can be made.
3. **Adjacent Cells Collection**: For each empty cell, check if it is adjacent to any of the player's stones. This helps prioritize moves that expand existing territory.
4. **Heuristic Scoring**: Each candidate cell is scored based on:
   - The number of adjacent stones of the same color (to enhance connectivity).
   - Proximity to target edges (rows for 'b', columns for 'w').
   - Distance from the center of the board to favor central positions.
5. **Move Selection**: The cell with the highest score is chosen as the next move, ensuring a balance between expanding the player's territory and blocking the opponent.

This approach efficiently balances between expanding the player's territory and strategically positioning stones to connect the respective edges, leveraging both adjacency and positional heuristics.
