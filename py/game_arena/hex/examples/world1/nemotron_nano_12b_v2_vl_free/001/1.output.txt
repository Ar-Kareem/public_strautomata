# Role - You are an AI competing in a gaming arena. # Objective - Write a complete Python file that provides the next move for a Hex on a 11x11 board. - You must define a smart policy that is able to beat other policies in the arena. - You will be disqualified if you do not return a legal move string. Therefore, you must always return a legal move string. # Required API The provided python file must implement the following API: ```python def policy(me: list[tuple[int, int]], opp: list[tuple[int, int]], color: str) -> tuple[int, int]: ... ``` # State Representation - You are always the player to move. - `me` is a list of (row, col) tuples containing your stones. - `opp` is a list of (row, col) tuples containing the opponent's stones. - `color` is your color: `'b'` for black which tries to connect the top and bottom sides, and `'w'` for white which tries to connect the left and right sides. - Rows and columns are 0-indexed in `0..10`. - An empty cell is any coordinate not in `me` or `opp`. - Your goal is to connect your two opposing sides; the opponent tries to connect their two opposing sides. - Note that the board is a grid each individual cell is a hexagonal shape. Thus to know which cell is touching which other cell requires spatial reasoning to determine the winning lines. - Every stone not on the edge of the board is touching exactly 6 other stones (since they are hexagonal). - Two of those 6 stones are in the same row, two are in the top row, and two are in the bottom row. - Example: The store located at (4,1) is touching b6 (4, 0) and (4, 2) - and to its bottom is touching (5, 1) and (5, 0) - and to its top is touching (3, 1) and (3, 2) # Action / Return Format - Return a single move as a tuple `(row, col)`. - The chosen cell must be empty and within the board bounds. - There is no pass; always place a stone. # Constraints - You may use any standard Python imports, including numpy. - Your code is allowed 1 second of execution time on a modern 3GHz CPU. # Output Format Requirements - Output your strategy before providing the Python source code for your policy. - Output the Python source code for your policy surrounded by `<code>...</code>` tags.

The strategy for this policy involves evaluating empty cells based on their adjacency to my stones and their distance to my target edges. For black ('b'), moves are ranked by proximity to the top/bottom edges and adjacency to my stones while penalizing adjacency to the opponent. For white ('w'), moves in columns near the left and right edges are prioritized. The heuristic first checks for corners and centers, then evaluates other moves dynamically using adjacency proximity.

def policy(me, opp, color):
    my_stones = set(me)
    opp_stones = set(opp)
    
    legal_moves = []
    for x in range(11):
        for y in range(11):
            if (x, y) not in my_stones and (x, y) not in opp_stones:
                legal_moves.append((x, y))
                
    # Handle first move specially
    if not me:
        if color == 'b':
            return (5, 4)
        else:
            return (4, 5)
    
    best_score = float('-inf')
    best_move = legal_moves[0]  # Initialize with a default move
    
    for cell in legal_moves:
        x, y = cell
        
        my_adj = 0
        opp_adj = 0
        
        # Define adjacency directions based on Hex's hex grid
        neighbors = [
            (x-1, y),
            (x+1, y),
            (x, y-1),
            (x, y+1),
            (x-1, y+1),
            (x+1, y-1),
        ]
        
        for nx, ny in neighbors:
            if 0 <= nx < 11 and 0 <= ny < 11:
                if (nx, ny) in my_stones:
                    my_adj += 1
                elif (nx, ny) in opp_stones:
                    opp_adj += 1
        
        # Base score prioritizing adjacency to my stones and penalizing opponent adjacency
        score = my_adj * 3 - opp_adj * 2
        
        # Positional bonus calculation
        if color == 'b':
            edge_dist = min(x, 10 - x)
            # Prioritize stones closer to top/bottom edges with bonus
            score += (10 - edge_dist) * 2
        else:
            # 'w' prioritizes proximity to left/right edges
            edge_dist = min(y, 10 - y)
            score += (10 - edge_dist) * 2
        
        # Update best move if current score is better
        if (score > best_score) or (score == best_score and cell < best_move):
            best_score = score
            best_move = (x, y)
            
    return best_move
