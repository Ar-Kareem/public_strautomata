The strategy for this policy is to use a combination of threat detection and strategic placement based on a modified version of the HexAI algorithm, integrating elements of breadth-first search (BFS) for connection evaluation and a minimax-like depth-1 lookahead with heuristic scoring. 

Key components:
1. **Connection Evaluation**: For each possible move, simulate placing a stone and evaluate how well it contributes to connecting your two sides (top-bottom for black, left-right for white). Use BFS to find the shortest path from one edge to the other through your stones.
2. **Threat Response**: Detect if the opponent has an imminent winning threat (one move away from connecting) and block it immediately.
3. **Center Control**: Prioritize central moves early in the game to maximize influence over the board.
4. **Edge and Corner Avoidance**: Avoid playing on edges unless necessary to block or complete a path, since edge stones have fewer expansion options.
5. **Symmetry and Pattern Recognition**: If the board is symmetric and no clear threat exists, mimic the opponent's symmetric move to maintain balance and reduce their advantage.
6. **Heuristic Scoring**: Score each empty cell based on:
   - Number of adjacent friendly stones (higher = better)
   - Proximity to own edges
   - Distance to opponent’s connected components
   - Whether it completes a bridge or fills a gap

The algorithm evaluates all empty cells, filters out illegal moves, scores them using the above heuristics, and picks the highest-scoring move. If a blocking move exists, it’s prioritized over all others.

We implement this efficiently with BFS and neighbor lookup using a fixed hexagonal adjacency rule: for cell (i, j), neighbors are:
- (i-1, j), (i-1, j+1)
- (i, j-1), (i, j+1)
- (i+1, j-1), (i+1, j)

We use a set for O(1) lookups of occupied cells and prioritize moves that connect to both your edges or extend existing chains.

from collections import deque

def policy(me: list[tuple[int, int]], opp: list[tuple[int, int]], color: str) -> tuple[int, int]:
    # Build set of occupied cells for quick lookup
    occupied = set(me) | set(opp)
    
    # Define neighbor offsets for hex grid
    # For cell (i, j), neighbors are:
    # (i-1, j), (i-1, j+1)
    # (i, j-1), (i, j+1)
    # (i+1, j-1), (i+1, j)
    neighbors_offsets = [
        (-1, 0), (-1, 1),
        (0, -1), (0, 1),
        (1, -1), (1, 0)
    ]
    
    def get_neighbors(r, c):
        """Return all valid hex neighbors of (r, c)"""
        for dr, dc in neighbors_offsets:
            nr, nc = r + dr, c + dc
            if 0 <= nr <= 10 and 0 <= nc <= 10:
                yield (nr, nc)
    
    def bfs_connects(starts, ends):
        """Check if any stone in `starts` can reach any stone in `ends` using BFS"""
        if not starts:
            return False
        visited = set(starts)
        queue = deque(starts)
        
        while queue:
            r, c = queue.popleft()
            if (r, c) in ends:
                return True
            for nr, nc in get_neighbors(r, c):
                if (nr, nc) not in visited and (nr, nc) in occupied:
                    visited.add((nr, nc))
                    queue.append((nr, nc))
        return False
    
    def get_edge_cells(color):
        """Return set of edge cells for the given color"""
        if color == 'b':  # Black: top to bottom
            return {(0, j) for j in range(11)} | {(10, j) for j in range(11)}
        else:  # White: left to right
            return {(i, 0) for i in range(11)} | {(i, 10) for i in range(11)}
    
    def count_adjacent_friends(r, c, player_stones):
        """Count how many of the 6 neighbors are occupied by player"""
        return sum(1 for nr, nc in get_neighbors(r, c) if (nr, nc) in player_stones)

        """Check if opponent is one move away from winning"""
        opp_stones = set(opp)
        target_edges = get_edge_cells(color)

        for r in range(11):
            for c in range(11):

                    continue

                if bfs_connects(opp_stones & target_edges, opp_stones & target_edges):

                    opp_stones.remove((r, c))
                    return (r, c)

    target_edges = get_edge_cells(color)
    player_stones = set(me)

    if len(me) == 0 and len(opp) == 0:
        return (5, 5)

    if len(me) == 0:

        if (5, 5) not in occupied:
            return (5, 5)
        # Else, take a symmetric position if opponent played on edge
        op = opp[0]
        # For white (left-right), mirror vertically if opponent is on left edge
        if color == 'w' and op[1] == 0:
            return (op[0], 10)
        # For black (top-bottom), mirror horizontally if opponent is on top edge
        elif color == 'b' and op[0] == 0:
            return (10, op[1])

        for nr, nc in get_neighbors(op[0], op[1]):
            if (nr, nc) not in occupied:

    best_score = -1
    best_move = None
    
    # Heuristic scoring function
    def evaluate_move(r, c):
        score = 0
        # Bonus for connecting to own edge
        if (r, c) in target_edges:
            score += 5
        
        # Count adjacent friendly stones
        adj_friends = count_adjacent_friends(r, c, player_stones)
        score += adj_friends * 3
        
        # Proximity to opponent's stones (penalize being isolated)
        adj_opp = sum(1 for nr, nc in get_neighbors(r, c) if (nr, nc) in set(opp))
        score -= adj_opp * 0.5
        
        # Prefer central positions
        center_dist = abs(r - 5) + abs(c - 5)
        score += (10 - center_dist) * 0.2

        # Check connectivity to both edges (if we have stones near both)

        if adj_friends > 0:

            test_stones = player_stones | {(r, c)}
            test_edges = test_stones & target_edges
            if len(test_edges) > 0 and bfs_connects(test_edges, test_edges):

                if bfs_connects(test_stones & target_edges, test_stones & target_edges):

    if candidates:

    for r in range(11):
        for c in range(11):

                return (r, c)

    raise RuntimeError("No legal move available")
