The goal is to connect all of the player's pieces into a contiguous group. The policy must generate valid moves according to the game rules and prioritize moves that capture opponent's pieces first, followed by moves that maximize the number of steps taken.

### Approach
1. **Generate Possible Moves**: For each of the player's pieces on the board, evaluate all possible directions (horizontal, vertical, and diagonals) for movement.
2. **Calculate Line Count**: For each direction, determine the number of pieces (both players) in the corresponding line (row, column, or diagonal).
3. **Validate Moves**: For each potential move, check if the destination is valid (within board boundaries) and ensure the path does not contain any enemy pieces. The destination cell must be either empty or contain an opponent's piece.
4. **Prioritize Moves**: Prioritize moves that capture opponent pieces. If no captures are possible, select the move with the maximum number of steps taken, as longer moves can provide better strategic positioning.

### Explanation
1. **Direction Handling**: The code evaluates all 8 possible directions (left, right, up, down, and four diagonals) for each piece.
2. **Line Count Calculation**: For each direction, the code calculates the number of pieces (count) in the corresponding line (row, column, or diagonal).
