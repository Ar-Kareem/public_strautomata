The strategy for this policy is based on a combination of material evaluation, threat assessment, and positional awareness, implemented with a minimax-like look-ahead (depth 2) and alpha-beta pruning principles adapted to the constraint of only receiving legal moves without board state reconstruction. Since we cannot reconstruct the full board state or use external chess libraries, we simulate the effects of moves by analyzing:

1. **Immediate material gain**: Prioritize moves that capture higher-value pieces (Queen > Rook > Bishop/Knight > Pawn).
2. **Checkmate detection**: Look for moves that put the opponent’s king in check with no escape (simplified by checking if the opponent’s king is under attack and all opponent moves fail to remove the threat—we infer this by checking if the opponent’s king has any legal squares to move to after our move, using basic adjacency and attack patterns).
3. **King safety**: Avoid moves that expose our own king to diagonal or straight-line attacks, especially if opponent's queen, rook, or bishop can exploit open lines.
4. **Center control**: Prefer moves that advance pawns or pieces toward the center (d4, d5, e4, e5).
5. **Pawn promotion**: Prioritize pawn promotions to queen if available.
6. **Castling**: If safe and available, prefer castling to improve king safety.
7. **Defensive awareness**: If our king is in check, we must respond only to check-evading moves (but legal_moves is provided so we only consider those).
8. **Fallback**: If no clear tactical move exists, choose the move that maximizes piece mobility or centralizes our pieces.

To implement the above without full board reconstruction, we do:
- A fast evaluation of each legal move’s impact using pattern matching and heuristic scoring.
- We maintain a memory of past positions to detect repetition or recurring threats (e.g., if opponent keeps attacking the same square).
- We track opponent king position via `pieces` each turn to assess check threats.

We assume the move generator provides only legal moves, so we do not validate legality—only select the best one.

We use a 2-ply heuristic: simulate the move, then assume opponent makes the best reply (capturing or threatening our material), and score accordingly.

def policy(pieces: dict[str, str], to_play: str, memory: dict) -> tuple[str, dict]:

    piece_values = {'P': 1, 'N': 3, 'B': 3, 'R': 5, 'Q': 9, 'K': 0}
    color = 'w' if to_play == 'white' else 'b'
    opponent_color = 'b' if color == 'w' else 'w'
    
    # Extract king positions
    our_king = None
    opponent_king = None
    for square, piece in pieces.items():
        if piece == color + 'K':
            our_king = square
        elif piece == opponent_color + 'K':
            opponent_king = square
    
    # If we can't find our king or opponent's king, return a random legal move (should not happen)
    if not our_king or not opponent_king:
        return next(iter(pieces.keys())) + next(iter(pieces.keys())), memory
    
    legal_moves = memory.get('previous_legal_moves', [])  # Optional: maintain for tracking
    # Update memory with current legal moves for future use
    memory['previous_legal_moves'] = legal_moves
    
    # If only one move, return it
    if len(legal_moves) == 1:
        return legal_moves[0], memory

    def square_to_coords(sq):
        file = ord(sq[0]) - ord('a')
        rank = int(sq[1]) - 1
        return file, rank
    
    # Helper: Convert coords to square
    def coords_to_square(file, rank):
        return chr(ord('a') + file) + str(rank + 1)

    def is_occupied(sq, clr):
        return pieces.get(sq, '')[:1] == clr

    def is_occupied_any(sq):
        return sq in pieces

    def get_attacks(square, piece_code):
        file, rank = square_to_coords(square)
        attacks = []
        piece = piece_code[1]
        clr = piece_code[0]
        
        if piece == 'P':
            dir_ = 1 if clr == 'w' else -1

            if 0 <= rank + dir_ <= 7:
                forward = coords_to_square(file, rank + dir_)
                if not is_occupied_any(forward):
                    attacks.append(forward)

            for df in [-1, 1]:

                    if is_occupied(capture_sq, opponent_color):
                        attacks.append(capture_sq)
        
        elif piece == 'N':

                nf, nr = file + df, rank + dr
                if 0 <= nf <= 7 and 0 <= nr <= 7:
                    sq = coords_to_square(nf, nr)
                    attacks.append(sq)

                    nf, nr = file + df*step, rank + dr*step
                    if not (0 <= nf <= 7 and 0 <= nr <= 7):
                        break
                    sq = coords_to_square(nf, nr)
                    attacks.append(sq)
                    if is_occupied_any(sq):
                        break
        
        elif piece == 'R':

                    sq = coords_to_square(nf, nr)

                    if is_occupied_any(sq):
                        break

        captured_piece = pieces.get(to_sq, '')

        if will_put_opponent_in_check(move):

            new_pieces = pieces.copy()
            del new_pieces[from_sq]

                new_pieces[to_sq] = color + promoted_piece.upper()

            king_file, king_rank = square_to_coords(opponent_king)

                    under_attack = False
                    for our_sq, our_pc in new_pieces.items():

                                under_attack = True
                                break
                    if not under_attack:

                        return 0  # It's check, but not checkmate

            return 1000000

        if not is_our_king_in_check():
