To solve this problem, we need to design a chess move selection policy that can compete effectively in a gaming arena. The policy should prioritize moves that lead to checkmate, winning material, improving king safety, and preventing immediate threats. The solution involves analyzing the current board state and the list of legal moves to make an intelligent decision.

### Approach
1. **Checkmate Check**: If any legal move results in checkmate (indicated by '#'), return that move immediately since it wins the game.
2. **Capturing Moves**: Prioritize moves that capture opponent's pieces. The value of the captured piece (pawn=1, knight/bishop=3, rook=5, queen=9) is used to determine the best capture. For en passant captures, the pawn value (1) is assumed.
3. **Non-Capturing Checks**: If no capturing moves are available, return the first non-capturing move that gives check (indicated by '+').
4. **King Safety and Development**: For remaining moves, prioritize castling (king-side or queen-side) as it improves king safety. Next, prioritize moves to central squares (d4, d5, e4, e5) as they control the center and aid development. If no such moves are found, return the first available move.

### Solution Code
```python
def policy(pieces: dict[str, str], to_play: str, legal_moves: list[str]) -> str:
    # Step 1: Check for checkmate
    for move in legal_moves:
        if move.endswith('#'):
            return move
    
    # Step 2: Capture moves with highest captured piece value
    piece_values = {'P': 1, 'N': 3, 'B': 3, 'R': 5, 'Q': 9, 'K': 0}
    best_capture_move = None
    max_captured_value = -1
    
    for move in legal_moves:
        if 'x' in move:
            parts = move.split('x')
            target_part = parts[-1]
            target_square = target_part[0:2]
            captured_piece = pieces.get(target_square)
            if captured_piece is not None:
                captured_value = piece_values[captured_piece[1]]
            else:
                if move[0].islower():
                    captured_value = 1
                else:
                    captured_value = 0
            if captured_value > max_captured_value:
                max_captured_value = captured_value
                best_capture_move = move
    
    if best_capture_move is not None:
        return best_capture_move
    
    # Step 3: Non-capturing checks
    for move in legal_moves:
        if 'x' not in move and move.endswith('+'):
            return move
    
    # Step 4: Non-capturing non-checking moves
    best_move = None
    best_score = -1
    
    for move in legal_moves:
        if 'x' not in move and not move.endswith('+'):
            score = 0
            if move in ['O-O', 'O-O-O']:
                score = 10
            else:
                if '=' in move:
                    target_square = move.split('=')[0]
                else:
                    target_square = move[-2:]
                if target_square in ['d4', 'd5', 'e4', 'e5']:
                    score = 1
            if score > best_score:
                best_score = score
                best_move = move
    
    if best_move is not None:
        return best_move
    
    return legal_moves[0]
```

### Explanation
1. **Checkmate Check**: The policy first checks if any move ends with '#', indicating a checkmate. If found, it returns immediately as this wins the game.
2. **Capture Moves**: For moves involving captures ('x'), the policy evaluates the value of the captured piece. Higher value captures (queen, rook, etc.) are prioritized. Special handling is done for en passant captures (treated as pawn captures with value 1).
3. **Non-Capturing Checks**: If no capturing moves are available, the policy looks for moves that give check (indicated by '+') and returns the first such move.
4. **King Safety and Development**: For remaining moves, the policy prioritizes castling (king-side or queen-side) for king safety. Next, it prioritizes moves to central squares (d4, d5, e4, e5) to control the center and develop pieces. If no such moves are found, the first available move in the list is returned.

This approach ensures that the policy makes strong, tactical moves while prioritizing safety and development, making it competitive in the gaming arena.
