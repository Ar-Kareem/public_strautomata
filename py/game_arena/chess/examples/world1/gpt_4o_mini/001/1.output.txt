The strategy for this policy is to prioritize tactical advantages and immediate threats. The algorithm first checks for any moves that can deliver checkmate immediately. If no immediate checkmate moves are available, it evaluates the potential to capture opponent's pieces while ensuring the player's king remains safe. The logic also includes considerations for improving piece positioning and controlling key squares. If no significant tactical advantage can be found, a move that keeps the overall positions stable and defensively sound is selected.

def policy(pieces: dict[str, str], to_play: str, legal_moves: list[str]) -> str:
    # Helper function to evaluate the significance of a move
    def evaluate_move(move):
        # Check if the move is a check or checkmate
        if 'x' in move:
            return 2  # Priority for capturing
        if '+' in move or 'checkmate' in move:
            return 3  # High priority for check/checkmate
        if '=' in move:
            return 4  # Priority for promotion
        return 1  # Default

    # Identify the best move
    best_move = None
    best_value = 0

    for move in legal_moves:
        move_value = evaluate_move(move)

        # Select the best move based on evaluation
        if move_value > best_value:
            best_value = move_value
            best_move = move

    return best_move
