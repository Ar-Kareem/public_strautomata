The strategy for this policy is to implement a multi-layered chess evaluation system that prioritizes:

1. **Immediate checkmate** — if any legal move delivers checkmate, return it immediately.
2. **Material gain** — prioritize moves that capture high-value pieces (Queen > Rook > Bishop/Knight > Pawn), especially if the captured piece is undefended.
3. **Threat prevention** — if the opponent has an imminent threat (e.g., a hanging queen or mate-in-one), prioritize defending or counterattacking.
4. **Piece development and control** — favor moves that develop pieces toward the center, control key squares, or improve king safety (especially castling if available and safe).
5. **Promotion** — if any pawn can promote, prefer promotion to Queen unless underpromotion is strategically better (e.g., avoiding stalemate or knight promotion for fork potential).
6. **Disambiguation and precision** — use evaluations to break ties between similar moves by favoring those that increase mobility or control central squares.
7. **King safety** — avoid moves that expose the king, especially in the endgame.

We use a simplified material value system: Q=9, R=5, B=N=3, P=1. We also perform a shallow 1-ply lookahead to detect if a move leaves our king in check AFTER the move (though the API ensures legal moves, so this is redundant — but we check opponent threats to our pieces).

We’ll evaluate each legal move by:
- Calculating material gain (if capture)
- Checking if it delivers checkmate (via actual simulation — we do a minimal board state check)
- Preferring center control (e.g., moves to d4, d5, e4, e5)
- Favoring castling if it improves king safety

Since we only have 1 second and legal_moves is small, we can simulate a lightweight board state for threat detection.

                        return True

                            return True

                                blocked = True
                                break

                            return True
