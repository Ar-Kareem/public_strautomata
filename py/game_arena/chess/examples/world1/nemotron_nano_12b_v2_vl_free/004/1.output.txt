To solve this problem, we need to create a policy function that selects the best legal move in a chess game based on a set of heuristics. The goal is to prioritize moves that lead to checkmate, improve material advantage, ensure king safety, and develop pieces effectively.

### Approach
The solution uses a heuristic-based approach to evaluate and prioritize moves. The key steps are:

1. **Promotions**: Moves that promote a pawn to a queen (high-value piece) are prioritized highest as they can significantly change the game's dynamics.
2. **Captures**: Moves that capture the highest value opponent piece are next prioritized, considering both the captured piece's value and the cost of the capturing piece.
3. **Check Moves**: Moves that result in the opponent's king being in check are prioritized to increase pressure on the opponent.
4. **Castling**: Moves that castle the king are prioritized for king safety.
5. **Default Move**: If no high-priority moves are found, the first legal move is returned.

### Solution Code
```python
def policy(pieces: dict[str, str], to_play: str, legal_moves: list[str]) -> str:
    value_map = {'Q': 900, 'R': 500, 'B': 300, 'N': 300, 'P': 100}

    scored_moves = []
    for move in legal_moves:
        score = 0

        # Check for promotions (prioritized highest)
        if '=' in move:
            _, promoted_piece = move.split('=', 1)
            current_score = 10000 + value_map.get(promoted_piece, 0)
        # Check for captures
        elif 'x' in move:
            dest_square = move.split('x', 1)[1]
            captured_piece = pieces.get(dest_square, '')
            if captured_piece:
                captured_val = value_map.get(captured_piece[1], 0)
                capturing_char = move[0] if move[0] in value_map else 'P'
                capturing_val = value_map.get(capturing_char, 0)
                current_score = captured_val - capturing_val
            else:
                current_score = 0
        # Check moves
        elif move.endswith('+'):
            current_score += 2500
        # Castling moves
        elif move.startswith('O-'):
            current_score += 2000
        else:
            current_score = 0

        scored_moves.append((-current_score, move))

    scored_moves.sort()
    return scored_moves[0][1]
```

### Explanation
1. **Promotions**: Moves that promote a pawn to a queen (e.g., `b8=Q`) are prioritized by assigning them the highest score (10000 plus the queen's value, making them top priority).
2. **Captures**: The function evaluates captures by the net gain of material, calculated as the value of the captured piece minus the value of the capturing piece. This ensures that moves resulting in higher net gains are prioritized.
3. **Check Moves**: Moves that result in the opponent's king being in check (indicated by `+` in the move string) are scored higher to apply pressure.
4. **Castling**: Moves that castle the king (e.g., `O-O` or `O-O-O`) are prioritized for king safety.
5. **Default Handling**: If no high-priority moves are found, the function defaults to the first move in the list, ensuring a valid move is always returned.

This approach efficiently balances between capturing high-value pieces, checking the opponent, and protecting the king, ensuring optimal move selection within the given constraints.
