To solve this problem, we need to develop a policy function in Python that selects the best move in a Chess game given the current board state, the player whose turn it is, and the list of legal moves. The policy should prioritize checkmate moves first, then capture the highest-value pieces, and fallback to other moves if necessary.

### Approach
1. **Check for Checkmate Moves**: The first priority is to check if any of the legal moves result in a checkmate. If such a move exists, it is selected immediately.
2. **Check for Captures**: If no checkmate is possible, the policy then checks for captures. The move that captures the highest-value piece is prioritized. If multiple captures have the same value, the move involving the lowest-value attacker piece is selected.
3. **Fallback**: If no captures or checkmate moves are available, the code defaults to the first move in the list of legal moves.

### Solution Code
```python
def policy(pieces: dict, to_play: str, legal_moves: list) -> str:
    # Define the value of each piece type.
    value_map = {'K': float('inf'), 'Q': 9, 'R': 5, 'B': 3, 'N': 3, 'P': 1}
    
    # Check for checkmate moves first.
    for move in legal_moves:
        if move.endswith('#'):
            return move
    
    # Check for captures.
    capture_candidates = []
    for move in legal_moves:
        if 'x' not in move:
            continue
        try:
            target_square = move.split('x', 1)[1]
        except IndexError:
            continue  # invalid move format
        if target_square not in pieces:
            continue
        captured_piece = pieces[target_square]
        if captured_piece[0] == ('w' if to_play == 'black' else 'b'):
            continue  # cannot capture own pieces
        captured_type = captured_piece[1]
        captured_val = value_map.get(captured_type, 0)
        if captured_val == 0:
            continue
        # Determine the attacker's type.
        attacker_part = move.split('x')[0]
        attacker_type = attacker_part[0] if len(attacker_part) == 1 else 'P'
        attacker_val = value_map.get(attacker_type, 0)
        net_gain = captured_val - attacker_val
        capture_candidates.append((net_gain, captured_val, attacker_val, move))
    
    if capture_candidates:
        # Sort by highest captured value, then lowest attacker value.
        capture_candidates.sort(key=lambda x: (-x[1], x[2]))
        best_candidates = [cm for cm in capture_candidates if cm[1] == capture_candidates[0][1]]
        # Among these, choose the one with the lowest attacker value (smallest loss).
        best_net = max(c[0] for c in best_candidates)
        best_net_moves = [cm for cm in best_candidates if cm[0] == best_net]
        return best_net_moves[0][3]
    
    # Check if there's any check move (ending with '+'), though not required by problem statement
    # but can be a good heuristic. Here, we simply return the first move.
    return legal_moves[0]
```

### Explanation
1. **Check for Checkmate**: The function first iterates through all legal moves to check if any move results in a checkmate (denoted by `#` in the move string). If found, it returns the first such move.
2. **Capture Evaluation**: The function then evaluates all captures by examining each move string for the presence of `x` (indicating a capture). For each valid capture move, it calculates the value of the captured piece and the attacking piece's value. The net gain (value of captured piece minus attacker's value) is used to prioritize moves. Moves are sorted by the value of the captured piece in descending order, and among moves with the same captured value, those with the lowest attacker value are preferred.
3. **Fallback Fallback**: If no captures or checkmates are found, the function defaults to returning the first move in the list of legal moves, which is a simple but necessary fallback.

This approach ensures that the policy adheres to chess strategy principles, prioritizing checkmate, maximizing material gain through captures, and using a fallback strategy when no other options are available.
