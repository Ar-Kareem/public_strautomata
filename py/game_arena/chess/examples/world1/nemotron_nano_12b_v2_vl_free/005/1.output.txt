### Strategy

The strategy for selecting the optimal move in chess is based on a priority-based heuristic system that prioritizes the most advantageous outcomes. The hierarchy of move selection is as follows:

1. **Promotion to Queen**: If a player has a pawn that can promote to a queen, this move is selected. Promoting to a queen is highly advantageous due to its high mobility and power.

2. **High-Value Capture**: If there are no queen promotions, the next priority is to capture the opponent's highest-value piece (queen, rook, bishop/knight, pawn). This helps minimize material loss and gain material advantage.

3. **Castling**: If the player has not promoted and no captures are available, the code prioritizes castling for king safety. Kingside castle is preferred over queenside castle as it centralizes the king more effectively.

4. **Default Move**: If none of the above criteria are met, the code defaults to the first move in the list of legal moves.

This approach ensures that the player makes strategic decisions based on known chess tactics, such as promoting pawns and capturing high-value pieces, while also prioritizing king safety through castling.

### Python Code

```python
def policy(pieces: dict[str, str], to_play: str, legal_moves: list[str]) -> str:
    # Helper function to determine the value of a piece type
    def piece_value(pc: str) -> int:
        if pc == 'Q' or pc == 'q':
            return 9
        elif pc == 'R' or pc == 'r':
            return 5
        elif pc == 'B' or pc == 'b' or pc == 'N' or pc == 'n':
            return 3
        elif pc == 'P' or pc == 'p':
            return 1
        return 0

    # Step 1: Check for pawn promotions first (prefer queen)
    promotions = [m for m in legal_moves if m.endswith('=Q')]
    if promotions:
        return promotions[0]

    # Step 2: Evaluate captures
    best_move = None
    max_captured_value = -1

    for move in legal_moves:
        if 'x' in move:
            # Extract the square being captured
            captured_square = move.split('x')[-1]
            captured_piece = pieces.get(captured_square, '')
            if not captured_piece:
                continue  # Not a valid capture (though impossible in legal moves)

            # Determine the value of the captured piece
            pc_type = captured_piece[1]
            move_value = piece_value(pc_type)

            # Update the best move if this capture is better
            if move_value > max_captured_value:
                max_captured_value = move_value
                best_move = move

    if best_move is not None:
        return best_move

    # Step 3: Prioritize castling for king safety
    if 'O-O' in legal_moves:
        return 'O-O'
    elif 'O-O-O' in legal_moves:
        return 'O-O-O'

    # Step 4: Return the first legal move if nothing else applies
    return legal_moves[0] if legal_moves else None
```

---

### Explanation

- **Promotions (Step 1):** The code first checks if there are any pawn promotions to queen available. It returns the first such move to take advantage of the queen’s power.
  
- **Captures (Step 2):** The code evaluates all legal moves that involve capturing an opponent's piece. It uses a helper function `piece_value` to assign values (9 for queens, 5 for rooks, etc.) and selects the move that captures the highest-value piece.

- **Castling (Step 3):** If no promotions or captures are available, the code checks for castling moves. Kingside castle is prioritized as it better protects the king.

- **Default (Step 4):** If no other priorities apply, the code returns the first move in the list of legal moves.

This strategy ensures that the AI makes the most strategically sound move based on the current game state, prioritizing material and safety.
