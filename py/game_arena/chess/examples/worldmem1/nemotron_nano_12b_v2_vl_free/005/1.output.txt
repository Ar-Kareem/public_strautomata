### Approach
The solution uses a heuristic-based approach to prioritize moves based on their strategic value. The steps are as follows:

1. **Check for Checkmate Moves**: If any move results in a checkmate (indicated by a `#` in the move string), return the first such move.
2. **Promotions**: Check for moves that promote a pawn, prioritizing promotions to higher-value pieces like queens and rooks.
3. **Capture Evaluation**: Evaluate moves based on the value of the piece being captured. Higher-value pieces (queens, rooks, knights, bishops) are prioritized over lower-value ones (pawns).
4. **Default Move**: If no checkmate, promotions, or high-value captures are found, return the first legal move as a fallback.

### Explanation
1. **Checkmate Check**: The code first checks for any move that results in a checkmate (denoted by `#`). If found, it returns the first such move to immediately end the game.
2. **Promotions**: If no checkmate move is found, the code looks for promotion moves. Pawn promotions to higher-value pieces (queens, rooks, bishops, knights) are prioritized, with queens being the most valuable.
3. **Capture Evaluation**: The code evaluates captures by checking the value of the piece being captured. Moves capturing higher-value pieces are prioritized. If a move includes a promotion, it is considered as a capture with added value.
4. **Fallback Move**: If no other moves meet the criteria, the code returns the first move from the legal moves list as a fallback.

This approach ensures that the most strategically advantageous move is chosen based on the current game state, prioritizing checkmate, promotions, and high-value captures.
