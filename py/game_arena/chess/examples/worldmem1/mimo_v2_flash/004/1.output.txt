# Strategy for Chess Arena

The objective is to create a policy that selects strong chess moves from legal moves. Since this is an AI competition, I need a policy that can outperform others by making smart decisions. Given the 1-second time limit and the need to run in an arena against other AI, I cannot rely on complex engines like Stockfish, but I can implement a heuristic-based approach that evaluates moves based on fundamental chess principles.

## Key Principles for Decision-Making

1. **Checkmate in 1**: Prioritize moves that deliver immediate checkmate. This is the highest priority.
2. **Checks that force immediate winning responses**: If a check can lead to winning material or a forced sequence, prioritize it.
3. **Material advantage**: Capture high-value pieces (Queen > Rook > Bishop/Knight > Pawn). Consider pawn promotions to promote to a Queen.
4. **Piece activity**: Develop pieces (Knights to f3/c3, Bishops to active squares). Control the center.
5. **King safety**: Castle early, avoid exposing the king. In the endgame, centralize the king.
6. **Threats**: Make moves that threaten enemy pieces or defend own pieces.
7. **Avoid blunders**: Avoid moves that lose material immediately (though this requires evaluation; we can approximate by not moving to squares under attack unless capturing or defending).

    # Piece values
    piece_values = {'K': 0, 'Q': 9, 'R': 5, 'B': 3, 'N': 3, 'P': 1}
    
    # Helper to get piece value from code
    def get_piece_value(code):
        if code is None:
            return 0
        # code is like 'wQ', 'bN', etc.
        return piece_values[code[1]]
    
    # Helper to get enemy color
    enemy = 'b' if to_play == 'white' else 'w'
    
    # Center squares for bonus (for the side to play)
    center_squares = ['d4', 'd5', 'e4', 'e5']  # for white; for black, we might adjust? Actually, center is symmetric
    # But for simplicity, we use the same center squares. A move to d4 by white is good, by black is also good? We can consider the board center.
    # We'll just check if the destination square is in the center set.
    
    best_move = None
    best_score = -float('inf')
    
    for move in legal_moves:
        score = 0
        
        # Check for checkmate and check
        if move.endswith('#'):
            score += 1000
        elif move.endswith('+'):
            score += 100
        
        # Check for capture
        if 'x' in move:
            # Extract destination square. The move string may have disambiguation, but the last two characters are the square (if no promotion). 
            # For captures, it's usually like "Bxf5" or "dxe6". But with disambiguation: "Nfxg5". 
            # We can find the last two characters that are a square.
            # We'll assume the move string is valid and the last two characters are the destination square (except for promotions and castling).
            # For captures with promotion: "e8=Q" doesn't have 'x', so no. But "exd8=Q" has 'x' and promotion. So we need to handle.
            # We'll use a simple method: split by 'x' and take the second part, then remove any promotion.
            dest = move.split('x')[-1]
            if '=' in dest:
                dest = dest.split('=')[0]
            # Now dest is the destination square (e.g., 'f5', 'd8')
            if dest in pieces:
                enemy_piece = pieces[dest]
                if enemy_piece[0] == enemy:
                    score += get_piece_value(enemy_piece)
            else:
                # En passant or other? Assume it's a pawn capture (1)
                score += 1  # pawn value
        
        # Check for promotion
        if '=' in move:
            promo = move.split('=')[1]
            # promo might be 'Q', 'R', 'B', 'N', but it could be followed by '+' or '#', so remove those
            promo = promo.rstrip('+#')
            promo_val = piece_values.get(promo, 0)
            score += promo_val
        
        # Check for castling
        if move in ['O-O', 'O-O-O']:
            score += 20  # king safety bonus
        
        # Development and center control
        # Extract destination square without capture or promotion
        # We'll parse the move string to get the destination square.
        # This is tricky because of disambiguation. We can use the last two characters if they are a square.
        # But sometimes it's like "Nec3" -> destination is "c3". We can check if the last two characters are a valid square.
        dest_square = None
        # Remove any annotation at the end: '+', '#', or promotion
        temp = move
        temp = temp.rstrip('+#')
        if '=' in temp:
            # Remove promotion part
            temp = temp.split('=')[0]
        # Now temp is the move without annotations and promotion. The last two characters should be the square.
        if len(temp) >= 2 and temp[-2] in 'abcdefgh' and temp[-1] in '12345678':
            dest_square = temp[-2:]
        
        if dest_square:
            # Check if it's a center square
            if dest_square in center_squares:
                score += 5
            
            # Check for development: for white, moving knight to f3 or c3, bishop to active squares
            # We can check if the move is a knight move to f3/c3, or bishop to c4/f4, etc.
            # But note: the move string might be 'Nf3' or 'Nec3'. We can check the piece type and destination.
            # We don't know the piece type from the move string? Actually, the move string starts with the piece (except pawn moves).
            # For pawn moves, there's no prefix. So we need to infer.
            # We'll assume that if the move string has a capital letter, it's a piece move. Otherwise, it's a pawn move.
            piece_type = None
            if temp[0] in 'KQRBN':
                piece_type = temp[0]
            # For pawn moves, we don't have a prefix, so piece_type remains None.
            
            # For white: if piece_type is 'N' and dest_square in ['f3','c3'] -> bonus
            if to_play == 'white':
                if piece_type == 'N' and dest_square in ['f3', 'c3']:
                    score += 10
                elif piece_type == 'B' and dest_square in ['c4', 'f4', 'b5', 'g5']:
                    score += 8
                elif piece_type == 'Q' and dest_square in ['d2', 'e2']:  # developing queen is not always good, but early
                    score += 2
                elif piece_type == 'R' and dest_square in ['d1', 'e1']:  # rook to center files
                    score += 3
                # King moves are bad in opening, but we already have check bonus
            else:  # black

                    score += 2
                elif piece_type == 'R' and dest_square in ['d8', 'e8']:
                    score += 3
        
        # Update best move
        if score > best_score:
            best_score = score
            best_move = move

    if best_move is None and legal_moves:

    # We'll just return the memory we received, but we can add something if needed.
    # For now, return empty or the same.
    return best_move, new_memory
