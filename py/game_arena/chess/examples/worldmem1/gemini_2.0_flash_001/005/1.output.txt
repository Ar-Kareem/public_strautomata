The strategy for this policy is a rule-based system that prioritizes:

1. **Checkmate:** Immediately take checkmate if available.
2. **Captures with High-Value Pieces:** Capture opponent pieces with higher value pieces (Queen > Rook > Bishop/Knight > Pawn).
3. **Checks:** Give checks to force the opponent King to move, disrupting their plans.
4. **King Safety:** Move the King to a safer position (e.g., castling).
5. **Pawn Promotion:** Promote pawns to Queens if possible.
6. **Random Move:** If none of the above conditions are met, select a random legal move to guarantee compliance.

This policy makes the code efficient and should be able to achieve reasonable performance within the time constraint. It is also written to handle all inputs from the prompt, where the legal moves are assumed correct.

        return 9

    elif piece == 'B':
        return 3
    elif piece == 'N':
        return 3
    elif piece == 'P':
        return 1
    elif piece == 'K':
        return 0  # King has no offensive value
    else:
        return 0
