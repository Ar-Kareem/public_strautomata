To solve this problem, we need to create a chess AI that can efficiently select the next move based on a given board state and the current player's turn. The solution must prioritize moves that lead to checkmate, followed by moves that provide material advantage, and other strategic considerations.

### Approach
1. **Parse the Board State**: Convert the given pieces dictionary into a `chess.Board` object, which represents the current state of the game.
2. **Generate Legal Moves**: Use the `chess.Board` object to generate all legal moves for the current player.
3. **Check for Checkmate Moves**: Iterate through all legal moves to check if any of them results in checkmate, and return the first such move if found.
4. **Evaluate Moves**: If no checkmate is found, evaluate each move based on its strategic impact. The evaluation considers material difference adjusted for the current player's perspective.
5. **Select the Best Move**: Sort the moves based on their evaluation score and return the best move, breaking ties lexicographically.

### Solution Code
```python
def policy(pieces: dict[str, str], to_play: str) -> str:
    import chess

    # Initialize the board and reset all pieces
    board = chess.Board()
    for i in range(64):
        board.clear(i)
    
    # Populate the board with given pieces
    for square, piece in pieces.items():
        # Parse square and piece components
        file_char = square[0]
        rank_char = square[1]
        file_idx = ord(file_char) - ord('a')
        rank_idx = int(rank_char) - 1
        square_int = rank_idx * 8 + file_idx
        
        color = chess.WHITE if piece[0] == 'w' else chess.BLACK
        piece_type = piece[1]
        board.set_piece_at(square_int, chess.Piece(piece_type, color))
    
    # Get all legal moves for the current player
    legal_moves = list(board.moves())
    
    # Check for checkmate moves first
    for move in legal_moves:
        new_board = chess.Board(board)
        new_board.push(move)
        if new_board.is_checkmate():
            return move.uci()
    
    # Evaluate each move's impact
    evaluations = []
    for move in legal_moves:
        temp_board = board.copy()
        temp_board.push(move)
        # Evaluate from white's perspective, adjust for current player
        score = temp_board.evaluate()
        if to_play == 'black':
            score = -score  # Minimize white's advantage
        evaluations.append((move.uci(), score))
    
    # Sort by descending score, then lexicographical order
    evaluations.sort(key=lambda x: (-x[1], x[0]))
    return evaluations[0][0]
```

### Explanation
1. **Board Initialization**: The `chess.Board` object is initialized and populated with pieces from the given dictionary, converting each piece's position and type into the chess module's internal representation.
2. **Legal Moves Generation**: The `board.moves()` method generates all legal moves for the current player, which are then converted to UCI format for evaluation.
3. **Checkmate Check**: Each move is tested to see if it results in checkmating the opponent. If such a move is found, it is returned immediately.
4. **Move Evaluation**: For each legal move, the board's state after executing the move is evaluated. The evaluation is adjusted based on the current player's perspective, and moves are sorted by score to determine the best move.
5. **Result Selection**: The move with the highest evaluation score is selected, ensuring optimal play according to material advantage, king safety, and other strategic factors, adhering to the problem's requirements.
