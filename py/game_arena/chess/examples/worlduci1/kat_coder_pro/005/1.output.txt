# Strategy

The strategy for this policy is to evaluate each legal move based on a combination of heuristics:

1. **Checkmate Detection**: If a move leads to a checkmate, prioritize it above all other moves
2. **Material Gain**: Calculate the immediate material value gained from captures
3. **Threat Assessment**: Identify and prevent immediate threats to the player's own pieces
4. **Positional Evaluation**: Consider piece activity and board control
5. **King Safety**: Prioritize moves that improve king safety

The policy will:
- First check if any move results in checkmate
- Then evaluate captures based on piece value (Queen=9, Rook=5, Bishop=3, Knight=3, Pawn=1)
- Consider defensive moves that protect threatened pieces
- Use positional bonuses for central control and piece development
- Fall back to random selection among moves with similar evaluation scores

    for move in legal_moves:

    if best_moves:
        return random.choice(best_moves)
    
    # Fallback - should not reach here
    return random.choice(legal_moves)

def square_to_coords(square: str) -> tuple[int, int]:
    """Convert algebraic notation to coordinates (0-7, 0-7)"""
    file = square[0]
    rank = int(square[1])
    col = ord(file) - ord('a')
    row = rank - 1
    return row, col

def coords_to_square(row: int, col: int) -> str:
    """Convert coordinates to algebraic notation"""
    file = chr(ord('a') + col)
    rank = str(row + 1)
    return file + rank
