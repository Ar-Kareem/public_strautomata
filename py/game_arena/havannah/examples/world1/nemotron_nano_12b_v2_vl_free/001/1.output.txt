To solve this problem, we need to create an AI that can play Havannah, a strategic board game, by selecting the best possible move based on a heuristic evaluation of potential moves. The goal is to maximize the score using a combination of proximity to the center, adjacency to friendly and opponent stones, and coverage of valid moves.

### Approach
1. **Understanding the Board and Moves**: The Havannah board is a hexagonal grid (15x15). Each cell has up to six neighbors, which are determined based on the hexagonal coordinate system. The valid moves are generated considering the current state of the board.
2. **Heuristic Evaluation**: The evaluation of each move is based on:
   - **Adjacency to Friendly Stones**: Cells adjacent to existing friendly stones are prioritized as they help in forming clusters.
   - **Proximity to the Center**: The center of the board is generally more valuable for strategic control.
   - **Blocking Opponent's Moves**: Cells adjacent to opponent's stones are penalized to disrupt their strategy.
3. **Scoring System**: Each move's score is calculated using a weighted sum of the adjacency factors and distance from the center. The weights are tuned to prioritize central control and friendly adjacency while minimizing opponent adjacency.

### Solution Code
```python
def policy(me, opp, valid_mask):
    import math

    # Convert to sets for O(1) lookups
    me_set = set(me)
    opp_set = set(opp)
    n = 15

    # Generate all valid moves
    valid_moves = []
    for r in range(n):
        for c in range(n):
            if valid_mask[r][c] and (r, c) not in me_set and (r, c) not in opp_set:
                valid_moves.append((r, c))

    if not valid_moves:
        raise RuntimeError("No valid moves")

    # Define neighbor directions for hexagonal grid
    dirs = [
        (-1, 0), (1, 0),   # Up and Down (same column)
        (0, -1), (0, 1),   # Left and Right (same row)
        (-1, -1), (-1, 1)  # Upper-left and Upper-right
    ]

    best_move = None
    max_score = -math.inf

    for (r, c) in valid_moves:
        adj_me = 0
        adj_opp = 0

        # Check all adjacent cells
        for dr, dc in dirs:
            nr = r + dr
            nc = c + dc
            if 0 <= nr < n and 0 <= nc < n:
                if (nr, nc) in me_set:
                    adj_me += 1
                elif (nr, nc) in opp_set:
                    adj_opp += 1

        # Calculate distance penalty to the center (7.5, 7.5)
        distance_penalty = (r - 7.5)**2 + (c - 7.5)**2

        # Heuristic weights
        score = (adj_me * 5) - (adj_opp * 3) - (distance_penalty * 0.1)

        # Update best move if current score is higher
        if score > max_score or (score == max_score and (r, c) == (7, 7)):
            max_score = score
            best_move = (r, c)

    return best_move
```

### Explanation
1. **Valid Moves Generation**: The code iterates through all cells on the board to find valid moves (cells marked as playable in `valid_mask` and not occupied by either player's stones).
2. **Neighbor Calculation**: For each valid move, the code checks all six possible neighboring cells using predefined directional vectors. This helps in determining the adjacency count.
3. **Heuristic Scoring**: The score for each move is calculated based on:
   - **Adjacent Friendly Stones**: Encourages building and expanding clusters.
   - **Adjacent Opponent Stones**: Penalizes moves adjacent to opponent's stones to block their potential structure.
   - **Distance to Center**: Discourages moves far from the center, promoting central control.
4. **Move Selection**: The move with the highest score is selected, with a tie-breaker favoring cells closer to the center in case of equal scores.

This heuristic approach efficiently balances strategic elements of Havannah, allowing the AI to make informed decisions without the computational expense of a full game tree search.
