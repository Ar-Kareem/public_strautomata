To solve this problem, we need to design a heuristic-based policy for playing Havannah, a strategic board game, on a 15x15 hexagonal grid. The goal is to create the next move that will help in forming winning structures such as bridges, rings, or forks while also blocking the opponent's potential winning moves.

### Approach
1. **Identify Valid Moves**: Determine all valid cells (from the `valid_mask`) where a stone can be placed next, considering that the cell must be empty and not occupied by either player.
2. **Score Heuristics**: Evaluate each valid move using a heuristic that considers:
   - **Adjacency to Friendly Stones**: The more adjacent friendly stones, the higher the score, encouraging the formation of connected clusters.
   - **Centrality**: Moves closer to the center of the board are given higher priority since they are strategic for forming bridges and forks.
   - **Opponent's Adjacency**: Penalize moves that are adjacent to opponent's stones, as these moves might be necessary to block potential threats from the opponent.
3. **Select the Best Move**: Choose the move with the highest score based on the computed heuristic.

### Solution Code

```python
def policy(me, opp, valid_mask):
    me_set = set(me)
    opp_set = set(opp)
    best_move = None
    max_score = -float('inf')
    
    # Generate all valid candidate moves
    valid_moves = []
    for row in range(15):
        for col in range(15):
            if valid_mask[row][col]:
                pos = (row, col)
                if pos not in me_set and pos not in opp_set:
                    valid_moves.append(pos)
    
    # If no moves left (unlikely per problem statement), return first valid cell
    if not valid_moves:
        return None  # Though problem states there is a move
    
    # Evaluate each valid move's score using heuristic
    for move in valid_moves:
        row, col = move
        
        # Calculate adjacent Me and Opp stones
        sum_me = 0
        sum_opp = 0
        dirs = [(-1, 0), (1, 0), (0, -1), (0, 1), (-1, -1), (-1, 1)]
        for dr, dc in dirs:
            nr, nc = row + dr, col + dc
            if 0 <= nr < 15 and 0 <= nc < 15:
                if valid_mask[nr][nc]:
                    if (nr, nc) in me_set:
                        sum_me += 1
                    elif (nr, nc) in opp_set:
                        sum_opp += 1
        
        # Heuristic scoring: prioritize adjacency to own stones and closeness to center
        distance_score = (row - 7) ** 2 + (col - 7) ** 2  # square of distance to center
        score = sum_me * 4 - distance_score * 5 - sum_opp * 2  # Weighted sum
        
        if score > max_score:
            max_score = score
            best_move = move
    
    return best_move
```

### Explanation
1. **Valid Moves Identification**: The code first identifies all valid cells where a stone can be placed by checking `valid_mask` and ensuring the cell is not occupied by either player.
2. **Scoring Mechanism**:
   - **Adjacent Friendly Stones (`sum_me`)**: Each move is scored based on how many of its adjacent cells (in the hexagonal grid) are occupied by the player's own stones. This encourages the formation of connected clusters, which can lead to forming bridges and forks.
   - **Centrality**: The distance from the move to the center of the board (7,7) is evaluated using squared distance for a Euclidean-like metric. Moves closer to the center are prioritized.
   - **Opponent's Stones (`sum_opp`)**: Penalize moves that are adjacent to opponent's stones to avoid creating opportunities for the opponent to form structures.
3. **Move Selection**: The move with the highest score is selected as the next move, balancing the need to cluster own stones, centralize, and block potential threats from the opponent.

This approach provides a balanced and computationally efficient strategy to maximize the player's chances of winning by focusing on central positioning, connectivity, and blocking potential threats.
