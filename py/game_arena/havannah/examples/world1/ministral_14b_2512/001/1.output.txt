### Strategy for the Havannah Policy

The strategy for this policy is a **hybrid approach** combining **pattern recognition, positional play, and threat response** to maximize winning chances. Here's the breakdown:

1. **Immediate Win Check**: First, check if placing a stone in any valid position results in an immediate win (ring, bridge, or fork). If so, prioritize that move.
2. **Block Opponent's Win**: If the opponent has a potential winning move (one stone away from completing a ring, bridge, or fork), block it immediately.
3. **Center Control**: Prioritize controlling the center of the board (especially the hexagonal core) to limit opponent mobility and create more opportunities for rings/bridges/forks.
4. **Edge and Corner Control**: Secure edges and corners to limit opponent's ability to form bridges or forks. Corners are especially valuable for bridge formation.
5. **Pattern-Based Play**: Use predefined patterns (e.g., potential rings, bridges, or forks) to guide stone placement. For example:
   - **Rings**: Look for 4-in-a-row or 5-in-a-row patterns that can form a loop.
   - **Bridges**: Prioritize connecting two corners or edges.
   - **Forks**: Aim to connect three edges (non-corner) to create a fork.
6. **Mobility and Flexibility**: Prefer moves that maximize future options (e.g., stones with more adjacent empty cells) to maintain flexibility.
7. **Randomness for Uncertainty**: If no clear strategic move is found, fall back to a semi-random move among high-value positions (e.g., center or near opponent's stones).

### Key Implementation Details:
- **Hexagonal Grid Handling**: Use the given adjacency rules to determine valid neighbors for each stone.
- **Winning Condition Checks**: Implement functions to detect rings, bridges, and forks after placing a stone.
- **Opponent Threat Detection**: Simulate opponent moves to identify potential winning threats.
- **Positional Scoring**: Assign scores to positions based on their strategic value (e.g., center > edges > corners).

from collections import defaultdict

def policy(me: list[tuple[int, int]], opp: list[tuple[int, int]], valid_mask) -> tuple[int, int]:

    board_size = len(valid_mask)
    me_set = set(me)
    opp_set = set(opp)
    valid_positions = np.argwhere(valid_mask)

    # Directions for hexagonal grid (6 neighbors)
    directions = [
        (0, 1),   # Right
        (1, 0),   # Down-right
        (1, -1),  # Down-left

        (-1, 1)   # Up-right

    # Check if a move wins immediately
    def is_win(move, player):
        row, col = move
        player_stones = me_set if player == 0 else opp_set
        # Check for ring (loop)
        if is_ring(move, player_stones):
            return True
        # Check for bridge (connects two corners)
        if is_bridge(move, player_stones):
            return True
        # Check for fork (connects three edges)
        if is_fork(move, player_stones):
            return True
        return False

    # Check if a move blocks opponent's win

        opp_stones = opp_set if player == 0 else me_set
        # Simulate opponent's move

        temp_stones.add((row, col))
        # Check if opponent would win with this move
        if is_win((row, col), 1 - player):
            return True
        return False

    # Check for ring (loop)
    def is_ring(move, stones):
        row, col = move
        visited = set()
        stack = [move]
        while stack:
            pos = stack.pop()
            if pos in visited:
                continue
            visited.add(pos)
            for neighbor in get_neighbors(pos):
                if neighbor in stones and neighbor not in visited:
                    stack.append(neighbor)
        # Check if the loop encloses any cell (simplified check)
        # For simplicity, assume a loop is formed if the move connects two existing stones in a cycle
        # This is a placeholder; a full implementation would require more complex checks
        return len(visited) >= 4  # At least 4 stones in a loop

    # Check for bridge (connects two corners)
    def is_bridge(move, stones):
        corners = [(0, 0), (0, board_size-1), (board_size-1, 0), (board_size-1, board_size-1)]
        row, col = move
        # Check if the move connects two corners
        for corner in corners:
            if corner in stones:
                # Check if the move connects to another corner
                for other_corner in corners:
                    if other_corner != corner and other_corner in stones:
                        # Check if there's a path between the two corners via the move
                        # This is a simplified check; a full implementation would require BFS
                        return True
        return False

    def is_fork(move, stones):

        for row in range(board_size):
            for col in range(board_size):
                if (row == 0 or row == board_size-1 or col == 0 or col == board_size-1) and \

            best_move = (row, col)

    if best_move is None:
        best_move = valid_positions[np.random.randint(0, len(valid_positions))]

    return best_move
