To address the problem of creating a smart policy for a 4x4 Tic Tac Toe game, we develop a heuristic-based approach that prioritizes immediate winning moves, blocks the opponent's winning moves, and then selects moves based on a scoring system that evaluates the strategic value of each cell.

---

### **Strategy Overview**

The strategy is divided into three key phases:

1. **Immediate Winning Move**: If placing a token in any empty cell results in a line of four for the current player, that move is chosen immediately.
2. **Blocking Opponent's Winning Move**: If the opponent has a move that would result in a win on their next turn, we block that move first.
3. **Heuristic-Based Strategy**: If no immediate win or block is available, we use a heuristic to choose the best move. The heuristic prioritizes:
   - **Line Count**: Cells that are part of more lines (row, column, and diagonals) are scored higher.
   - **Proximity to Center**: Cells closer to the center of the board are prioritized over those at the edges.
4. **Tie-Breaking**: If multiple cells have the same priority, the one closer to the center of the board is chosen.

---

### **Python Implementation**

```python
def check_win(board, player):
    # Check rows
    for row in board:
        if all(cell == player for cell in row):
            return True
    # Check columns
    for c in range(4):
        if all(board[r][c] == player for r in range(4)):
            return True
    # Check main diagonal
    if all(board[i][i] == player for i in range(4)):
        return True
    # Check anti-diagonal
    if all(board[i][3 - i] == player for i in range(4)):
        return True
    return False

def policy(board):
    # Step 1: Check for winning move
    for row in range(4):
        for col in range(4):
            if board[row][col] == 0:
                temp = [row.copy() for row in board]  # Make a deep copy of the board
                temp[row][col] = 1
                if check_win(temp, 1):
                    return (row, col)
                # Back to original board state for next checks
                temp[row][col] = 0

    # Step 2: Block opponent's winning move
    for row in range(4):
        for col in range(4):
            if board[row][col] == 0:
                temp = [row.copy() for row in board]  # Deep copy of the board
                temp[row][col] = -1
                if check_win(temp, -1):
                    return (row, col)
                # Back to original board state

    # Step 3: Heuristic-based move selection
    best_cell = None
    best_score = float('-inf')

    for row in range(4):
        for col in range(4):
            if board[row][col] == 0:
                # Simulate placing 1 at this cell
                new_board = [row.copy() for row in board]
                new_board[row][col] = 1

                # Calculate number of lines (row, column, or diagonals) that pass through this cell
                line_count = 2  # row and column
                if row == col or row + col == 3:
                    line_count += 1  # add 1 for each diagonal

                # Distance score to the center of the board (1.5, 1.5)
                distance = (row - 1.5) ** 2 + (col - 1.5) ** 2

                # Heuristic: prioritize more lines, closer to center
                # Use a weighted combination of line count and distance from center
                score = line_count * 1000 + distance * 10
                if score > best_score:
                    best_score = score
                    best_cell = (row, col)

    return best_cell
```

---

### **Strategy Explanation**

This policy ensures that the AI:

- **Wins immediately** if a move can complete a row, column, or diagonal of four 1's.
- **Blocks the opponent** from winning on their next turn.
- Chooses moves that:
  - Are part of more lines (row, column, and/or diagonals), increasing the chance of future wins.
  - Are closer to the center of the board, which is statistically more valuable in many game scenarios.

This approach is efficient and ensures the AI makes optimal moves within the time constraints of a game environment.
