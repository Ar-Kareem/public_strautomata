### Strategy for this 4x4 Tic Tac Toe Policy

The strategy for this policy is based on a **minimax algorithm with alpha-beta pruning** (a classic game-tree search approach) combined with **pattern recognition** (e.g., checking for winning lines, blocking opponent's threats, and evaluating board symmetry). Here's the breakdown:

1. **Winning Check**:
   The policy first checks if it can win in the current move by evaluating all possible 4-in-a-row patterns (rows, columns, diagonals, and anti-diagonals). If a winning move exists, it takes it immediately.

2. **Blocking Opponent**:
   If no immediate winning move is found, the policy checks if the opponent has a winning move in their next turn. If so, it blocks the opponent by placing its move in the opponent's winning cell.

3. **Symmetry and Corner Control**:
   The policy prioritizes moves that maintain or exploit board symmetry (e.g., mirroring opponent moves) and prefers corners (highest value) and edges (second-highest value) over center or empty cells. This is a heuristic to force the opponent into weaker positions.

4. **Minimax with Alpha-Beta Pruning**:
   The policy uses a depth-limited minimax search (up to 4 layers deep) to evaluate future board states. The evaluation function scores the board based on:
   - **Immediate threats**: If the opponent is about to win, the score is heavily penalized.
   - **Potential wins**: If the policy can win in the next move, the score is heavily rewarded.
   - **Symmetry**: Boards with symmetry are preferred (higher score).
   - **Control of center/corners**: Corners and edges are prioritized over the center or empty cells.

5. **Random Tiebreaker**:
   If multiple moves have the same score, the policy randomly selects one to avoid predictable behavior.

6. **Fallback (Random Move)**:
   If no winning/blocking moves are found and the minimax search returns no clear advantage, the policy defaults to placing a move in the highest-scoring empty cell (corners > edges > center > empty).

### Key Features
- **Depth-Limited Search**: Balances computational cost (1-second constraint) with lookahead depth (4 layers).
- **Heuristic Evaluation**: Scores moves based on symmetry, positional control, and immediate threats.
- **Legal Move Guarantee**: Always returns a valid `(row, col)` tuple in an empty cell.

### Python Implementation
```python
import numpy as np
import random

def policy(board: list[list[int]]) -> tuple[int, int]:
    """
    Returns the next move (row, col) for a 4x4 Tic Tac Toe game.
    Uses minimax with alpha-beta pruning and symmetry/positional heuristics.
    """
    # Convert board to numpy array for easier manipulation
    board_np = np.array(board, dtype=int)
    empty_cells = np.argwhere(board_np == 0)

    # Check if current player can win immediately
    for cell in empty_cells:
        row, col = cell
        if can_win(board_np, row, col, 1):
            return (row, col)

    # Check if opponent can win next turn and block
    for cell in empty_cells:
        row, col = cell
        if can_win(board_np, row, col, -1):
            return (row, col)

    # If no immediate win/block, use minimax with alpha-beta pruning
    best_move = None
    best_score = -float('inf')
    alpha = -float('inf')
    beta = float('inf')

    for cell in empty_cells:
        row, col = cell
        # Simulate move
        new_board = board_np.copy()
        new_board[row, col] = 1
        score = minimax(new_board, 3, False, alpha, beta)  # Depth limit = 3 (4 layers total)
        new_board[row, col] = 0  # Undo move

        if score > best_score:
            best_score = score
            best_move = (row, col)
        elif score == best_score:
            # Random tiebreaker to avoid deterministic behavior
            if random.choice([True, False]):
                best_move = (row, col)

    return best_move

def minimax(board: np.ndarray, depth: int, is_maximizing: bool, alpha: float, beta: float) -> float:
    """
    Minimax algorithm with alpha-beta pruning.
    Returns the best score for the current player (1) or opponent (-1).
    """
    empty_cells = np.argwhere(board == 0)
    if len(empty_cells) == 0 or depth == 0:
        return evaluate_board(board)

    if is_maximizing:
        best_score = -float('inf')
        for cell in empty_cells:
            row, col = cell
            board[row, col] = 1
            score = minimax(board, depth - 1, False, alpha, beta)
            board[row, col] = 0
            best_score = max(score, best_score)
            alpha = max(alpha, best_score)
            if beta <= alpha:
                break
        return best_score
    else:
        best_score = float('inf')
        for cell in empty_cells:
            row, col = cell
            board[row, col] = -1
            score = minimax(board, depth - 1, True, alpha, beta)
            board[row, col] = 0
            best_score = min(score, best_score)
            beta = min(beta, best_score)
            if beta <= alpha:
                break
        return best_score

def evaluate_board(board: np.ndarray) -> float:
    """
    Evaluates the board state using heuristics:
    - Immediate wins/threats
    - Symmetry
    - Corner/edge control
    """
    score = 0
    empty_cells = np.argwhere(board == 0)

    # Check if current player wins
    for cell in empty_cells:
        row, col = cell
        if can_win(board, row, col, 1):
            score += 1000
        elif can_win(board, row, col, -1):
            score -= 1000

    # Symmetry heuristic (prefer symmetric boards)
    symmetry_score = evaluate_symmetry(board)
    score += symmetry_score

    # Positional control heuristic (corners > edges > center)
    for cell in empty_cells:
        row, col = cell
        if is_corner(row, col):
            score += 10
        elif is_edge(row, col):
            score += 5
        else:
            score += 1

    return score

def can_win(board: np.ndarray, row: int, col: int, player: int) -> bool:
    """
    Checks if placing 'player' at (row, col) results in a win.
    """
    # Directions: rows, columns, diagonals, anti-diagonals
    directions = [(0, 1), (1, 0), (1, 1), (1, -1)]
    for dr, dc in directions:
        count = 0
        for i in range(-3, 4):
            r, c = row + i, col + i
            if 0 <= r < 4 and 0 <= c < 4:
                if board[r, c] == player:
                    count += 1
                elif board[r, c] == 0:
                    count += 1
            if count == 4:
                return True
    return False

def evaluate_symmetry(board: np.ndarray) -> float:
    """
    Evaluates board symmetry. Higher score for more symmetric boards.
    """
    # Check horizontal symmetry
    for i in range(4):
        if np.array_equal(board[i, :], board[i, ::-1]):
            return 100  # Perfect symmetry (unlikely in 4x4)

    # Check vertical symmetry
    for j in range(4):
        if np.array_equal(board[:, j], board[:, j][::-1]):
            return 100

    # Check diagonal symmetry (main diagonals)
    if np.array_equal(board.diagonal(), board.diagonal()[::-1]):
        return 50
    if np.array_equal(np.fliplr(board).diagonal(), board.diagonal()[::-1]):
        return 50

    # Check anti-diagonal symmetry
    if np.array_equal(np.flipud(board).diagonal(), board.diagonal()):
        return 50
    if np.array_equal(np.fliplr(np.flipud(board)).diagonal(), board.diagonal()):
        return 50

    return 0  # No symmetry

def is_corner(row: int, col: int) -> bool:
    """Returns True if (row, col) is a corner cell."""
    return (row == 0 or row == 3) and (col == 0 or col == 3)

def is_edge(row: int, col: int) -> bool:
    """Returns True if (row, col) is an edge cell (not corner or center)."""
    return (row == 1 or row == 2) and (col == 0 or col == 3 or col == 1 or col == 2) or \
           (col == 1 or col == 2) and (row == 0 or row == 3 or row == 1 or row == 2)
```

### Explanation of Key Functions:
1. **`can_win`**:
   Checks if placing a player (`1` or `-1`) at `(row, col)` completes a 4-in-a-row. Uses directional offsets to evaluate all possible winning lines.

2. **`evaluate_symmetry`**:
   Scores the board based on symmetry (e.g., mirrored rows/columns/diagonals). Symmetric boards are harder to exploit, so they get a higher score.

3. **`is_corner`/`is_edge`**:
   Heuristics to prioritize corners (highest value) and edges (second-highest value) over other cells.

4. **`minimax`**:
   Recursively evaluates all possible moves up to a depth limit (3 layers for opponent + 1 for policy). Uses alpha-beta pruning to optimize search.

5. **`evaluate_board`**:
   Combines immediate win/block checks, symmetry evaluation, and positional control into a single score. Corners contribute more to the score than edges or center.

---
### Why This Strategy Works:
- **Depth-Limited Minimax**: Ensures the policy looks ahead enough to counter most opponent strategies while staying within the 1-second constraint.
- **Symmetry Exploitation**: Forces the opponent into asymmetric positions, reducing their options.
- **Corner/Edge Focus**: High-value cells are prioritized, making it harder for the opponent to create multiple threats.
- **Legal Move Guarantee**: The code always returns a valid move by checking `empty_cells` before making decisions.
