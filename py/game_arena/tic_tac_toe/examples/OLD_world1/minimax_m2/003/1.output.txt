

To solve this problem, we need to determine the best move for a player in a 3x3 Tic Tac Toe game. The player is always represented by '1' in the board state, while the opponent is represented by '-1'. The board is a 3x3 grid where each cell can be '0' (empty), '1' (player's piece), or '-1' (opponent's piece). The goal is to select a move that maximizes the player's chances of winning, assuming the opponent also plays optimally.

### Approach
1. **Problem Analysis**: The Tic Tac Toe game state can be represented as a 3x3 grid. The solution involves evaluating all possible future game states from the current position to determine the optimal move for the player. This is a classic minimax problem where the player (maximizer) tries to win, and the opponent (minimizer) tries to prevent that.
2. **State Representation**: The board state is converted into a tuple of 9 integers (flattening the 3x3 grid into a 1D array for efficient hashing and state comparison.
3. **Terminal State Check**: Before generating future moves, we check if the current state is terminal (i.e., someone has won or the board is full). If so, no further moves are needed.
4. **Minimax Algorithm**: For non-terminal states, we use the minimax algorithm:
   - **Maximizing Player (Current Player)**: The current player (either '1' or '-1') selects the move that leads to the highest possible outcome (win for '1' is best, draw is next, loss is worst).
   - **Minimizing Player (Opponent)**: The opponent selects the move that leads to the lowest possible outcome from the player's perspective.
5. **Memoization**: To optimize the solution, we store previously computed state values in a dictionary (memo) to avoid redundant calculations, significantly reducing the computational complexity from exponential to polynomial time for the Tic Tac Toe state space.
6. **Best Move Selection**: For the current state, we generate all possible next states by placing the current player's piece in each empty cell. The best move is the one that leads to the highest value for the player, based on the minimax evaluation of the resulting state.

### Solution Code
```python
memo = {}

def state_value(state):
    if state in memo:
        return memo[state]
    
    for i in range(3):
        row_sum = state[3*i] + state[3*i+1] + state[3*i+2]
        if row_sum == 3:
            memo[state] = 1
            return 1
        if row_sum == -3:
            memo[state] = -1
            return -1
            
    for j in range(3):
        col_sum = state[j] + state[j+3] + state[j+6]
        if col_sum == 3:
            memo[state] = 1
            return 1
        if col_sum == -3:
            memo[state] = -1
            return -1
            
    diag1_sum = state[0] + state[4] + state[8]
    if diag1_sum == 3:
        memo[state] = 1
        return 1
    if diag1_sum == -3:
        memo[state] = -1
        return -1
        
    diag2_sum = state[2] + state[4] + state[6]
    if diag2_sum == 3:
        memo[state] = 1
        return 1
    if diag2_sum == -3:
        memo[state] = -1
        return -1
        
    if 0 not in state:
        memo[state] = 0
        return 0
        
    count1 = state.count(1)
    count_minus1 = state.count(-1)
    
    if count1 == count_minus1:
        current_player = 1
    else:
        current_player = -1
        
    next_states = []
    for i in range(9):
        if state[i] == 0:
            next_state = list(state)
            next_state[i] = current_player
            next_state = tuple(next_state)
            next_states.append(next_state)
            
    values = []
    for ns in next_states:
        values.append(state_value(ns))
        
    if current_player == 1:
        best = max(values)
    else:
        best = min(values)
        
    memo[state] = best
    return best

def policy(board: list[list[int]]) -> tuple[int, int]:
    flat_board = (board[0][0], board[0][1], board[0][2],
                  board[1][0], board[1][1], board[1][2],
                  board[2][0], board[2][1], board[2][2])
                  
    count1 = flat_board.count(1)
    count_minus1 = flat_board.count(-1)
    
    if count1 != count_minus1:
        for i in range(9):
            if flat_board[i] == 0:
                return (i // 3, i % 3)
        return (0, 0)
        
    terminal = False
    for i in range(3):
        row_sum = flat_board[3*i] + flat_board[3*i+1] + flat_board[3*i+2]
        if row_sum == 3 or row_sum == -3:
            terminal = True
            break
            
    if not terminal:
        for j in range(3):
            col_sum = flat_board[j] + flat_board[j+3] + flat_board[j+6]
            if col_sum == 3 or col_sum == -3:
                terminal = True
                break
                
    if not terminal:
        diag1_sum = flat_board[0] + flat_board[4] + flat_board[8]
        diag2_sum = flat_board[2] + flat_board[4] + flat_board[6]
        if diag1_sum == 3 or diag1_sum == -3 or diag2_sum == 3 or diag2_sum == -3:
            terminal = True
            
    if not terminal:
        if 0 not in flat_board:
            terminal = True
            
    if terminal:
        for i in range(9):
            if flat_board[i] == 0:
                return (i // 3, i % 3)
        return (0, 0)
        
    best_move = None
    best_value = -2
    
    for i in range(9):
        if flat_board[i] == 0:
            next_state = list(flat_board)
            next_state[i] = 1
            next_state = tuple(next_state)
            value_next = state_value(next_state)
            if value_next > best_value:
                best_value = value_next
                best_move = (i // 3, i % 3)
                
    return best_move if best_move is not None else (0, 0)
```

### Explanation
1. **Memoization Dictionary**: The `memo` dictionary stores the evaluated state values to avoid redundant computations, optimizing the solution.
2. **State Evaluation**: The `state_value` function checks if a state is terminal (win or full) and computes the value using the minimax algorithm for non-terminal states.
3. **Policy Function**: The `policy` function processes the current board state:
   - **Input Handling**: Converts the 3x3 board into a flat tuple for efficient processing.
   - **Turn Check**: Determines if it's the player's turn (counts of '1' and '-1' are equal).
   - **Terminal Check**: Quickly checks if the game has ended (win or full).
   - **Best Move Calculation**: For non-terminal states, generates all possible moves, evaluates each resulting state using minimax, and selects the move leading to the highest value.
4. **Return Value**: Returns the best move as a tuple `(row, column)` in the range 0-2.

This approach efficiently navigates the Tic Tac Toe game tree using minimax and memoization, ensuring optimal play within the problem constraints.
