**Minimax Algorithm Outline:**

*   **`minimax(board, depth, is_maximizing_player)`**:
    *   **Base Cases**:
        *   If the game is won by player `1` (me), return a high positive score (e.g., `10 - depth` to prefer faster wins).
        *   If the game is won by player `-1` (opponent), return a high negative score (e.g., `-10 + depth` to prefer slower losses, or just `-10`).
        *   If the game is a draw (no empty cells and no winner), return `0`.
    *   **Recursive Step**:
        *   If `is_maximizing_player` (it's my turn):
            *   Initialize `max_eval = -infinity`.
            *   Iterate through all empty cells on the board.
            *   For each empty cell:
                *   Make the move (place `1`).
                *   Call `minimax(new_board, depth + 1, False)`.
                *   `max_eval = max(max_eval, eval)`.
                *   Undo the move (reset cell to `0`).
            *   Return `max_eval`.
        *   If `not is_maximizing_player` (it's opponent's turn):
            *   Initialize `min_eval = +infinity`.
            *   Iterate through all empty cells.
            *   For each empty cell:
                *   Make the move (place `-1`).
                *   Call `minimax(new_board, depth + 1, True)`.
                *   `min_eval = min(min_eval, eval)`.
                *   Undo the move.
            *   Return `min_eval`.

*   **`policy(board)`**:
    *   Initialize `best_score = -infinity`.
    *   Initialize `best_move = (-1, -1)`.
    *   Iterate through all empty cells on the initial `board`.
    *   For each empty cell `(r, c)`:
        *   Make the move (place `1` at `(r, c)`).
        *   `score = minimax(new_board, 0, False)` (now it's opponent's turn).
        *   Undo the move.
        *   If `score > best_score`:
            *   `best_score = score`.
            *   `best_move = (r, c)`.
    *   Return `best_move`.

def minimax(board, depth, is_maximizing_player):

    if winner == 1:
        return 10

        return -10

        best_eval = float('inf')

            board[r][c] = -1 # Make the move
            eval = minimax(board, depth + 1, True) # Recurse for me
            board[r][c] = 0 # Undo the move
            best_eval = min(best_eval, eval)

**Edge case: What if there are no empty cells left for my move?**
The `policy` function itself iterates over empty cells on the *initial* board. If `get_empty_cells(board)` returns an empty list in `policy`, it means the game is already over (draw or someone won). In a competitive environment, `policy` should only be called if a move is possible. For robustness, I can add a check in `policy` itself.

         return (1, 1) # Prefer center if empty

        # This shouldn't happen if the game is still active, but as a safeguard.
        # If no empty cells, game is over, no move can be made.
        # Return an invalid move or raise an error. For this problem,
        # assuming it's always called when a move is possible.
        raise ValueError("No empty cells available on the board.")

    for r, c in empty_cells:
        # Create a deep copy of the board for mutable operations
        # or just modify and revert, which is faster.

        score = minimax(board, 0, False) # Now opponent's turn (player -1)
        board[r][c] = 0 # Undo the move

            best_score = score
            best_move = (r, c)

    # For Tic Tac Toe, this is usually fine.
    # An alternative is to randomly pick one of the best moves.
    return best_move

**Board Copying vs. In-place Modification and Reversion:**
Using `board[r][c] = 1` and then `board[r][c] = 0` (reverting) is generally faster than creating a deep copy of the board for each recursive call, especially for small boards. This is the common practice in minimax implementations.
