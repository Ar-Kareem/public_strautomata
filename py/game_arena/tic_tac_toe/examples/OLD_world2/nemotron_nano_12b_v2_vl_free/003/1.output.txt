To solve this problem, we need to create a function `policy` that determines the next optimal move for a 4x4 Tic Tac Toe game. The goal is to efficiently check for immediate winning moves, block the opponent's winning moves, and use a heuristic-based approach to select the best move when no immediate threats are present.

### Approach
1. **Immediate Win Check**: Check each empty cell to see if placing the current player's mark there would result in a win (four in a row, column, or diagonal).
2. **Block Opponent's Win**: Check if the opponent can win in their next move and block by placing the current player's mark in that cell.
3. **Heuristic-Based Move**: If no immediate wins or blocks are found, use a predefined priority order to select the best move. This order prioritizes the center cells, followed by corners, then edges, and finally the remaining cells.

### Solution Code
```python
def policy(board):
    def is_win(player, row, col):
        # Check row
        row_count = 0
        for c in range(4):
            if c == col:
                continue
            if board[row][c] == player:
                row_count += 1
        if row_count == 3:
            return True
        
        # Check column
        col_count = 0
        for r in range(4):
            if r == row:
                continue
            if board[r][col] == player:
                col_count += 1
        if col_count == 3:
            return True
        
        # Check main diagonal (row == col)
        if row == col:
            diag_count = 0
            for i in range(4):
                if (i, i) == (row, col):
                    continue
                if board[i][i] == player:
                    diag_count += 1
            if diag_count == 3:
                return True
        
        # Check anti-diagonal (row + col == 3)
        if row + col == 3:
            anti_count = 0
            for i in range(4):
                j = 3 - i
                if (i, j) == (row, col):
                    continue
                if board[i][j] == player:
                    anti_count += 1
            if anti_count == 3:
                return True
        
        return False
    
    # Step 1: Check if current player can win immediately
    for r in range(4):
        for c in range(4):
            if board[r][c] == 0 and is_win(1, r, c):
                return (r, c)
    
    # Step 2: Check if opponent can win and block it
    for r in range(4):
        for c in range(4):
            if board[r][c] == 0 and is_win(-1, r, c):
                return (r, c)
    
    # Step 3: Choose best move based on predefined priority order
    priority_order = [
        (1, 1), (1, 2), (2, 1), (2, 2),  # Center cells
        (0, 0), (0, 3), (3, 0), (3, 3),  # Corners
        (0, 1), (0, 2), (3, 1), (3, 2),  # Top and bottom edge centers
        (1, 0), (1, 3), (2, 0), (2, 3)   # Side edges
    ]
    
    # Find the first empty cell in the priority order
    for r, c in priority_order:
        if board[r][c] == 0:
            return (r, c)
    
    # Fallback (should not occur)
    return (-1, -1)
```

### Explanation
1. **Immediate Win Check**: The function `is_win` checks if placing the player's mark at a specific cell would result in a win. This is done by checking if the row, column, or either diagonal (if applicable) has three of the player's marks already, which would form a winning line when the current cell is filled.
2. **Blocking Opponent**: Similarly, the code checks each empty cell to see if placing the opponent's symbol there would result in a win. If such a cell is found, it is blocked by placing the current player's mark there.
3. **Heuristic Moves**: The predefined priority order ensures that the AI selects the most strategic cells first, starting from the center, then corners, edges, and other positions. This heuristic approach is efficient and ensures the AI makes strategic moves based on typical Tic Tac Toe strategies.

This approach ensures that the AI responds optimally in critical situations (immediate wins or blocks) and uses a structured heuristic to make the best possible move when no immediate threats exist, all within the given time constraints.
