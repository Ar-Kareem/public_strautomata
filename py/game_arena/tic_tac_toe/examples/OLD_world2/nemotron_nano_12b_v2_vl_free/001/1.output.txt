To solve this problem, we need to create a program that determines the optimal next move for a player in a 4x4 Tic Tac Toe game. The game involves two players taking turns to place their marks on a grid, with the goal of getting four of their marks in a row, either horizontally, vertically, or diagonally. The solution should find the best possible move from the current state of the board, prioritizing winning moves, blocking the opponent's potential win, and using a heuristic to choose the best move when no immediate threats exist.

### Approach
1. **Immediate Win Check**: The first priority is to check if the current player can place a mark in any empty cell to immediately win the game. The cells are checked in a predefined priority order to ensure the best possible move is selected if multiple winning moves exist.
2. **Blocking Opponent's Win**: If there is no immediate winning move, the next priority is to check for cells where the opponent can win in their next move. The AI blocks these cells by placing its mark there, preventing the opponent from winning.
3. **Heuristic-based Move**: If no immediate win or block is necessary, the AI selects the next move based on a heuristic that prioritizes cells in the center, followed by corners, and then edges. This heuristic is based on the assumption that central and corner cells provide better opportunities to form winning lines.

### Solution Code
```python
def policy(board: list[list[int]]) -> tuple[int, int]:
    def is_win(p, i, j):
        # Check row
        if all(board[i][c] == p for c in range(4)):
            return True
        # Check column
        if all(board[r][j] == p for r in range(4)):
            return True
        # Check main diagonal
        if i == j and all(board[x][x] == p for x in range(4)):
            return True
        # Check anti-diagonal
        if i + j == 3 and all(board[x][3 - x] == p for x in range(4)):
            return True
        return False

    # Priority order: center, corners, edges
    priority = [
        (1, 1), (1, 2), (2, 1), (2, 2),
        (0, 0), (0, 3), (3, 0), (3, 3),
        (0, 1), (0, 2), (1, 0), (1, 3),
        (2, 0), (2, 3), (3, 1), (3, 2)
    ]

    # Step 1: Check for immediate win
    for move in priority:
        i, j = move
        if board[i][j] == 0:
            if is_win(1, i, j):
                return (i, j)

    # Step 2: Check to block opponent's win
    for move in priority:
        i, j = move
        if board[i][j] == 0:
            if is_win(-1, i, j):
                return (i, j)

    # Step 3: Take highest priority available cell
    for move in priority:
        i, j = move
        if board[i][j] == 0:
            return (i, j)

    # Fallback (should not be reached as there's always a move available)
    return (0, 0)
```

### Explanation
1. **Immediate Win Check**: The code first iterates through all cells in the priority order. For each empty cell, it simulates placing the AI's mark and checks if this results in a win. If a winning move is found, it is returned immediately.
2. **Blocking Opponent's Win**: If no winning move is found, the code checks each empty cell to see if placing the opponent's mark in any of these cells would result in a win for the opponent. If such a cell is found, it is returned as the AI's move to block the opponent.
3. **Heuristic-based Move**: If there are no immediate wins or blocks, the code selects the next move by iterating through a predefined priority list of cells. This list prioritizes central cells, followed by corners, and then edges, ensuring the AI places its mark in positions that statistically offer the best chances of winning in subsequent moves.

This approach ensures that the AI efficiently finds the optimal move by combining immediate threat checks with a strategic heuristic, balancing both aggressive and defensive play.
