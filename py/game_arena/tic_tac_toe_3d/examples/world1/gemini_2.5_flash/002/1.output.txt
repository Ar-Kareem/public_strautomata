2.  **`policy(board)` Function Logic:**
    *   **Winning Move:** Iterate through all empty cells. For each empty cell, place my marker (`1`), then check if I win. If so, return that move immediately.
    *   **Blocking Move:** If no winning move for me, iterate through all empty cells. For each empty cell, place the opponent's marker (`-1`), then check if the opponent wins. If so, return that move to block them.
    *   **Strategic Move (Center, Corners, Edges):**

        *   **Corners:** After centers, corners are usually good: `(0, 0, 0)`, `(0, 0, 2)`, `(0, 2, 0)`, etc. There are 8 corners.
        *   **Edges:** Finally, other edge positions.
    *   **Default Move:** If no strategic move is found (shouldn't happen with the center/corners/edges approach unless the board is completely full of those), just return the first available empty cell.

4.  **Refined `policy` Flow:**

    *   Precompute `WIN_LINES` once (global constant or private helper).
    *   Find all `empty_cells`.

    *   **Opponent Blocking Move:**

    *   **Fall-back:** If for some reason none of the above (e.g., if strategic positions are already taken by me or opponent), return the first available empty cell.

    Determines the next optimal move for a 3x3x3 3D Tic Tac Toe game.
    Priorities: Win -> Block -> Strategic (Center -> Face Centers -> Corners) -> Any Empty
    """

    # 1. Check for a winning move
    for x, y, z in empty_cells:
        board[x][y][z] = 1  # Temporarily make the move
        if check_win(board, 1):
            board[x][y][z] = 0  # Undo the move
            return (x, y, z)

    # but not leading to a win/block immediately.
    if empty_cells:
        return empty_cells[0]

    # This case should ideally not be reached if the game is still active
    # and a move is required.
    raise Exception("No available moves found. Board might be full or game over.")
