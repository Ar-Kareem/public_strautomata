To solve this problem, we need to create an AI agent that can play Breakthrough, a board game where the objective is to either capture all the opponent's pieces or move any of your pieces to the opponent's home row. The solution must be efficient and capable of making optimal moves within a short time frame.

### Approach
The approach involves generating all possible legal moves for the current player, evaluating each move based on a heuristic to determine the best possible move. The key steps are:

1. **Generate Legal Moves**: For each piece, check all possible straight forward moves and diagonal captures.
2. **Check for Winning Moves**: Prioritize any moves that allow a piece to reach the opponent's home row, as these are immediate wins.
3. **Evaluate Captures**: If no winning move is found, prioritize moves that capture the opponent's pieces.
4. **Move Evaluation**: For non-capturing moves, determine the move that advances the piece closest to the opponent's home row.

### Solution Code
```python
def policy(me, opp, color):
    me_set = set(me)
    opp_set = set(opp)
    direction = 1 if color == 'w' else -1
    target_row = 0 if color == 'b' else 7  # Opponent's home row
    all_moves = []

    # Generate all possible moves
    for (r, c) in me:
        # Check straight move
        new_r = r + direction
        new_c = c
        if 0 <= new_r < 8:
            new_pos = (new_r, new_c)
            if new_pos not in me_set and new_pos not in opp_set:
                all_moves.append(((r, c), new_pos))
        
        # Check diagonal captures
        for delta_col in (-1, 1):
            new_r = r + direction
            new_c = c + delta_col
            if 0 <= new_r < 8 and 0 <= new_c < 8:
                new_pos = (new_r, new_c)
                if new_pos in opp_set:
                    all_moves.append(((r, c), new_pos))
    
    # Check for any move that reaches the target row immediately
    for move in all_moves:
        _, to_pos = move
        if to_pos[0] == target_row:
            return move
    
    # Check for valid capture moves
    opp_set = set(opp)  # Reconstruct to avoid set modification issues
    captures = [m for m in all_moves if m[1] in opp_set]
    if captures:
        best_capture = None
        best_priority = float('-inf')
        for capture in captures:
            _, (new_r, _) = capture
            if color == 'b':
                priority = -new_r
            else:
                priority = new_r
            if priority > best_priority:
                best_priority = priority
                best_capture = capture
        return best_capture
    
    # No captures, select best move towards target
    best_move = None
    best_val = float('-inf')
    for move in all_moves:
        _, (new_r, _) = move
        if color == 'b':
            value = -new_r
        else:
            value = new_r
        if value > best_val:
            best_val = value
            best_move = move
    return best_move
```

### Explanation
1. **Generating Legal Moves**: The code first generates all possible moves for each piece. This includes both straight forward moves and diagonal captures.
2. **Winning Moves Check**: The code checks if any move results in reaching the opponent's home row (immediate win). If such a move exists, it is returned immediately.
3. **Evaluating Captures**: If no winning moves are found, the code evaluates all possible capture moves. The best capture is determined based on how close it places the piece to the target row.
4. **Non-Capturing Moves**: If no captures are possible, the code selects the move that brings the piece closest to the target row, which helps in progressing towards the winning condition.

This approach ensures that the AI prioritizes winning moves, captures when possible, and otherwise moves towards the opponent's home row strategically. The use of sets for quick lookup and efficient move generation ensures the solution runs within the required time constraints.
