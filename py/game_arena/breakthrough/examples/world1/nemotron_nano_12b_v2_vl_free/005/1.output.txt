To solve the problem of finding the best move for a Breakthrough game on an 8x8 board, we need to implement a strategy that prioritizes the following:

1. **Promotion moves** (i.e., moving a piece to the opponent's home row to win immediately).
2. **Capturing enemy pieces**, especially those that are closest to their own home row, as these reduce the opponent's ability to advance.
3. **Maximizing the advancement of our own pieces** toward the target row.

The key insight is that breaking ties between non-capture moves is ineffective since all moves contribute equally to advancing toward the target row. Therefore, we prioritize capture moves first and break ties based on the **enemy pawn's distance to their home row**.

---

### Strategy Overview

1. **Promotion Moves**: If any move leads to a piece reaching the opponent's home row, that move is selected immediately.
2. **Capture Moves**:
   - For black (`'b'`), prioritize capturing white pawns that are closest to their home row (row 7).
   - For white (`'w'`), prioritize capturing black pawns that are closest to their home row (row 0).
3. **Non-Capture Moves**: If no captures are possible, choose any valid move that advances toward the target row.

The score for a move is calculated as:
- **1000** for a capture (to prioritize over non-captures),
- **+ enemy_distance**, where `enemy_distance` is the distance of the captured piece to its own home row.

---

### Python Implementation

```python
def policy(me, opp, color):
    target_home = 0 if color == 'b' else 7
    moves = []

    # Generate all possible moves
    for (r, c) in me:
        row_incr = -1 if color == 'b' else 1
        new_r = r + row_incr

        if 0 <= new_r < 8:
            # Straight move
            new_pos_straight = (new_r, c)
            if new_pos_straight not in me:
                is_capture = False
                if new_pos_straight in opp:
                    is_capture = True
                moves.append(((r, c), new_pos_straight, is_capture))

            # Diagonal left
            new_col = c - 1
            if 0 <= new_col < 8:
                new_pos_diag = (new_r, new_col)
                if new_pos_diag not in me and new_pos_diag in opp:
                    moves.append(((r, c), new_pos_diag, True))

                # Diagonal right
                new_col = c + 1
                new_pos_diag = (new_r, new_col)
                if new_pos_diag not in me and new_pos_diag in opp:
                    moves.append(((r, c), new_pos_diag, True))

    # Step 1: Check for promotion moves
    for move in moves:
        _, new_pos, _ = move
        if new_pos[0] == target_home:
            return (new_pos, new_pos)

    # Step 2: Prioritize capture moves
    capture_moves = [m for m in moves if m[2]]

    if capture_moves:
        best_score = -1
        best_move = None
        for move in capture_moves:
            from_pos, to_pos, _ = move
            captured_row = to_pos[0]
            if color == 'b':
                # Captured piece is white, enemy home is 7
                enemy_distance = 7 - captured_row
            else:
                # Captured piece is black, enemy home is 0
                enemy_distance = captured_row
            score = 1000 + enemy_distance
            if score > best_score:
                best_score = score
                best_move = move
        return (best_move[0], best_move[1])

    # Step 3: If no capture, return any valid move
    # Since all moves are valid and move forward, return the first one
    return moves[0][0], moves[0][1] if moves else None
```

---

### Explanation

- **Move Generation**: The function generates all valid moves for the current player's pieces, checking both straight and diagonal moves.
- **Promotion Check**: If any move leads to reaching the opponent's home row, it is immediately returned as the best move.
- **Capture Scoring**:
  - For each capture move, the captured pawn's distance to its own home row is computed.
  - Positive scores are used to prioritize captures of pieces that are closer to the opponent's home row.
- **Non-Capture Moves**: If no captures are possible, any valid move is selected (since all non-capture moves contribute equally to progress toward the target).

This approach ensures the AI makes a legal move within the time constraint and prioritizes moves that reduce the opponent's ability to win while advancing toward its own target.
