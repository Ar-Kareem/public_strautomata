To solve this problem, we need to create a policy function for the game of Breakthrough that selects the optimal next move for the current player. The policy must return a legal move based on a state evaluation that considers the positions of both players' pieces. The solution involves using a minimax algorithm with alpha-beta pruning to a fixed depth, combined with a heuristic evaluation function that prioritizes pieces closer to promotion.

### Approach
1. **Problem Analysis**: 
   - The game is played on an 8x8 board where each player starts with pieces in their home rows (black at row 7, white at row 0).
   - Black moves downward towards row 0, while white moves upward towards row 7.
   - A player wins by either capturing all opponent pieces or getting any piece to the opponent's home row.
   - The policy must return a legal move (current position to a new position, capturing an opponent piece if possible).

2. **Key Insights**:
   - **Heuristic Evaluation**: Pieces closer to the opponent's home row in their movement direction are valued higher. We use predefined tables to assign values based on the piece's position relative to their goal.
   - **Move Generation**: For each piece, generate all possible forward and diagonal captures within bounds, ensuring no move to a square occupied by a friendly piece.
   - **Terminal State Check**: Before evaluation, check if the current player has won by reaching the opponent's home row or captured all opponent pieces.
   - **Recursive Search**: Use negamax with alpha-beta pruning to evaluate moves up to depth 2. The search alternates between maximizing and minimizing players, evaluating terminal states and using the heuristic for non-terminal states.

3. **Algorithm Selection**:
   - **Negamax with Alpha-Beta Pruning**: This algorithm simplifies the minimax by using a single function for both players by negating scores. Alpha-beta pruning cuts off branches that won't improve the current best score.
   - **Depth Limitation**: Due to computational constraints, the search depth is limited to 2 (one move of the opponent and one move of the player).
   - **Heuristic Evaluation**: Combines the positional strength of pieces (based on distance to goal and central control) with material advantage.

### Solution Code
```python
import sys
sys.setrecursionlimit(10000)

black_piece_value = [
    [7, 10, 12, 12, 12, 12, 10, 7],
    [6, 9, 11, 11, 11, 11, 9, 6],
    [5, 8, 10, 10, 10, 10, 8, 5],
    [4, 7, 9, 9, 9, 9, 7, 4],
    [3, 6, 8, 8, 8, 8, 6, 3],
    [2, 5, 7, 7, 7, 7, 5, 2],
    [1, 4, 6, 6, 6, 6, 4, 1],
    [0, 3, 5, 5, 5, 5, 3, 0]
]

white_piece_value = [
    [0, 3, 5, 5, 5, 5, 3, 0],
    [1, 4, 6, 6, 6, 6, 4, 1],
    [2, 5, 7, 7, 7, 7, 5, 2],
    [3, 6, 8, 8, 8, 8, 6, 3],
    [4, 7, 9, 9, 9, 9, 7, 4],
    [5, 8, 10, 10, 10, 10, 8, 5],
    [6, 9, 11, 11, 11, 11, 9, 6],
    [7, 10, 12, 12, 12, 12, 10, 7]
]

def make_move(me, opp, move, color):
    from_pos, to_pos = move
    new_me = []
    moved = False
    for piece in me:
        if piece == from_pos:
            new_me.append(to_pos)
            moved = True
        else:
            new_me.append(piece)
    if not moved:
        new_me = me[:]
    new_opp = [p for p in opp if p != to_pos]
    return new_me, new_opp

def generate_moves(me, opp, color):
    me_set = set(me)
    opp_set = set(opp)
    moves = []
    for piece in me:
        r, c = piece
        if color == 'b':
            directions = [(-1, 0), (-1, 1), (-1, -1)]
        else:
            directions = [(1, 0), (1, 1), (1, -1)]
        for dr, dc in directions:
            nr = r + dr
            nc = c + dc
            if 0 <= nr < 8 and 0 <= nc < 8:
                new_pos = (nr, nc)
                if new_pos in me_set:
                    continue
                if new_pos in opp_set:
                    moves.append((piece, new_pos))
                else:
                    moves.append((piece, new_pos))
    return moves

def evaluate(me, opp, color):
    if color == 'b':
        if any(r == 0 for (r, c) in me):
            return 1000000
        if not opp:
            return 1000000
        if not me:
            return -1000000
        our_val = sum(black_piece_value[r][c] for (r, c) in me)
        opp_val = sum(white_piece_value[r][c] for (r, c) in opp)
        return our_val - opp_val
    else:
        if any(r == 7 for (r, c) in me):
            return 1000000
        if not opp:
            return 1000000
        if not me:
            return -1000000
        our_val = sum(white_piece_value[r][c] for (r, c) in me)
        opp_val = sum(black_piece_value[r][c] for (r, c) in opp)
        return our_val - opp_val

def negamax(me, opp, color, depth, max_depth, alpha, beta):
    if color == 'b':
        if any(r == 0 for (r, c) in me):
            return 1000000, None
        if not opp:
            return 1000000, None
        if not me:
            return -1000000, None
    else:
        if any(r == 7 for (r, c) in me):
            return 1000000, None
        if not opp:
            return 1000000, None
        if not me:
            return -1000000, None

    if depth == max_depth:
        eval_val = evaluate(me, opp, color)
        return eval_val, None

    moves = generate_moves(me, opp, color)
    if not moves:
        return evaluate(me, opp, color), None

    best_score = -10**9
    best_move = None
    for move in moves:
        new_me, new_opp = make_move(me, opp, move, color)
        next_color = 'w' if color == 'b' else 'b'
        score_inner, _ = negamax(new_me, new_opp, next_color, depth+1, max_depth, -beta, -alpha)
        if score_inner > best_score:
            best_score = score_inner
            best_move = move
        alpha = max(alpha, score_inner)
        if beta <= alpha:
            break
            
    return best_score, best_move

def policy(me, opp, color):
    _, best_move = negamax(me, opp, color, 0, 2, -10**9, 10**9)
    if best_move is None:
        moves = generate_moves(me, opp, color)
        if moves:
            return moves[0]
        else:
            return (me[0][0], me[0][1], me[0][0], me[0][1])
    return best_move
```

### Explanation
1. **Evaluation Function**:
   - Uses predefined tables to assign values to pieces based on their positions. Black pieces gain value as they move toward row 0, while white pieces gain value moving toward row 7.
   - The evaluation is `our_value - opponent_value`, where `our_value` is the sum of values of the current player's pieces, and `opponent_value` is the sum for the opponent's pieces.

2. **Move Generation**:
   - For each piece, checks all possible forward and diagonal moves. Captures are allowed if an opponent piece is in the target square.
   - Ensures moves stay within the 8x8 board and avoid moving to squares occupied by friendly pieces.

3. **Terminal State Handling**:
   - Before evaluation, checks if the current player has won (a piece reached the opponent's home row or all opponent pieces are captured) or lost (no legal moves remaining).

4. **Negamax with Alpha-Beta Pruning**:
   - Recursively evaluates moves to a fixed depth (2 plies). The negamax function returns the best score and the corresponding move.
   - Alpha-beta pruning optimizes the search by cutting off branches that won't improve the current best score.

5. **Policy Function**:
   - Calls the negamax function to find the best move. If no move is found (shouldn't occur in valid states), it returns the first possible move to ensure a legal move is always returned. This approach efficiently balances between exploring the game tree and leveraging heuristic evaluations to make informed decisions within the time constraint.
