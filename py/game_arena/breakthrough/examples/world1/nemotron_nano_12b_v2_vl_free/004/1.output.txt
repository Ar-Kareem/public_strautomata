To solve this problem, we need to create a Python function that determines the next move for a game of Breakthrough. The function should prioritize winning moves, followed by capturing moves, and then moves that advance the player's pieces towards promotion. The solution must be efficient and return a legal move within the given constraints.

### Approach
1. **Generate Legal Moves**: For each piece of the current player, generate all possible legal moves. This includes moving one space straight forward and capturing diagonally forward if the target square contains an opponent's piece.
2. **Check for Winning Moves**: Immediately return any move that results in promoting a piece to the opponent's home row, as this ends the game instantly.
3. **Prioritize Captures**: If there are no winning moves, prioritize moves that capture an opponent's piece. Captures are sorted based on the row of the captured piece to prioritize those closest to the opponent's home row.
4. **Advance Towards Promotion**: If there are no captures, select the move that advances the player's piece closest to their promotion row. This is done by sorting the moves based on the target row's proximity to the promotion row.

### Solution Code
```python
def policy(me: list[tuple[int, int]], opp: list[tuple[int, int]], color: str) -> tuple[tuple[int, int], tuple[int, int]]:
    moves = []
    for (r, c) in me:
        if color == 'w':
            new_r = r + 1
        else:
            new_r = r - 1
        
        if 0 <= new_r < 8:
            target = (new_r, c)
            if target not in me and target not in opp:
                moves.append(((r, c), target))
        
        for dc in [-1, 1]:
            if color == 'w':
                new_r_diag = r + 1
            else:
                new_r_diag = r - 1
            new_c = c + dc
            if 0 <= new_r_diag < 8 and 0 <= new_c < 8:
                target = (new_r_diag, new_c)
                if target in opp:
                    moves.append(((r, c), target))
    
    promote_row = 7 if color == 'w' else 0
    
    for move in moves:
        _, to_pos = move
        if to_pos[0] == promote_row:
            return move
    
    capture_moves = [m for m in moves if m[1] in opp]
    non_capture_moves = [m for m in moves if m[1] not in opp]
    
    if capture_moves:
        if color == 'w':
            sorted_captures = sorted(capture_moves, key=lambda m: m[1][0], reverse=True)
        else:
            sorted_captures = sorted(capture_moves, key=lambda m: m[1][0])
        return sorted_captures[0]
    
    if color == 'w':
        non_capture_moves.sort(key=lambda m: m[1][0], reverse=True)
    else:
        non_capture_moves.sort(key=lambda m: m[1][0])
    
    return non_capture_moves[0] if non_capture_moves else moves[0]
```

### Explanation
1. **Generate Legal Moves**: The function iterates over each piece controlled by the current player and generates all possible straight and diagonal moves. Captures are identified by checking if the target square contains an opponent's piece.
2. **Check for Winning Moves**: The function checks if any move results in promotion (reaching the opponent's home row). If such a move is found, it is returned immediately.
3. **Prioritize Captures**: If no winning moves are found, the function separates moves into captures and non-captures. Captures are sorted based on the row of the captured piece, with higher priority given to pieces closer to the opponent's home row.
4. **Advance Towards Promotion**: If no captures are available, the function sorts non-capture moves based on how close they bring the piece to the promotion row. Moves are selected to advance the player's pieces as close as possible to their win condition.

This approach efficiently balances between capturing enemy pieces, advancing towards victory, and making strategic moves to increase the chances of winning.
