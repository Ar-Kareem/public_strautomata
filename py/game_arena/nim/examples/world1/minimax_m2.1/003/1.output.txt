# Nim Game Strategy: Misère Nim with Perfect Play

I will implement a smart policy for the game of Nim with 4 piles using the Misère play convention (the player who takes the last object loses). This requires a sophisticated strategy that adapts based on the game state.

## Strategy Overview

The optimal strategy for Misère Nim is well-known in combinatorial game theory:

1. **Endgame Detection**: When all piles have size 1, the strategy is to control parity - leave an even number of piles for the opponent so they are forced to take the last object.

2. **Midgame Play**: When not all piles are size 1, play like standard (normal) Nim - use XOR (nim-sum) calculations to maintain a winning position.

3. **XOR Strategy**: The goal is to make the XOR of all pile sizes equal to 0 after your move. When in a winning position (non-zero XOR), there exists exactly one move that creates a zero XOR. When in a losing position (zero XOR), any move you make will create a non-zero XOR for the opponent.

## Code Implementation

```python
def policy(piles: list[int]) -> str:
    """
    Determine the optimal move in a 4-pile Misère Nim game.
    
    Misère Nim Strategy:
    1. If all piles are size 1: Control parity (leave even number of piles for opponent)
    2. Otherwise: Use XOR strategy to maintain winning position
    
    Args:
        piles: List of 4 integers representing pile sizes
        
    Returns:
        String in format 'pile,take' representing the optimal move
    """
    
    # Check if game is over (all piles zero)
    if all(p == 0 for p in piles):
        return "0,0"  # No moves available
    
    # Count piles with size 1
    count_ones = sum(1 for p in piles if p == 1)
    
    # Strategy 1: All piles are size 1 (endgame)
    if count_ones == 4:
        # Take 1 from a pile to leave odd number (3) for opponent
        # They will be forced to leave even, and eventually take last
        for i, p in enumerate(piles):
            if p > 0:
                return f"{i},1"
    
    # Strategy 2: Mixed pile sizes (midgame) - use XOR strategy
    # Calculate XOR of all piles
    xor_sum = 0
    for p in piles:
        xor_sum ^= p
    
    # If XOR is 0, we're in losing position - make any legal move
    if xor_sum == 0:
        for i, p in enumerate(piles):
            if p > 0:
                # Take all from a pile with size > 1 if possible
                if p > 1:
                    return f"{i},{p}"
                # Otherwise take 1 from a size-1 pile
                return f"{i},1"
    
    # If XOR is non-zero, we can force a win - find winning move
    # Find a pile where p > (p XOR xor_sum) to make the move
    for i, p in enumerate(piles):
        target = p ^ xor_sum  # The value this pile should become
        if target < p:
            take = p - target
            if 1 <= take <= p:
                return f"{i},{take}"
    
    # Fallback (should never reach here if logic is correct)
    for i, p in enumerate(piles):
        if p > 0:
            return f"{i},1"
```
