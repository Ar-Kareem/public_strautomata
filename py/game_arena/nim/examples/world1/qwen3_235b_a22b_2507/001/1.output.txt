The logic for this policy is based on the theory of the misère version of the game of Nim, where the player to take the last object loses. This variant is known as "misère Nim." In standard Nim (normal play), the winning strategy is to play to make the bitwise XOR (nim-sum) of the pile sizes zero. However, in misère Nim, the strategy is slightly different, particularly when the game approaches the end.

The key insights for misère Nim with multiple piles are:
1. If all piles are of size 1, then the outcome depends on the number of piles: if there is an odd number of non-empty piles, the current player will lose if both play optimally.
2. If there is at least one pile with more than 1 object, then the strategy is almost the same as in normal Nim — we aim to make the XOR of the piles zero, **except** when doing so would leave only piles of size 1. In that case, we must deviate slightly to avoid leaving an odd number of 1s.

The optimal strategy can be summarized as:
- If all piles are 0 or 1, and there are an odd number of 1s, we are in a losing position — but we still have to move.
- Otherwise, compute the XOR of all piles.
  - If the XOR is zero, then we are in a losing position; we must make any legal move, preferably reducing a pile with more than 1 to try to force the opponent into a trap.
  - If the XOR is non-zero, we aim to make a move that sets the XOR to zero, **unless** doing so would leave only 1s and an odd number of them. In that case, we instead make a move that leaves an even number of 1s.

    big = 0
    for x in piles:
        if x > 1:
            big += 1

    # Case 1: no pile has more than 1
    if big == 0:
        # All non-zero piles are of size 1. Take one from the first non-empty pile.
        for i in range(4):

    elif big == 1:

                ones += 1

            take = piles[big_index]

            # Leave an odd number of 1s by reducing the big pile to 1
            take = piles[big_index] - 1
