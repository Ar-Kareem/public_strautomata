The strategy for this policy is based on the theory of **misère Nim**, where the player who takes the last object **loses** (as opposed to normal play). For a 4-pile Nim game under misère rules, we can adapt the standard **Nim-sum (XOR)** strategy with a special adjustment when the game reaches certain endgame states.

### Key Insights:
1. In normal Nim (last move wins), the winning strategy is to make the XOR (nim-sum) of all pile sizes zero.
2. In **misère Nim**, the same strategy works **except when the game is about to enter endgame** (i.e., when all piles are 0 or 1 and there’s an odd number of non-empty piles).
3. For 4-pile misère Nim with no restriction on moves, the optimal strategy is:
   - Play **normal XOR strategy** unless **all piles are ≤ 1**.
   - When all piles are ≤ 1, you must leave an **odd number** of 1s (because taking the last object loses).
   - However, if not all piles are ≤ 1, the optimal play is the same as normal Nim: make the XOR of all piles zero.
4. But special care must be taken when **only one pile has more than 1** — in misère play, we often want to **avoid** reducing the last large pile to 1 if it forces a losing position.

Our policy will:
- Compute the nim-sum (XOR of all pile values).
- If there is a pile with **more than 1 object**, and **NOT all piles are ≤1**, use **normal play**: make a move that results in nim-sum = 0.
- Else (when all piles are 0 or 1), make a move that leaves an **odd number** of 1s (to force opponent to take last).
- If we are in a state where all but one pile are ≤1, we apply special misère adjustment: sometimes reduce the large pile to 1 **only if necessary**, otherwise reduce to 0 if safe.

This approach ensures optimal play in misère 4-pile Nim.

def policy(piles: List[int]) -> str:

    big_piles = [i for i in range(4) if piles[i] > 1]
    ones = sum(1 for p in piles if p == 1)
    zeros = sum(1 for p in piles if p == 0)

        non_zero = 4 - zeros

        for i in range(4):

                if (non_zero - 1) % 2 == 1:
                    return f"{i},1"

            if piles[i] == 1:
                return f"{i},1"

    if xor_sum != 0:

        for i in range(4):

            target = piles[i] ^ xor_sum

                return f"{i},{take}"

    for i in range(4):
