To solve this problem, we need to implement a policy for a 4-pile Nim game where the player who takes the last object loses. This variant of Nim is known as misère Nim. The strategy involves analyzing the game state to determine the optimal move that forces the opponent into a losing position.

### Approach
1. **Check All Piles Are 1**: If all piles have exactly 1 object, the outcome depends on the number of piles. If the count of piles is even, the current player can force a win by reducing the number of piles to an odd number. If the count is odd, the player is in a losing position regardless of the move, but must still make a legal move.
2. **Compute Nim-Sum**: For configurations where not all piles are 1, compute the nim-sum (XOR of all pile sizes). If the nim-sum is non-zero, the current player can win by making a move that results in a nim-sum of zero. If the nim-sum is zero, the current player is in a losing position but must return a valid move.

### Solution Code
```python
def policy(piles: list[int]) -> str:
    # Check if all piles are 1
    all_ones = all(p == 1 for p in piles)
    if all_ones:
        count = sum(piles)  # This equals the number of piles since all are 1s
        if count % 2 == 0:
            # If even count, take one to leave odd count which is a losing position for the opponent
            for i in range(4):
                if piles[i] > 0:
                    return f"{i},1"
        else:
            # If odd count, we have to take a pile, but we can't win. Just return any valid move.
            for i in range(4):
                if piles[i] > 0:
                    return f"{i},1"
    
    # Compute the nim-sum for the current state
    nim_sum = piles[0] ^ piles[1] ^ piles[2] ^ piles[3]
    
    if nim_sum != 0:
        # Find a move that makes the nim-sum zero
        for i in range(4):
            if piles[i] == 0:
                continue
            target = piles[i] ^ nim_sum
            if target < piles[i]:
                take = piles[i] - target
                return f"{i},{take}"
        # If no such target found (shouldn't happen), take one from the first non-zero pile
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
    else:
        # Nim-sum is zero; take one from the first non-zero pile as a losing move
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
```

### Explanation
1. **All Piles Are 1**: The code first checks if all piles have exactly 1 object. If the count of such piles is even, the current player can take one pile, leaving an odd number of piles, which forces the opponent into a losing position. If the count is odd, any move will result in an even number of piles, which is a winning position for the opponent.
2. **Nim-Sum Calculation**: For configurations where not all piles are 1, the code computes the nim-sum. If the nim-sum is non-zero, the player can reduce it to zero by adjusting one of the piles. This move forces the opponent into a losing position. If the nim-sum is zero, the player is in a losing position and must return any valid move, as any move will lead to the opponent's advantage.

This approach efficiently determines the optimal move using bitwise operations and conditional checks, ensuring the policy complies with the game's rules and maximizes the chances of winning.
