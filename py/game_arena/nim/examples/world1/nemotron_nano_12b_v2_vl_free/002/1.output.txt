To solve this problem, we need to create an optimal policy for a misère version of Nim with 4 piles, where the player who takes the last object loses. The strategy will involve using the XOR (nim-sum) operation to determine optimal moves, and handling edge cases where all piles are of size 1.

### Approach
1. **Check for Special Case (All Non-zero Piles are 1)**: If all non-zero piles are of size 1, the game is in a special state where the player must take the last object to lose. In this case, the optimal move is to take 1 token from any non-zero pile.
2. **Compute Nim-Sum**: For the general case where not all non-zero piles are size 1, compute the nim-sum (XOR of all pile sizes). The nim-sum helps determine the optimal move in normal play Nim.
3. **Find Optimal Move**: If the nim-sum is non-zero, the player can win by taking tokens from a pile such that the new nim-sum becomes zero. This is done by finding a pile where reducing it to the XOR of itself with the nim-sum leaves a valid number of tokens to take.
4. **Fallback Move**: If the nim-sum is zero, the player is in a losing position but must return a legal move. In this case, the player takes 1 token from the first non-zero pile.

### Solution Code

```python
def policy(piles: list[int]) -> str:
    # Check if all non-zero piles have size 1
    non_zero = [x for x in piles if x > 0]
    if len(non_zero) > 0 and all(x == 1 for x in non_zero):
        # All non-zero piles are size 1, take one from the first
        for i in range(4):
            if piles[i] == 1:
                return f"{i},1"
    
    # Compute nim-sum
    nim_sum = 0
    for p in piles:
        nim_sum ^= p
    
    # If nim_sum is non-zero, find a winning move
    for idx in range(4):
        x = piles[idx]
        if x == 0:
            continue
        desired = nim_sum ^ x
        if desired < x:
            take = x - desired
            return f"{idx},{take}"
    
    # If nim_sum is zero, make any legal move
    for i in range(4):
        if piles[i] > 0:
            return f"{i},1"
    
    # This line is theoretically unreachable due to game rules
    return ""
```

### Explanation
- **Special Case Handling**: When all non-zero piles are size 1, the player will take 1 token from the first available pile. This ensures that the opponent is left with an odd number of non-zero piles, which is a losing position.
- **Nim-Sum Calculation**: The nim-sum (XOR of all pile sizes) determines the optimal move. If the nim-sum is non-zero, the player can force a win by reducing a pile to make the nim-sum zero.
- **Fallback Move**: If the nim-sum is zero, the player is in a losing position but must return a legal move. The code then takes 1 token from the first non-zero pile, which is a legal move but may not be optimal in the long term.

This approach ensures that the code adheres to the rules of misère Nim, handling both general and special move scenarios optimally.
