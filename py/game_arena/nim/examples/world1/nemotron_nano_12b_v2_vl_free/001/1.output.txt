To solve this problem, we need to create an AI player for a variant of Nim where the player who takes the last object loses. The solution must use a strategy that ensures the AI can force the opponent into a losing position whenever possible.

### Approach
1. **Check for All Ones**: If all piles have exactly one object, the game outcome depends on the parity of the number of piles. If the number of piles is even, the current player can win by taking one object, leaving an odd number of piles. This forces the opponent into a losing position. If the number of piles is odd, the current player must take a move that leads to a loss, but still returns a valid move.
2. **Normal Nim Strategy**: For positions where not all piles have one object, use the standard Nim strategy based on the XOR (nim-sum) of all pile sizes. The goal is to reduce the nim-sum to zero, forcing the opponent into a losing position. If the nim-sum is non-zero, find a pile to reduce such that the new nim-sum becomes zero. If the nim-sum is zero, the player is in a losing position and must return a valid move.

### Solution Code
```python
def policy(piles: list[int]) -> str:
    # Check if all piles are 1
    all_ones = all(p == 1 for p in piles)
    if all_ones:
        # Take one from the first non-zero pile
        for i in range(4):
            if piles[i] == 1:
                return f"{i},1"
    else:
        # Compute the nim-sum
        nim_sum = 0
        for p in piles:
            nim_sum ^= p
        # Find a move to make nim_sum zero
        for i in range(4):
            target = piles[i] ^ nim_sum
            if target < piles[i]:
                take = piles[i] - target
                return f"{i},{take}"
        # Fallback if no move found (nim_sum is zero)
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
```

### Explanation
1. **All Ones Check**: The code first checks if all piles have exactly one object. If true, it takes one object from the first non-zero pile. This move is optimal if the number of piles is even (forcing the opponent into a losing position) and unavoidable if the number is odd (leading to a loss).
2. **Nim-Sum Calculation**: The nim-sum is calculated using the XOR of all pile sizes. This helps determine the optimal move in the normal Nim strategy.
3. **Target Calculation**: For each pile, compute the target size using XOR with the nim-sum. If reducing the current pile to this target is possible, the move is made by taking the difference between the current and target sizes.
4. **Fallback Move**: If no valid move is found (which should only happen if the nim-sum is zero and all moves leave the opponent with a winning position), the code defaults to taking one object from the first non-zero pile.

This approach ensures that the AI uses the optimal strategy for Nim, adapting for the misère variant by handling the special case where all piles are of size one.
