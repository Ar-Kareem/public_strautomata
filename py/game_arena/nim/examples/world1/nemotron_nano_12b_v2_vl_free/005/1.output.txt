To solve this problem, we need to create an AI policy for a variant of the Nim game where the player who takes the last object loses. The solution involves a combination of the Sprague-Grundy theorem for normal Nim and special handling for the misère end-game scenario where all piles contain exactly one object.

### Approach
1. **Check for All Ones**: If all piles contain exactly one object, the outcome depends on the parity of the number of piles. If the count of piles is even, the current player can win by reducing the count to an odd number. Otherwise, the player is in a losing position regardless.
2. **Nim-Sum Calculation**: For states where not all piles are one, use the Sprague-Grundy theorem. Compute the nim-sum (XOR) of all pile sizes. If the nim-sum is non-zero, the current player can force a win by making a move that results in a nim-sum of zero.
3. **Misère Adjustment**: If the nim-sum is zero, the current player is in a losing position and must make a legal move that leaves the opponent in a non-losing state.

### Solution Code
```python
def policy(piles: list[int]) -> str:
    # Check if all piles are exactly 1
    all_ones = all(p == 1 for p in piles)
    if all_ones:
        # If all piles are 1, take one from the first pile (any pile would work)
        # This ensures the opponent is left with an odd number of piles if original count was even.
        return "0,1"
    
    # Calculate the nim-sum of the piles
    nim_sum = 0
    for p in piles:
        nim_sum ^= p
    
    # If nim-sum is non-zero, find a move to set it to zero
    if nim_sum != 0:
        for i in range(4):
            if piles[i] == 0:
                continue
            desired = piles[i] ^ nim_sum
            if desired < piles[i]:
                return f"{i},{piles[i] - desired}"
    
    # If nim-sum is zero (no winning move in normal play), take 1 from the first non-empty pile
    for i in range(4):
        if piles[i] > 0:
            return f"{i},1"
```

### Explanation
1. **All Ones Check**: The code first checks if all piles are exactly 1. If so, it returns a move that takes one object from the first pile. This ensures the opponent faces an odd number of piles, which is a losing position under misère rules.
2. **Nim-Sum Calculation**: The nim-sum is computed by XORing all pile sizes. If the result is non-zero, the current player can force a win by adjusting one pile's size to make the nim-sum zero. This involves finding a pile where reducing its size to `pile ^ nim_sum` is feasible.
3. **Fallback Move**: If the nim-sum is zero (indicating a losing position in normal play), the code defaults to taking one object from the first non-empty pile, ensuring a legal move is always returned.

This approach ensures optimal play by leveraging the nim-sum strategy for general cases and handling the special misère case where all piles are one object.
