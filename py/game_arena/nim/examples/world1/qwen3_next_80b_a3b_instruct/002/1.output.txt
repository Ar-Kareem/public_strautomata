The game described is a variant of Nim called "Misère Nim" — where the player who takes the last object loses. This is different from standard Nim, where taking the last object wins.

In standard Nim, the winning strategy is based on the XOR (nim-sum) of all pile sizes: if the nim-sum is zero, the position is losing for the player about to move; otherwise, it's winning.

However, in Misère Nim, the strategy is slightly different. For Misère Nim with more than one pile containing more than one object, the strategy is identical to standard Nim. The critical difference arises when all remaining piles have size 1 — then the parity (odd/even count of piles) determines the outcome: if the number of piles of size 1 is odd, the current player loses with perfect play (because the opponent can force them to take the last one).

But here’s the key insight for 4-pile Misère Nim:

1. **If all piles are size 1**: There are 4 piles (even), so the current player can win by taking one pile entirely, leaving 3 piles of 1 for the opponent — who will be forced to take the last one. So in this case, the winning move is to take any one pile completely (leaving 3 piles of 1).

2. **If exactly one pile has more than 1 object**, and the rest are 0 or 1:
   - We want to leave an odd number of piles of size 1 for the opponent. So if there are `k` piles of size 1, and one pile `p > 1`, then:
     - If `k` is even, reduce the big pile to 1 → now we have `k+1` piles of size 1 (odd) → opponent loses.
     - If `k` is odd, reduce the big pile to 0 → now we have `k` piles of size 1 (odd) → opponent loses.
     - Actually, correction: since we lose by taking the last object, we want to *force the opponent to take the last object*. So we want to leave an *odd* number of piles of size 1 for the opponent.

import numpy as np

def policy(piles: list[int]) -> str:
    # Count number of piles greater than 1
    large_count = sum(1 for p in piles if p > 1)
    ones_count = sum(1 for p in piles if p == 1)
    
    # Case 1: All piles are 0 or 1 (i.e., no large piles)
    if large_count == 0:
        # We have 4 piles total, some are 0, some are 1. We need to take one object from a pile of 1.
        # Find any pile with 1 and take it all.
        for i in range(4):
            if piles[i] == 1:
                return f"{i},1"
        # If no pile is 1? (shouldn't happen since large_count=0 and not all zero)
        # Take from first non-zero pile
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
    
    # Case 2: Exactly one pile > 1
    elif large_count == 1:
        # Find the large pile index
        big_idx = -1
        for i in range(4):
            if piles[i] > 1:
                big_idx = i
                break
        # If we have even number of ones, reduce big pile to 1
        if ones_count % 2 == 0:
            take = piles[big_idx] - 1
            return f"{big_idx},{take}"
        else:
            # Reduce big pile to 0
            take = piles[big_idx]
            return f"{big_idx},{take}"
    
    # Case 3: Two or more piles > 1
    else:
        # Compute nim-sum
        nim_sum = 0
        for p in piles:
            nim_sum ^= p
        
        # If nim-sum is 0, we're in a losing position. We need to make any legal move.
        if nim_sum == 0:
            # Find first non-zero pile and take 1 (minimal move)
            for i in range(4):
                if piles[i] > 0:
                    return f"{i},1"
            # All zero? shouldn't happen
            return "0,1"
        
        # Otherwise, find a move that makes nim_sum = 0
        for i in range(4):
            if piles[i] == 0:
                continue
            # What should be the new size of pile i to make nim_sum 0?
            new_size = piles[i] ^ nim_sum
            if new_size < piles[i]:  # We can reduce this pile to new_size
                take = piles[i] - new_size
                return f"{i},{take}"
        
        # If we didn't find a move? Shouldn't happen due to Nim theory.
        # Fallback: take one from first non-zero pile
        for i in range(4):
            if piles[i] > 0:
