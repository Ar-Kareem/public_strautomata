The strategy for this policy is based on the optimal solution for **Misère Nim**.
The game of Nim is solved using the Nim-sum (XOR sum) of the pile sizes. In **Normal Play** (last player to move wins), the winning strategy is to make a move such that the Nim-sum becomes zero. In **Misère Play** (last player to move loses), the strategy is identical to Normal Play for most of the game, with a specific exception near the end.

The policy logic is as follows:
1.  **Analyze the State**: Count how many piles have size greater than 1 (`large_piles`).
2.  **Misère Exception (Endgame Control)**: If there is exactly one pile with more than 1 item, we must deviate from the standard Nim-sum strategy. In this scenario, we can force a win by reducing the large pile to either 0 or 1 such that the total count of piles with size 1 remaining on the board is **odd**. This forces the opponent into a chain of forced moves that results in them taking the last object.
3.  **Standard Strategy**: If there are two or more piles with size greater than 1, we follow the standard Nim strategy:
    - Compute the Nim-sum of all piles.
    - If the Nim-sum is non-zero (winning position), find a move that results in a Nim-sum of zero. This is done by finding a pile $p$ where $p \oplus \text{Nim-sum} < p$.
    - If the Nim-sum is zero (losing position), play any legal move (e.g., take 1 from the first available pile) and hope the opponent makes a mistake.
4.  **All 1s**: If all piles are size 0 or 1, the optimal move is forced (take 1), and the outcome is determined by the parity of the pile count.

def policy(piles: list[int]) -> str:
    """
    Returns the next move for a Misere Nim game (4 piles).
    Format: 'pile,take' where pile is 0-3 and take is amount to remove.
    """
    # Identify indices of piles with more than 1 item
    large_pile_indices = [i for i, p in enumerate(piles) if p > 1]
    
    move_pile = -1
    move_take = -1

    # STRATEGY: MISERE EXCEPTION
    # If exactly one pile is > 1, we must leave an ODD number of piles of size 1.
    if len(large_pile_indices) == 1:
        idx = large_pile_indices[0]
        # Count current piles of size 1 (excluding the large one, obviously)
        ones_count = sum(1 for p in piles if p == 1)
        
        # If ones_count is even, we need to add a 1 (reduce large pile to 1).
        # Total ones becomes (even + 1) = odd.
        # If ones_count is odd, we need to add 0 (reduce large pile to 0).
        # Total ones becomes (odd + 0) = odd.
        if ones_count % 2 == 0:
            target_size = 1
        else:
            target_size = 0
            
        move_pile = idx
        move_take = piles[idx] - target_size

    # STRATEGY: NORMAL NIM
    # If there are 0 or >= 2 large piles, standard Nim-sum logic applies.
    # (Note: if 0 large piles, only 1s remain, implies Nim-sum logic works or moves are forced).
    else:
        nim_sum = 0
        for p in piles:
            nim_sum ^= p
        
        if nim_sum != 0:
            # Winning position: find a move to make nim_sum 0
            for i, p in enumerate(piles):
                target = p ^ nim_sum
                if target < p:
                    move_pile = i
                    move_take = p - target
                    break
        else:
            # Losing position: must make a move. Take 1 from first non-empty pile.
            for i, p in enumerate(piles):
                if p > 0:
                    move_pile = i
                    move_take = 1
                    break
    
    return f"{move_pile},{move_take}"
