# Nim Strategy

In standard Nim, the losing positions are those where the bitwise XOR of all pile sizes is 0. However, the problem description states "The player who takes the last object loses", which makes this an *misère* variant.

In misère Nim:
- If all piles are size 1, the optimal move is to leave an odd number of piles (if possible) for the opponent.
- Otherwise, play exactly like normal Nim (aim for XOR = 0) because the strategy is the same until the endgame.

The policy should:
1. Check if all piles have size 1. If so, remove 1 object from a pile to leave an odd number of piles.
2. Calculate the XOR of all piles. If it's 0, we're in a losing position, so make any legal move.
3. If XOR is non-zero, find a pile where we can reduce it to make the XOR 0.

<code>
def policy(piles: list[int]) -> str:
    # Check if all piles are size 1
    if all(p == 1 for p in piles):
        # Find a pile with 1 object to remove
        for i in range(4):
            if piles[i] == 1:
                return f"{i},1"
    
    # Calculate XOR of all piles
    xor_sum = 0
    for p in piles:
        xor_sum ^= p
    
    # If XOR is 0, we're in a losing position
    if xor_sum == 0:
        # Make any legal move - find first non-empty pile
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
    
    # Find a pile where we can make XOR 0
    for i in range(4):
        if piles[i] > 0:
            # Calculate what the pile should be after our move
            target = piles[i] ^ xor_sum
            if target < piles[i]:
                # We can reduce this pile to target
                take = piles[i] - target
                return f"{i},{take}"
    
    # This should never happen with valid input
    return "0,1"
