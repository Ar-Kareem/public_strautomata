The strategy for this policy is to play **optimal misère Nim** for 4 piles.

Key idea:
- In normal Nim, the winning rule is based on making the xor (`nim-sum`) equal to 0.
- Here, the rule is **misère**: the player who takes the last object **loses**.
- Misère Nim is the same as normal Nim except for the special case where **all nonzero piles are size 1**.

Optimal policy:
1. If all nonzero piles are of size 1:
   - Leave an **odd** number of size-1 piles to the opponent.
   - Equivalently:
     - if the number of 1-heaps is even, remove 1 from any nonzero pile;
     - if it is odd, no winning move exists, so return any legal move.
2. Otherwise:
   - If exactly one pile is larger than 1:
     - Let `ones` be the number of size-1 piles.
     - Reduce the large pile to:
       - `1` if `ones` is odd
       - `0` if `ones` is even
   - If at least two piles are larger than 1:
     - Play like normal Nim: make the xor of pile sizes equal to 0.
3. As a safety fallback, always return the first legal move if anything unexpected happens.

This guarantees a legal move and achieves perfect play.

<code>
def policy(piles: list[int]) -> str:
    # Always return a legal move string "pile,take".
    # Misère Nim optimal strategy for 4 piles.

    # Safety: find all non-empty piles
    nonzero = [i for i, x in enumerate(piles) if x > 0]
    if not nonzero:
        # Should not normally be called on terminal states, but stay safe.
        return "0,0"

    ones = sum(1 for x in piles if x == 1)
    big = [(i, x) for i, x in enumerate(piles) if x > 1]

    # Case 1: all nonzero piles are size 1
    if len(big) == 0:
        # Winning move iff number of 1-heaps is even:
        # remove one whole heap, leaving odd count to opponent.
        i = nonzero[0]
        return f"{i},1"

    # Case 2: exactly one pile bigger than 1
    if len(big) == 1:
        i, x = big[0]
        # In misère Nim, leave opponent with:
        # - odd number of ones if we reduce big pile to 0 when ones even
        # - odd number of ones if we reduce big pile to 1 when ones odd
        target = 1 if (ones % 2 == 1) else 0
        take = x - target
        if 1 <= take <= x:
            return f"{i},{take}"

    # Case 3: at least two piles > 1 => normal Nim move
    xor_sum = 0
    for x in piles:
        xor_sum ^= x

    if xor_sum != 0:
        for i, x in enumerate(piles):
            target = x ^ xor_sum
            if target < x:
                take = x - target
                if 1 <= take <= x:
                    return f"{i},{take}"

    # Losing position or any unexpected issue: return first legal move
    for i, x in enumerate(piles):
        if x > 0:
            return f"{i},1"

    # Final safety fallback
    return "0,0"
