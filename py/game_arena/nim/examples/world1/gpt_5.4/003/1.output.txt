The strategy for this policy is to play optimal **misère Nim** for 4 piles.

Key idea:
- In normal Nim, winning play is based on making the xor (`nim-sum`) equal to 0.
- But here **taking the last object loses**, so this is **misère Nim**.
- Misère Nim is the same as normal Nim except for the special case where **all nonzero piles are size 1**.
  - If all remaining nonzero piles are 1s, then you want to leave an **odd** number of 1-piles to the opponent.
- More generally:
  - If all nonzero piles are 1s:
    - If the count of 1-piles is even, remove 1 from any nonempty pile.
    - If the count is odd, every move loses with perfect opposition, but we must still return a legal move, so remove 1 from any nonempty pile.
  - Otherwise, if there is at least one pile larger than 1:
    - Use the normal Nim xor move, but when reducing a pile, avoid moving into the bad all-ones parity when applicable.
    - Equivalently for misère Nim:
      - If there is exactly one pile larger than 1:
        - Let `ones` be the number of piles equal to 1.
        - Reduce that large pile to:
          - `1` if `ones` is odd
          - `0` if `ones` is even
      - If there are at least two piles larger than 1:
        - Make the xor equal to 0 as in normal Nim.
- A final fallback always returns some legal move, guaranteeing no disqualification.

<code>
def policy(piles: list[int]) -> str:
    # Ensure length 4 assumption is handled robustly
    n = len(piles)

    # Collect pile categories
    nonzero = [i for i, x in enumerate(piles) if x > 0]
    ones = [i for i, x in enumerate(piles) if x == 1]
    big = [i for i, x in enumerate(piles) if x > 1]

    # No legal move exists in a terminal state, but return a syntactically valid default.
    # Arena should not call policy on terminal states.
    if not nonzero:
        return "0,0"

    # Misère Nim special handling

    # Case 1: all nonzero piles are size 1
    if len(big) == 0:
        # Optimal: leave an odd number of 1-heaps to opponent.
        # If count is even, remove one to make it odd.
        # If count is odd, no winning move exists; remove one legally.
        i = nonzero[0]
        return f"{i},1"

    # Case 2: exactly one pile larger than 1
    if len(big) == 1:
        i = big[0]
        ones_count = len(ones)
        target = 1 if (ones_count % 2 == 1) else 0
        take = piles[i] - target
        if take >= 1:
            return f"{i},{take}"

    # Case 3: at least two piles larger than 1
    # Play normal Nim: make xor zero
    xor_sum = 0
    for x in piles:
        xor_sum ^= x

    if xor_sum != 0:
        for i, x in enumerate(piles):
            target = x ^ xor_sum
            if target < x:
                take = x - target
                if 1 <= take <= x:
                    return f"{i},{take}"

    # Fallback: always return a legal move
    for i, x in enumerate(piles):
        if x > 0:
            return f"{i},1"

    # Unreachable unless terminal
    return "0,0"
