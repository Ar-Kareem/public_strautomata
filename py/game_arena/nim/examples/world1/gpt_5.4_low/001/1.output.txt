The strategy for this policy is to play **optimal misère Nim** for 4 piles.

Key idea:
- In normal Nim, you try to move to a position with xor (`nim-sum`) equal to 0.
- In **misère** Nim, that remains correct **except** for the endgame where all nonempty piles are size 1.
- Special handling:
  1. **If all nonempty piles are size 1**: this is losing iff the number of 1-heaps is odd. So the best move is to remove exactly 1 from any nonempty pile, leaving an odd number of 1-heaps to the opponent.
  2. **If exactly one pile is bigger than 1**: reduce that pile so the opponent sees an odd number of 1-heaps.
     - If the number of 1-heaps is even, reduce the big pile to 1.
     - If the number of 1-heaps is odd, remove the big pile entirely.
  3. **Otherwise**: use the standard Nim xor move to make the xor 0.

The code also includes a guaranteed legal fallback move.

<code>
def policy(piles: list[int]) -> str:
    # Guaranteed legal fallback: take 1 from the first nonempty pile.
    def fallback():
        for i, p in enumerate(piles):
            if p > 0:
                return f"{i},1"
        # Terminal position should normally not be queried.
        return "0,1"

    nonzero = [(i, p) for i, p in enumerate(piles) if p > 0]
    if not nonzero:
        return fallback()

    ones = [i for i, p in nonzero if p == 1]
    bigs = [(i, p) for i, p in nonzero if p > 1]

    # Case 1: all nonempty piles are size 1
    if len(bigs) == 0:
        # Remove one whole heap (the only legal type of move here).
        # This is optimal because it leaves odd count to opponent when possible.
        return f"{ones[0]},1"

    # Case 2: exactly one pile > 1
    if len(bigs) == 1:
        big_i, big_p = bigs[0]
        one_count = len(ones)

        # Leave an odd number of 1-heaps to the opponent.
        if one_count % 2 == 0:
            # Reduce big heap to 1
            take = big_p - 1
            if take >= 1:
                return f"{big_i},{take}"
        else:
            # Remove big heap entirely
            take = big_p
            return f"{big_i},{take}"

        return fallback()

    # Case 3: at least two piles > 1 -> normal Nim move
    x = 0
    for p in piles:
        x ^= p

    if x != 0:
        for i, p in enumerate(piles):
            target = p ^ x
            if target < p:
                take = p - target
                if take >= 1:
                    return f"{i},{take}"

    # If xor is already 0, this is a losing position with perfect play.
    # Still return a legal move.
    return fallback()
