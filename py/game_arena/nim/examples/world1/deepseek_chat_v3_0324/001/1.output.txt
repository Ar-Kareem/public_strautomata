Let me explain the logic for designing an optimal Nim policy where the player who takes the last object loses (misère game):

1. **Nim Game Understanding**:
   - In standard Nim (last-move wins), optimal play involves calculating the binary XOR (nim-sum) of all pile sizes.
   - For misère Nim (last-move loses), the strategy is almost identical to standard Nim except when all piles have size 1.

2. **Misère Nim Strategy**:
   - If the position is "non-singular" (normal play), play the standard Nim strategy (make nim-sum zero).
   - For positions where all piles are size 1:
     - If the number of piles is odd, take a whole pile to make it even (take last object)
     - If even, force opponent into odd position by taking a partial pile (but this can't be done when all are 1)

3. **Implementation Approach**:
   - Calculate the nim-sum (binary XOR) of all pile sizes
   - If the nim-sum is non-zero (non-terminal position), use standard Nim strategy
   - If all piles are size 1: follow misère endgame strategy
   - Otherwise, find the winning move that makes nim-sum zero

Here's the Python implementation:

<code>
import functools
import operator

def policy(piles: list[int]) -> str:
    # Calculate nim-sum (binary XOR of all piles)
    nim_sum = functools.reduce(operator.xor, piles)
    
    # Check if all piles are size 1 (misère endgame)
    if all(pile <= 1 for pile in piles):
        # Odd number of 1's - must take last one (losing position)
        # We can make opponent take last by taking whole pile
        # Or minimize damage by taking from largest pile if possible
        for i, pile in enumerate(piles):
            if pile > 0:
                return f"{i},{pile}"
    
    # Normal Nim strategy - find move that makes nim-sum zero
    for i, pile in enumerate(piles):
        if pile == 0:
            continue
        target_size = pile ^ nim_sum
        if target_size < pile:
            take = pile - target_size
            return f"{i},{take}"
    
    # Fallback: if no optimal move found, take from largest pile
    max_pile = max(piles)
    for i, pile in reversed(list(enumerate(piles))):
        if pile == max_pile and pile > 0:
            return f"{i},1"
    
    # Default move (should never reach here in valid game states)
    for i, pile in enumerate(piles):
        if pile > 0:
            return f"{i},1"
    return "0,1"  # failsafe
</code>

This policy:
1. First checks for the misère endgame scenario (all piles size 1)
2. Uses standard Nim strategy for normal positions by calculating nim-sum
3. Has fallbacks to ensure a legal move is always returned (though optimal when possible)
4. Should perform well against non-optimal players and other strategies

The algorithm runs in O(n) time (where n=4 is the number of piles), well within the 1 second constraint.
