**Strategy**

The game is Nim with the misère rule: the player who takes the last stone loses.  
The optimal (perfect‑play) strategy for misère Nim differs from ordinary Nim only when **all non‑empty piles have size 1**. In that special case the outcome is decided by parity: an even number of such piles is a winning position for the player to move, an odd number is a losing position. The player must simply take one stone from a pile (i.e., remove that pile entirely) to flip the parity.

For all other positions (i.e., when at least one pile contains more than one stone) the game behaves exactly like ordinary Nim: compute the Nim‑sum (XOR) of the pile sizes and, if the Nim‑sum is non‑zero, make a move that leaves the Nim‑sum equal to 0. This is done by selecting a pile larger than the Nim‑sum and reducing it to the Nim‑sum value.

The algorithm therefore works in two phases:

1. **Detect the special case** – if there is any pile > 1.
2. **Compute the Nim‑sum**.
   * If a big pile exists and the Nim‑sum ≠ 0 → normal Nim move (make XOR = 0).
   * If a big pile exists and the Nim‑sum = 0 → any legal move is fine (the player is already in a losing position).
3. **Otherwise (all piles are 0 or 1)** – use the parity rule.
   * If the number of piles containing one stone is even → winning → remove a single stone from any pile (flip parity to odd).
   * If the number is odd → losing → still return a legal move (remove one stone from any pile).

The code below follows this logic, runs in constant time for the fixed 4‑pile board, and always returns a legal move string in the required `'pile,take'` format.

from typing import List

def policy(piles: List[int]) -> str:
    """
    Return the optimal move for a misère Nim game with exactly 4 piles.
    Move format: 'pile,take' where pile is 0‑based index and take is the number
    of stones removed (1 ≤ take ≤ current size of the pile).
    """
    # If all piles are empty there is no move – this situation never occurs
    # during normal play, but we guard against it.
    if all(x == 0 for x in piles):
        # According to the arena constraints, this case should not be reachable.
        # Returning an empty string would be illegal; we raise an exception.
        raise ValueError("No legal move – all piles are zero")

    # ----------------------------------------------------------------------
    # 1. Identify whether any pile has more than one stone.
    # ----------------------------------------------------------------------
    any_big = any(x > 1 for x in piles)

    # ----------------------------------------------------------------------
    # 2. Compute Nim‑sum (xor of all pile sizes).
    # ----------------------------------------------------------------------
    xor_sum = 0
    for size in piles:
        xor_sum ^= size

    # ----------------------------------------------------------------------
    # 3. Normal Nim handling (misère Nim special case only when all piles ≤ 1).
    # ----------------------------------------------------------------------
    if any_big:
        # Normal Nim rule applies when a heap of size ≥ 2 exists.
        if xor_sum != 0:
            # Find a pile we can reduce to make the xor zero.
            for i, size in enumerate(piles):
                if size > xor_sum:
                    take = size - xor_sum   # number of stones to remove
                    # take is guaranteed to be ≥ 1 because size > xor_sum.
                    return f"{i},{take}"
        # xor_sum == 0 → current player is in a losing position.
        # Return any legal move (remove 1 from the first non‑empty pile).
        for i, size in enumerate(piles):
            if size > 0:
                return f"{i},1"
    else:
        # All piles are 0 or 1 → misère Nim parity rule.
        count_ones = sum(size == 1 for size in piles)

        # If count_ones is even we are in a winning position → flip parity.
        if count_ones % 2 == 0:
            for i, size in enumerate(piles):
                if size == 1:
                    return f"{i},1"
        else:
            # Losing position → any legal move suffices.
            for i, size in enumerate(piles):
                if size == 1:
                    return f"{i},1"

    # The logic above guarantees we never reach here for a reachable state.
    # Fallback (should not be used):
    for i, size in enumerate(piles):
        if size > 0:
            return f"{i},1"
