Looking at this problem, I need to implement optimal play for Misère Nim (where taking the last object loses, unlike standard Nim where it wins).

**Strategy Analysis:**

For Misère Nim, the key insight is that the strategy differs from standard Nim only in the endgame:

1. **Endgame (all piles ≤ 1):** The player facing an odd number of 1-piles loses. So I want to leave my opponent with an odd number of 1-piles.

2. **Exactly one pile > 1:** This is a transitional state. I control whether to leave odd or even 1-piles by reducing the large pile to either 0 or 1:
   - If current count of 1-piles is even: reduce large pile to 1 (total becomes odd)
   - If current count of 1-piles is odd: reduce large pile to 0 (total stays odd)

3. **Two or more piles > 1:** Play standard Nim using XOR (nim-sum):
   - If nim-sum = 0: losing position, make any legal move
   - If nim-sum ≠ 0: winning position, reduce some pile `p` to `p XOR nim_sum` to make the resulting nim-sum 0

The mathematical foundation: In standard Nim with piles > 1, positions with nim-sum = 0 are P-positions (losing for current player). In Misère, this remains true as long as at least one pile > 1. The modification only affects the endgame resolution.

**Verification examples:**
- `[5, 1, 1, 1]`: one pile > 1, 3 ones (odd) → reduce 5 to 0, leaving opponent with 3 ones → opponent loses
- `[5, 1, 1, 0]`: one pile > 1, 2 ones (even) → reduce 5 to 1, leaving opponent with 3 ones → opponent loses
- `[1, 3, 5, 7]`: nim_sum = 0, losing position for us

def policy(piles: list[int]) -> str:
    nim_sum = piles[0] ^ piles[1] ^ piles[2] ^ piles[3]
    
    large_count = sum(1 for p in piles if p > 1)
    ones = sum(1 for p in piles if p == 1)
    
    if large_count == 0:
        # Endgame: all piles are 0 or 1
        for i, p in enumerate(piles):
            if p == 1:
                return f"{i},1"
    
    elif large_count == 1:
        # Exactly one pile > 1: control the endgame
        for i, p in enumerate(piles):
            if p > 1:
                # Leave opponent with odd number of 1-piles
                if ones % 2 == 0:
                    take = p - 1  # Reduce to 1, making total ones odd
                else:
                    take = p  # Reduce to 0, keeping total ones odd
                return f"{i},{take}"
    
    else:  # large_count >= 2
        if nim_sum == 0:
            # Losing position, make any legal move
            for i, p in enumerate(piles):
                if p > 0:
                    return f"{i},1"
        else:
            # Winning position, standard Nim move to make nim_sum = 0
            for i, p in enumerate(piles):
                target = p ^ nim_sum
                if target < p:
                    return f"{i},{p - target}"
    
    # Fallback (shouldn't reach in valid game state)
    for i, p in enumerate(piles):
        if p > 0:
            return f"{i},1"
    return "0,1"
