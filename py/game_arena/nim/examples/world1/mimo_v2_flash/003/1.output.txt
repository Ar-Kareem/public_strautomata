The strategy for this policy is based on the theory of Nim with a "misere" (last player loses) variant. Since the rules state "The player who takes the last object loses", this is a misere Nim game.

For 4 piles, the optimal strategy is:
1.  **Non-Terminal States (More than 1 pile has >0 stones):** Play the same as standard Nim. Calculate the Nim-sum (XOR sum) of all pile sizes. If the Nim-sum is non-zero, make a move that brings the Nim-sum to zero. If the Nim-sum is zero (and it's not a losing endgame), make any legal move (though usually we try to preserve symmetry or keep the game going, but strictly speaking, if you are in a zero Nim-sum state against a perfect opponent, you will lose). However, since we want to beat other policies, we assume the opponent is not necessarily perfect, so we pick the move that minimizes Nim-sum (which is 0) if possible.
2.  **Endgame (Exactly 1 pile has >0 stones):** If there is only one pile left, you must take **all** stones minus 1. This leaves 1 stone for the opponent, forcing them to take the last stone and lose.
    *   *Correction*: Wait, if you take all stones, you take the last stone and lose. If you take all but 1, the opponent takes the last stone and loses. Correct.
    *   *Edge Case*: If the single pile has size 1, you are forced to take it (take 1), and you lose. There is no winning move.

The code implements a `policy` function that calculates the Nim-sum (XOR of all piles), checks if the game is in the single-pile endgame, and otherwise attempts to reduce the Nim-sum to 0. If the Nim-sum is already 0 (a theoretical losing position), it defaults to taking 1 from the first non-empty pile.

import sys

def policy(piles: list[int]) -> str:
    # Calculate Nim-sum (XOR of all piles)
    xor_sum = 0
    non_zero_count = 0
    last_non_zero_index = -1
    
    for i, p in enumerate(piles):
        xor_sum ^= p
        if p > 0:
            non_zero_count += 1
            last_non_zero_index = i

    # Misere Nim Strategy:
    
    # 1. Endgame case: Only one pile remains.
    # In Misere Nim, if only one pile is left, you must leave 1 stone for the opponent.
    # So you take (size - 1). If size is 1, you lose (must take 1).
    if non_zero_count == 1:
        pile_idx = last_non_zero_index
        take = piles[pile_idx] - 1
        if take < 1: # Case where pile size is 1
            take = 1 # Forced to take last stone
        return f"{pile_idx},{take}"

    # 2. General case (more than 1 pile).
    # Play standard Nim strategy: If xor_sum != 0, move to make xor_sum 0.
    # If xor_sum == 0, we are in a losing position. We must make a legal move.
    # We choose the move that leaves the opponent with the smallest total stones, 
    # or simply removes 1 from the first pile.
    
    if xor_sum != 0:
        # Find a pile where (pile_size XOR xor_sum) < pile_size
        for i in range(4):
            target = piles[i] ^ xor_sum
            if target < piles[i]:
                take = piles[i] - target
                return f"{i},{take}"
    
    # Fallback for when xor_sum is 0 (losing position) or if logic fails (should not happen)
    # Pick the first non-empty pile and take 1.
    for i in range(4):
        if piles[i] > 0:
            return f"{i},1"
            
    # This line is theoretically unreachable if called on a non-terminal game state
    return "0,1"
