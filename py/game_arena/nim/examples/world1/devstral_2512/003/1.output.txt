To solve this Nim game problem, I'll implement a policy that uses the concept of "Nim-sum" to determine the optimal move. The Nim-sum is the bitwise XOR of all pile sizes. If the Nim-sum is zero, the current player is in a losing position (assuming the opponent plays optimally), and any move will leave the opponent in a winning position. If the Nim-sum is non-zero, the current player can force a win by making a move that leaves the opponent with a Nim-sum of zero.

### Approach
1. **Calculate the Nim-sum**: Compute the bitwise XOR of all pile sizes.
2. **Determine the target Nim-sum**: If the current Nim-sum is zero, the player is in a losing position, and any legal move is acceptable (though it won't lead to a win). If the Nim-sum is non-zero, the player should make a move that reduces the Nim-sum to zero.
3. **Find the optimal move**: For each pile, compute the new Nim-sum if that pile were reduced to a certain size. The goal is to find a pile where the new Nim-sum (after reducing the pile) is zero. The amount to take from that pile is the difference between the current pile size and the target size (which is the current pile size XOR the Nim-sum).

### Solution Code
```python
def policy(piles: list[int]) -> str:
    # Calculate the Nim-sum
    nim_sum = 0
    for pile in piles:
        nim_sum ^= pile

    # If Nim-sum is zero, any move is acceptable (but we'll still try to make a smart move)
    if nim_sum == 0:
        # Find the first non-zero pile and take 1 object
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
    else:
        # Find the pile to modify to make the Nim-sum zero
        for i in range(4):
            if (piles[i] ^ nim_sum) < piles[i]:
                take = piles[i] - (piles[i] ^ nim_sum)
                return f"{i},{take}"

    # Fallback: take 1 from the first non-zero pile (shouldn't reach here if piles are non-zero)
    for i in range(4):
        if piles[i] > 0:
            return f"{i},1"
```

### Explanation
1. **Nim-sum Calculation**: The Nim-sum is computed by XOR-ing all pile sizes. This helps in determining the current game state.
2. **Optimal Move Selection**: If the Nim-sum is non-zero, the code iterates through each pile to find a move that will make the new Nim-sum zero. This is done by checking if the XOR of the pile size and the Nim-sum is less than the current pile size, which ensures the move is valid.
3. **Fallback for Zero Nim-sum**: If the Nim-sum is zero, the player is in a losing position, and the code defaults to taking one object from the first non-zero pile, which is a simple heuristic to prolong the game.
4. **Legal Move Guarantee**: The code ensures that the move is always legal by checking the pile size before making a move and only taking a valid number of objects.

This approach ensures that the AI plays optimally, leveraging the mathematical properties of the Nim game to force a win whenever possible.
