To solve this Nim game problem, I'll implement a policy that uses the concept of "Nim-sum" to determine the optimal move. The Nim-sum is the bitwise XOR of all pile sizes. If the Nim-sum is non-zero, the current player can force a win by making a move that brings the Nim-sum to zero. If the Nim-sum is already zero, any move will leave the opponent in a winning position, so we'll make a random legal move.

### Approach
1. **Calculate the Nim-sum**: Compute the bitwise XOR of all pile sizes.
2. **Determine the optimal move**:
   - If the Nim-sum is non-zero, find a pile where the XOR of the pile size and the Nim-sum is less than the current pile size. This will be the pile to modify.
   - The number of objects to take from this pile is the difference between the current pile size and the result of the XOR operation.
3. **Handle the case where Nim-sum is zero**: If the Nim-sum is zero, any move will leave the opponent in a winning position. In this case, we'll make a move that takes 1 object from the first non-empty pile.

### Solution Code
```python
def policy(piles: list[int]) -> str:
    # Calculate the Nim-sum
    nim_sum = 0
    for pile in piles:
        nim_sum ^= pile

    # If Nim-sum is zero, make a random legal move (take 1 from the first non-empty pile)
    if nim_sum == 0:
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"

    # Find the pile to modify
    for i in range(4):
        if (piles[i] ^ nim_sum) < piles[i]:
            take = piles[i] - (piles[i] ^ nim_sum)
            return f"{i},{take}"

    # Fallback: take 1 from the first non-empty pile (should theoretically never reach here if Nim-sum is non-zero)
    for i in range(4):
        if piles[i] > 0:
            return f"{i},1"
```

### Explanation
1. **Nim-sum Calculation**: The Nim-sum is computed by performing a bitwise XOR operation on all pile sizes. This helps in determining the winning strategy.
2. **Optimal Move Determination**: If the Nim-sum is non-zero, the algorithm searches for a pile where modifying its size to the result of the XOR operation between the pile size and the Nim-sum will balance the game. The number of objects to take is calculated as the difference between the current pile size and this XOR result.
3. **Handling Zero Nim-sum**: If the Nim-sum is zero, the algorithm defaults to taking one object from the first non-empty pile, as any move in this scenario will not be optimal, but it ensures a legal move is made.

This approach ensures that the AI makes optimal moves to force a win whenever possible, adhering to the rules of the Nim game.
