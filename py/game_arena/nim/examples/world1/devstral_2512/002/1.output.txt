To solve this Nim game problem, I'll implement a policy that uses the concept of "Nim-sum" to determine the optimal move. The Nim-sum is the bitwise XOR of the sizes of all piles. If the Nim-sum is zero, the current player is in a losing position (assuming the opponent plays optimally), and any move will leave the opponent in a winning position. If the Nim-sum is non-zero, the current player can force a win by making a move that leaves the Nim-sum as zero.

### Approach
1. **Calculate the Nim-sum**: Compute the bitwise XOR of all pile sizes.
2. **Check for terminal state**: If all piles are zero, the game is over, but since the player to move loses, this state should not occur in the policy function.
3. **Determine the optimal move**:
   - If the Nim-sum is zero, the current player is in a losing position. In this case, the policy should make a random legal move (since any move will leave the opponent in a winning position).
   - If the Nim-sum is non-zero, find a pile where the XOR of the pile size and the Nim-sum is less than the current pile size. This ensures that after taking the appropriate number of objects, the new Nim-sum will be zero.

### Solution Code
```python
def policy(piles: list[int]) -> str:
    # Calculate the Nim-sum
    nim_sum = 0
    for pile in piles:
        nim_sum ^= pile

    # If Nim-sum is zero, make a random legal move (since any move is losing)
    if nim_sum == 0:
        # Find the first non-zero pile and take 1 object
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
    else:
        # Find the pile to modify
        for i in range(4):
            if (piles[i] ^ nim_sum) < piles[i]:
                take = piles[i] - (piles[i] ^ nim_sum)
                return f"{i},{take}"

    # Fallback: if no move found (shouldn't happen for valid piles)
    for i in range(4):
        if piles[i] > 0:
            return f"{i},1"
```

### Explanation
1. **Nim-sum Calculation**: The Nim-sum is computed by XORing all pile sizes. This helps in determining the winning or losing position.
2. **Losing Position Handling**: If the Nim-sum is zero, the current player is in a losing position. The policy then makes a move by taking 1 object from the first non-zero pile, as any move will leave the opponent in a winning position.
3. **Winning Position Handling**: If the Nim-sum is non-zero, the policy searches for a pile where the XOR of the pile size and the Nim-sum is less than the current pile size. The number of objects to take is calculated as the difference between the current pile size and this XOR result, ensuring the Nim-sum becomes zero after the move.
4. **Fallback**: In case no optimal move is found (which theoretically shouldn't happen for valid pile configurations), the policy defaults to taking 1 object from the first non-zero pile.

This approach ensures that the policy always returns a legal move and plays optimally by leveraging the mathematical properties of the Nim game.
