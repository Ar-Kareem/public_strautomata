### Thinking Process

The game described is a variant of the classic **Nim game**, but with a twist: the player who takes the last object **loses** (misère Nim). In standard Nim, the player who takes the last object wins, and the optimal strategy is based on the **Nim-sum** (XOR of all pile sizes). However, in misère Nim, the strategy is more complex, especially for small pile sizes.

#### Key Observations:
1. **Standard Nim Strategy**: In standard Nim, the player should aim to leave the opponent with a Nim-sum of 0. The Nim-sum is calculated as the XOR of all pile sizes. If the Nim-sum is non-zero, the player can force a win by making a move that leaves the Nim-sum as 0.
2. **Misère Nim Strategy**: In misère Nim, the strategy is similar to standard Nim except when all piles are of size 1. In such cases, the player must avoid taking the last object. The general approach is:
   - If the Nim-sum is non-zero, play as in standard Nim (leave Nim-sum 0).
   - If the Nim-sum is zero and there are multiple piles, the player should force the opponent into a position where they have to take the last object. This often involves leaving the opponent with all piles of size 1 (if the number of piles is odd, the opponent will lose).
   - If the Nim-sum is zero and there is only one pile, the player must take all but one object (if possible) to force the opponent to take the last object.

#### Special Cases:
- If all piles are 1, the player must take all but one pile (if possible) to leave the opponent with an odd number of piles of size 1 (forcing them to take the last object).
- If there is only one pile, the player must take all but one object (if the pile size > 1) to force the opponent to take the last object.

#### Implementation:
1. Calculate the Nim-sum (XOR of all pile sizes).
2. If the Nim-sum is non-zero, find a move that leaves the Nim-sum as 0 (standard Nim strategy).
3. If the Nim-sum is zero:
   - If there are multiple piles, try to leave the opponent with all piles of size 1 (if possible).
   - If there is only one pile, take all but one object (if possible).
   - If all piles are 1, take all but one pile (if possible) to leave an odd number of piles of size 1.

```python
def policy(piles: list[int]) -> str:
    # Calculate Nim-sum
    nim_sum = 0
    for pile in piles:
        nim_sum ^= pile

    # Case 1: Nim-sum is non-zero (standard Nim strategy)
    if nim_sum != 0:
        for i in range(4):
            if piles[i] > 0:
                target = nim_sum ^ piles[i]
                if target < piles[i]:
                    return f"{i},{piles[i] - target}"
    else:
        # Case 2: Nim-sum is zero (misère Nim strategy)
        # Subcase 2.1: All piles are 1 (must leave odd number of piles)
        if all(pile == 1 for pile in piles):
            # Take all but one pile (if possible)
            for i in range(4):
                if piles[i] == 1:
                    continue
                else:
                    return f"{i},{piles[i] - 1}"
            # If all piles are 1, take all but one (but this is impossible since all are 1)
            # So take from any pile (but this is a losing position, so just take 1 from any pile)
            return f"0,1"
        else:
            # Subcase 2.2: Not all piles are 1, try to leave all piles as 1
            # Find a pile where we can reduce it to 1
            for i in range(4):
                if piles[i] > 1:
                    return f"{i},{piles[i] - 1}"
            # If all piles are 1, this is a losing position (should not happen due to earlier check)
            return f"0,1"

    # Fallback (should not reach here if logic is correct)
    return "0,1"
```

### Explanation of the Code:
1. **Nim-sum Calculation**: The Nim-sum is computed as the XOR of all pile sizes.
2. **Non-zero Nim-sum**: If the Nim-sum is non-zero, the code finds a move that leaves the Nim-sum as 0 (standard Nim strategy).
3. **Zero Nim-sum**:
   - If all piles are 1, the code tries to leave an odd number of piles of size 1 by taking all but one object from a pile (if possible).
   - If not all piles are 1, the code tries to reduce a pile to 1 (to leave the opponent with all piles of size 1).
4. **Fallback**: A fallback move is included (though it should theoretically never be reached if the logic is correct).

This policy should be able to beat other standard or misère Nim strategies in the arena.
