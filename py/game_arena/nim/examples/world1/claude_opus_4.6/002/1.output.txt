## Strategy

This is a classic Nim game where the player who takes the last object **loses** (misère Nim). The optimal strategy for misère Nim is well-known:

1. **Compute the Nim-sum** (XOR of all pile sizes).
2. **If all piles have size ≤ 1** (endgame): If the number of non-empty piles is odd, you're in a losing position (you'll be forced to take the last object). If even, take from any non-empty pile.
3. **If some piles have size > 1** (normal play): 
   - If the Nim-sum is 0, you're in a losing position — make any legal move and hope the opponent errs.
   - If the Nim-sum is non-zero, find a pile where `pile_size XOR nim_sum < pile_size`, then reduce that pile to `pile_size XOR nim_sum`. **But** with the misère twist: after making the move, check if all remaining piles would be ≤ 1. If so, ensure an **odd** number of piles remain (so the opponent faces an even number of single-object piles, meaning they take the last one).

This is the complete optimal (perfect play) strategy for misère Nim.

def policy(piles: list[int]) -> str:
    nim_sum = 0
    for p in piles:
        nim_sum ^= p
    
    # Check if all piles are <= 1
    all_small = all(p <= 1 for p in piles)
    
    if all_small:
        # Misère endgame: we want to leave an odd number of piles with 1 object
        # (so opponent faces odd piles and is forced to take the last)
        # Actually: if we must move, we want to leave an EVEN number of 1-piles
        # so the opponent takes last. Wait, let me think carefully:
        # The player who takes the last object LOSES.
        # If there are k piles of 1, and it's our turn:
        #   - We must take 1 from some pile, leaving k-1 piles of 1 for opponent.
        #   - If k is odd: we take one, leaving even for opponent. Opponent and we alternate.
        #     k odd means we take the last one (lose). So k odd = losing.
        #   - If k is even: we take one, leaving odd for opponent. Opponent takes last (loses). k even = winning.
        # So if k is even, just take from any non-empty pile.
        # If k is odd, we're losing but must move anyway.
        for i in range(4):
            if piles[i] == 1:
                return f"{i},1"
    else:
        # Some pile has size > 1
        # Misère strategy: play like normal Nim, but aim to leave an ODD number of 
        # piles of size 1 (and no piles of size > 1) at the end.
        
        if nim_sum == 0:
            # Losing position, just make any legal move
            for i in range(4):
                if piles[i] > 0:
                    return f"{i},1"
        
        # Nim-sum is non-zero: we can make a winning move
        for i in range(4):
            if piles[i] > 0:
                target = piles[i] ^ nim_sum
                if target < piles[i]:
                    # This would be the normal Nim move: reduce pile i to target
                    # Check the misère condition: after this move, 
                    # would all piles be <= 1?
                    new_piles = piles[:]
                    new_piles[i] = target
                    
                    if all(p <= 1 for p in new_piles):
                        # We need to leave an ODD number of size-1 piles
                        ones = sum(1 for p in new_piles if p == 1)
                        if ones % 2 == 1:
                            take = piles[i] - target
                            return f"{i},{take}"
                        else:
                            # Adjust: instead of reducing to target, reduce to target^1
                            # which flips the parity. If target >= 1, reduce to target-1 or target+1
                            # Actually, reduce to 1 if target==0, or 0 if target==1
                            adjusted = 1 if target == 0 else 0
                            take = piles[i] - adjusted
                            if take >= 1:
                                return f"{i},{take}"
                    else:
                        take = piles[i] - target
                        return f"{i},{take}"
        
        # Fallback (shouldn't reach here with nim_sum != 0)
        for i in range(4):
            if piles[i] > 0:
                return f"{i},1"
    
    # Ultimate fallback
    for i in range(4):
        if piles[i] > 0:
            return f"{i},1"
