To address the problem of determining the next move in a game of Go on a 19x19 board, we can use a heuristic-based approach that prioritizes strategic positions and stone connectivity. The primary goal is to select a move that maximizes the player's chances of controlling territory and limiting the opponent's influence.

---

### **Strategy Overview**

The AI evaluates each legal move based on a move's **strategic importance** using the following components:

1. **Position Value**:
   - **Corners** (e.g., (1,1), (1,19), (19,1), (19,19)) are assigned the highest score due to their difficulty to invade in Go.
   - **Edge positions** (those on the first or last row or column, excluding corners) are given a moderate score.
   - **Central positions** receive the lowest base score, as they are more vulnerable to invasion.

2. **Adjacency Bonus**:
   - A move that connects with existing stones of the player is highly valued.
   - Each adjacent stone (orthogonally) contributes a bonus score to encourage expansion and group formation.

3. **Liberty Count**:
   - The number of **empty adjacent cells** (orthogonal only) around the proposed move is considered.
   - More liberties mean the stone is less likely to be captured, increasing the move's value.

The total score for a move is a weighted sum of these three components, favoring moves that are both strategic (near corners/edges) and advantageous in terms of adjacency and liberties.

---

### **Heuristic Weighting**

- **Position Value**: 5 for corners, 3 for edges, 1 otherwise.
- **Adjacency Bonus**: 3 per adjacent player stone.
- **Liberty Multiplier**: 1 per liberty (empty adjacent cell).

This balance ensures that:
- Corners are prioritized for early placement.
- Connecting to existing stones is encouraged to build larger groups.
- Moves with more liberties are safer and less likely to be captured.

---

### **Implementation**

The following Python code implements the strategy as described. It uses sets for efficient lookups and iterates through all board positions to evaluate each move.

```python
def policy(me: list[tuple[int, int]], opponent: list[tuple[int, int]]) -> tuple[int, int]:
    me_set = set(me)
    opponent_set = set(opponent)
    directions = [(-1, 0), (1, 0), (0, -1), (0, 1)]
    best_move = (0, 0)
    max_score = -float('inf')

    for x in range(1, 20):
        for y in range(1, 20):
            # Skip already occupied cells
            if (x, y) in me_set or (x, y) in opponent_set:
                continue

            # Position value: corners (5), edges (3), others (1)
            if x in (1, 19) and y in (1, 19):
                pos_val = 5
            elif x in (1, 19) or y in (1, 19):
                pos_val = 3
            else:
                pos_val = 1

            # Adjacency bonus: count adjacent me stones
            adj_count = 0
            for dx, dy in directions:
                nx, ny = x + dx, y + dy
                if (nx, ny) in me_set:
                    adj_count += 1
            adj_score = adj_count * 3

            # Liberties: empty adjacent cells
            liberties = 0
            for dx, dy in directions:
                nx, ny = x + dx, y + dy
                if 1 <= nx <= 19 and 1 <= ny <= 19:
                    if (nx, ny) not in me_set and (nx, ny) not in opponent_set:
                        liberties += 1

            # Total score for the move
            current_score = pos_val + adj_score + liberties

            # Update best move if this move is better
            if current_score > max_score:
                max_score = current_score
                best_move = (x, y)

    return best_move
```

---

### **Strategy Summary**

This policy is designed to be:
- **Robust** in early game by prioritizing corners.
- **Aggressive** in mid-game by favoring adjacent expansions.
- **Defensive** by favoring moves with more liberties, reducing the risk of capture.

The heuristic is simple, fast, and well-suited for real-time evaluation within the one-second time constraint. It balances between positional value and connectivity while leveraging the board geometry for optimal placement.
