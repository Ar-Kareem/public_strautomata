Here's the final Python code for the policy function that decides whether to "HIT" or "STAY" in the simplified Blackjack-like game:

The function `policy(hand, target)` evaluates the current hand's adjusted value and decides whether to draw another card ("HIT") or stop ("STAY"). The decision is based on:
- The **probability of busting** when drawing a card.
- The **expected improvement** in your hand's proximity to the target score.

---

### ✅ Key Logic Overview:

- **Hand Value Calculation**:  
  The function computes the adjusted hand value based on the target. Aces are treated as 11 initially, but if the sum exceeds the target, they are adjusted to 1 to avoid busting.

- **Bust Detection**:  
  If the current hand is already bust (sum of cards exceeds target), the function returns `"STAY"` immediately.

- **Remaining Cards Assessment**:  
  The code identifies which cards are still available in the deck and evaluates the risk and potential benefit of drawing each.

- **Bust Risk Evaluation**:  
  If drawing a new card would likely lead to a bust (probability > 50%), the function chooses to `"STAY"`.  
  Otherwise, it chooses to `"HIT"` to try to get closer to the target.
