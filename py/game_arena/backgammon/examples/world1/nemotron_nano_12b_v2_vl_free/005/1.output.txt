1. **Re-entering checkers from the bar** when available.
2. **Moving checkers toward the home board** using the dice values.
3. **Bearing off** when allowed (i.e., when all checkers are in the player’s home board).
4. **Valid moves must be played** when available; otherwise, the move is considered a "pass."

Here is a robust and clear implementation of the `policy` function that prioritizes re-entry when possible, then attempts to build a prime in the home board, and finally falls back to moving checkers closer to home.

### ✅ Policy Strategy Overview

1. **Bar Checkers First (Critical Rule)**  
   If the player has any checkers on the bar, they must be re-entered into the opponent's home board (points 18-23). If multiple re-entry options are possible, the higher die is used first.

2. **Bearing Off**  
   Once all checkers are in the home board (0–5), bearing off is prioritized. The most valuable bearing off move is chosen first.

3. **General Movement**  
   If no entry or bearing off is possible, the code will attempt to move checkers toward the home board, prioritizing the highest-value moves (i.e., toward the home board).

### ✅ Return Format

The function returns a legal move string in the format:  
`<ORDER>:<FROM1>,<FROM2>`  
Where:
- `<ORDER>` is `'H'` (higher die first) or `'L'` (lower die first).
- `<FROM1>` and `<FROM2>` are checker starting positions (e.g., `A0`, `B`, `P`).
