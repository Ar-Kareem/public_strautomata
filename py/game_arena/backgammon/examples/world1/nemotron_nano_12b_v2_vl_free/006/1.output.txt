**Strategy Overview:**  
This policy prioritizes efficient checker movement and optimal dice utilization based on current game state. The key principles are:
1. **Re-enter checkers from the bar** using the highest valid die first
2. **Move toward your home board** (points 0-5) when possible
3. **Prefer moves that trap opponent checkers** or consolidate positions
4. **Always use both dice when possible** and prioritize the **higher die** if required to move first
5. **Bear off** by moving checkers off your home board (points 0-5)

**Key Components Explained:**

1. **Bar Handling**: Checkers on the bar must be re-entered. The code tries to use the highest die first for re-entry.

2. **Move Generation**: For each valid dice permutation (considering both dice orders), generate all legal moves using recursive depth-2 search to handle multiple checker paths.

3. **Move Scoring**: Heuristic evaluation prioritizes:
   - Moving toward home (lower target points)
   - Hitting opponent singles with more own checkers behind
   - Using both dice when possible
   - Moving higher-numbered checkers in the home board

4. **Validity Checks**: Always verify move legality before returning, ensuring compliance with game rules.

5. **Formatting**: The `format_move` function converts the move structure into the required string format, correctly applying the H/L prefix based on die values.
