**Strategy overview**  

I’ll implement a dual-phase policy for Backgammon:  

1. **Bear-off priority** – If all checkers are in home board, apply an optimized bear-off using rule-based logic (biggest die moves biggest checker where possible).  

2. **Running game** – When not bearing off:  
   - Try to avoid leaving single checkers (“blots”) unless necessary.  
   - Prefer moving from higher points to clear the backfield when safe.  
   - If opponent has fewer borne-off checkers, play more defensively; else, prioritize advancing back checkers.  

3. **Bar re-entry** – If any checkers on the bar, choose highest legal point that does not leave another blot if possible.  

4. **Safe vs aggressive tradeoff** – Use a simple evaluation function:  
   - Score = (my borne-off) – (opp borne-off) + (0.1 × home-board control) – (0.3 × blot count).  
   Choose the legal move maximizing this score.  

5. **Move ordering** – We test all legal moves generated by a helper; if none found (pass), return `"H:P,P"`.  

The policy will be deterministic, fast, and avoids illegal moves by filtering via engine constraints.
