To build a strong backgammon policy, I need to:
1. Always return a legal move—this is critical to avoid disqualification.
2. Evaluate moves strategically to win games against other AIs.
3. Handle all game phases: opening, midgame, bearing off.

Key strategic principles:
- **Safety first**: Avoid leaving single checkers (blots) if possible, especially when opponent can hit.
- **Hit opponent** when safe, especially if it sends them to the bar.
- **Build primes** (consecutive blocked points) to trap opponent.
- **Run** when ahead in the race (bearing-off phase).
- **Use both dice** whenever possible—wasting moves is bad.
- **Prioritize entering from bar** if blocked.

Since the state representation uses absolute points 0–23, but I don’t know which direction each player moves, I must rely on the engine to enforce movement rules. My policy will generate all legal moves given the dice and state, then choose the best using a heuristic evaluation.

**Heuristic evaluation** will consider:
1. **Race advantage**: difference in pip count (total distance to bear off).
2. **Board safety**: number of blots after move.
3. **Blocking**: points with 2+ of my checkers.
4. **Hitting**: sending opponent to bar.
5. **Bearing off progress**.

I’ll implement a simple one-ply lookahead: for each legal move, compute the resulting board, evaluate with a scoring function, pick the highest score.

Edge cases:
- If no legal move except pass, return `H:P,P`.
- If only one die can be played, the move string still includes two `FROM` tokens (second is `P`).
- Must handle `state['dice']` length 0, 1, or 2.

**Implementation plan**:
1. Parse state.
2. Generate all legal moves via brute-force simulation: try moving each possible checker for each die order, respecting bar and hit rules.
3. Score each resulting board.
4. Choose best move, format string.
5. Fallback to any legal move if something fails.
