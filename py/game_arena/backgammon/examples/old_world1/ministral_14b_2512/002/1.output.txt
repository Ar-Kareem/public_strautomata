### Strategy Overview

My policy combines several key principles to create a strong Backgammon AI:

1. **Prioritize Bar Moves**: If I have checkers on the bar, I'll always move them first (as required by rules) using the highest possible die to maximize entry points.

2. **Avoid Blots**: I'll avoid moving onto points occupied by 2+ opponent checkers (as per rules), but I'll also try to avoid creating blots (single opponent checkers) when possible.

3. **Home Board Focus**: When bearing off is allowed, I'll prioritize moving checkers from the highest points first to maximize scoring opportunities.

4. **Doubling Dice**: When both dice can be played, I'll always play both (as required). When only one die can be played, I'll use the higher die.

5. **Positional Play**:
   - In the opening phase, I'll aim to build anchors (multiple checkers on key points)
   - In the middle game, I'll try to create prime (unbreakable walls) to block opponent progress
   - In the endgame, I'll focus on bearing off efficiently

6. **Probability-Based Decisions**: I'll use simple probability calculations to evaluate move safety and effectiveness.

7. **Special Cases Handling**:
   - If I have a prime, I'll try to extend it
   - If I'm about to be hit, I'll prioritize defensive moves
   - If I have multiple good options, I'll choose the one that creates the most advantageous position

The implementation uses a combination of rule-based checks and simple evaluation functions to select moves that maximize both immediate and long-term advantages.

### Key Features of the Implementation:

1. **Legal Move Generation**: First generates all legal moves according to the rules.

2. **Move Evaluation**: Uses a scoring system that considers:
   - Moving from the bar (high priority)
   - Position relative to home board
   - Avoiding blots
   - Bearing off opportunities
   - Defensive considerations
   - Prime extension/creation

3. **Double Dice Handling**: When both dice can be played, evaluates all possible combinations to find the best pair.

4. **Order Determination**: Properly determines whether to use 'H' or 'L' based on die values.

5. **Fallback Handling**: If no good combination is found, falls back to the best single move.

This implementation balances immediate tactical advantages with longer-term strategic considerations while strictly adhering to all game rules and legal move constraints.
